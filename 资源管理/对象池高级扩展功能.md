# 对象池高级扩展功能

## 摘要
在掌握了基础对象池后，高级扩展功能成为应对复杂场景的关键。本文介绍如何基于 ObjectPoolPro 进一步扩展对象池功能，实现泛型支持、自动回收和场景隔离等高级特性，满足企业级项目的需求。

## 正文

### 背景
在上一节中，我们实现了一个功能完善的对象池工具 ObjectPoolPro，用于优化频繁生成和销毁的 GameObject 对象。本节将在此基础上进一步扩展对象池功能，增加更高级的特性，包括：

- **泛型对象池 GenericObjectPool**：支持非 GameObject 类型的对象复用，可用于逻辑对象（Buff、行为树节点、路径点等）的池化管理。
- **AutoRecycle 自动回收组件**：可挂载在 GameObject 上的脚本，在物体失活或指定时间后自动回收到对象池。
- **场景隔离机制**：按场景 ID 分类管理对象池，场景切换时自动清理，防止跨场景资源残留。

接下来分三部分详细讲解每个特性的实现。

### 一、泛型对象池 GenericObjectPool

#### 为什么需要泛型对象池？
不仅是游戏物体需要池化，纯逻辑对象（技能 Buff、AI 行为树节点、路径点）被频繁创建和销毁同样会产生 GC 压力。泛型对象池可以复用任意类型的对象。

```csharp
using System.Collections.Generic;
using UnityEngine;

public static class GenericObjectPool<T> where T : class, new()
{
    private static readonly Stack<T> _pool = new Stack<T>();

    public static T Get()
    {
        return _pool.Count > 0 ? _pool.Pop() : new T();
    }

    public static void Recycle(T obj)
    {
        if (obj is IPoolable poolable)
        {
            poolable.OnRecycle();
        }
        _pool.Push(obj);
    }

    public static void Clear()
    {
        _pool.Clear();
    }
}

public interface IPoolable
{
    void OnRecycle();
}
```

使用示例：

```csharp
public class Buff : IPoolable
{
    public string BuffName { get; set; }
    public float Duration { get; set; }

    public void Activate() { /* 激活逻辑 */ }

    public void OnRecycle()
    {
        BuffName = null;
        Duration = 0f;
    }
}

// 使用
var buff = GenericObjectPool<Buff>.Get();
buff.BuffName = "加速";
buff.Activate();
// 用完回收
GenericObjectPool<Buff>.Recycle(buff);
```

**设计要点**：
- 泛型约束 `where T : class, new()` 确保可实例化
- `IPoolable` 接口让对象在回收时执行重置逻辑，避免状态残留
- 容量上限和预热功能可根据项目需求添加

### 二、AutoRecycle 自动回收组件

自动回收组件挂载在池化对象上，在物体失活或超时后自动回池，无需手动调用 Recycle：

```csharp
using UnityEngine;
using System.Collections;

public class AutoRecycle : MonoBehaviour
{
    [Tooltip("超时自动回收（秒），0表示禁用超时回收")]
    public float timeout = 0f;

    private WaitForSeconds _wait;
    private Coroutine _timeoutRoutine;
    private IObjectPool<GameObject> _pool;

    public void Initialize(IObjectPool<GameObject> pool)
    {
        _pool = pool;
    }

    private void OnEnable()
    {
        if (timeout > 0f)
        {
            _wait ??= new WaitForSeconds(timeout);
            _timeoutRoutine = StartCoroutine(TimeoutRecycle());
        }
    }

    private void OnDisable()
    {
        if (_timeoutRoutine != null)
        {
            StopCoroutine(_timeoutRoutine);
            _timeoutRoutine = null;
        }
        if (_pool != null && gameObject.activeInHierarchy == false)
        {
            _pool.Release(gameObject);
            _pool = null;
        }
    }

    private IEnumerator TimeoutRecycle()
    {
        yield return _wait;
        gameObject.SetActive(false);
    }
}
```

**使用要点**：
- 在对象从池中取出时调用 `autoRecycle.Initialize(pool)` 绑定所属池
- `OnDisable` 中判断：如果是超时或主动 SetActive(false)，自动回池
- 超时回收可防止对象因逻辑遗漏而永远留在场景中

### 三、场景隔离机制

多场景切换时，对象池中的资源如果不释放会持续累积。场景隔离机制为每个场景建立独立的对象池：

```csharp
using System.Collections.Generic;
using UnityEngine;
using UnityEngine.SceneManagement;

public static class ObjectPoolPro
{
    private static Dictionary<int, Dictionary<GameObject, IObjectPool<GameObject>>> _scenePools;
    private static Dictionary<GameObject, GameObject> _instanceToPrefabMap;

    static ObjectPoolPro()
    {
        _scenePools = new Dictionary<int, Dictionary<GameObject, IObjectPool<GameObject>>>();
        _instanceToPrefabMap = new Dictionary<GameObject, GameObject>();
        SceneManager.sceneUnloaded += OnSceneUnloaded;
    }

    private static int CurrentSceneId => SceneManager.GetActiveScene().buildIndex;

    public static void RegisterPrefab(GameObject prefab, int preloadCount)
    {
        var sceneId = CurrentSceneId;
        if (!_scenePools.ContainsKey(sceneId))
            _scenePools[sceneId] = new Dictionary<GameObject, IObjectPool<GameObject>>();

        var pools = _scenePools[sceneId];
        if (!pools.ContainsKey(prefab))
        {
            pools[prefab] = new UnityEngine.Pool.ObjectPool<GameObject>(
                createFunc: () => GameObject.Instantiate(prefab),
                actionOnGet: obj => obj.SetActive(true),
                actionOnRelease: obj => obj.SetActive(false),
                actionOnDestroy: Object.Destroy
            );
        }

        var pool = pools[prefab];
        for (int i = 0; i < preloadCount; i++)
        {
            var obj = pool.Get();
            pool.Release(obj);
        }
    }

    public static GameObject Get(GameObject prefab)
    {
        RegisterPrefab(prefab, 0);
        var sceneId = CurrentSceneId;
        var pool = _scenePools[sceneId][prefab];
        var instance = pool.Get();
        _instanceToPrefabMap[instance] = prefab;
        return instance;
    }

    public static void Recycle(GameObject instance)
    {
        if (_instanceToPrefabMap.TryGetValue(instance, out var prefab))
        {
            var sceneId = CurrentSceneId;
            if (_scenePools.TryGetValue(sceneId, out var pools) && pools.TryGetValue(prefab, out var pool))
            {
                pool.Release(instance);
                _instanceToPrefabMap.Remove(instance);
                return;
            }
        }
        Object.Destroy(instance);
    }

    private static void OnSceneUnloaded(Scene scene)
    {
        var sceneId = scene.buildIndex;
        if (_scenePools.TryGetValue(sceneId, out var pools))
        {
            foreach (var pool in pools.Values)
                pool.Clear();
            _scenePools.Remove(sceneId);
        }

        // 清理已销毁实例的映射
        var toRemove = new List<GameObject>();
        foreach (var kv in _instanceToPrefabMap)
        {
            if (kv.Key == null)
                toRemove.Add(kv.Key);
        }
        foreach (var key in toRemove)
            _instanceToPrefabMap.Remove(key);
    }
}
```

**核心设计**：
- 以场景 ID 为外层键，预制体为内层键，构建二维池字典
- `instanceToPrefabMap` 记录实例到预制体的映射，方便回收时找到归属池
- `sceneUnloaded` 事件中清理对应场景的所有池
- 使用 `UnityEngine.Pool.ObjectPool`（Unity 2021+ 内置）而非手写池

**多场景 additive 模式**：若使用 additive 多场景加载，可将 `CurrentSceneId` 改为从实例的 `obj.scene.buildIndex` 获取，而非当前活动场景。

### 实现方案

1. **逻辑对象泛型池**：为频繁 new/GC 的逻辑类（Buff、行为树节点、消息体）建立 `GenericObjectPool<T>`，实现 `IPoolable` 接口进行重置。
2. **AutoRecycle 组件**：所有池化物体挂载 AutoRecycle，通过 `Initialize` 绑定所属池，`OnDisable` 自动回收。
3. **场景隔离**：用 `ObjectPoolPro` 静态类管理所有池，以场景 ID + 预制体构建二维字典，订阅 `SceneManager.sceneUnloaded` 清理。
4. **预热策略**：在场景初始化阶段调用 `RegisterPrefab` 预先创建对象，避免运行时首次 Get 的卡顿。

### 总结
对象池的高级扩展让对象管理从"能跑"走向"工业化"。泛型池降低了逻辑对象的 GC 压力，AutoRecycle 简化了回收流程，场景隔离解决了跨场景资源泄漏。三者的组合构成了大型 Unity 项目中资源管理的坚实基础。
