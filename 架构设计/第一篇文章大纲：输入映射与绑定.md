

### 第一篇文章大纲：输入映射与绑定

本篇文章将从最基础的输入映射概念入手，对比 Unreal 的传统输入系统与 Unity 的新旧输入系统，并深入讲解如何进行按键和轴的绑定。

1.  **引言：**
    
    -   Unity 的旧输入管理器简单直观，但扩展性差；新输入系统功能强大，但概念复杂。
        
    -   Unreal 的输入系统在设计上更强调“映射”和“绑定”的概念，这与 Unity 的新输入系统有异曲同工之妙。
        
2.  **Unity 输入系统回顾：**
    
    -   **旧输入管理器（Input Manager）：**
        
        -   讲解如何通过 `Input.GetButton/GetAxis` 来查询按键或轴的状态。
            
        -   优点是简单，缺点是硬编码和难以维护。
            
    -   **新输入系统（Input System Package）：**
        
        -   讲解**输入动作（Input Actions）**的概念，以及如何通过 `PlayerInput` 组件进行事件回调。
            
        -   强调其**可重绑定**、**多设备支持**等优点。
            
3.  **Unreal 传统输入系统详解：**
    
    -   **输入映射（Input Mappings）：**
        
        -   讲解在**项目设置（Project Settings）**中配置 **Action Mappings** 和 **Axis Mappings**。
            
        -   **Action Mappings：** 用于处理离散的按键事件，如“跳跃”、“开火”。
            
        -   **Axis Mappings：** 用于处理连续的轴向输入，如“移动”、“转向”。
            
    -   **输入绑定（Input Bindings）：**
        
        -   讲解如何在 C++ 或蓝图中，通过 `InputComponent->BindAction` 和 `BindAxis` 来将配置好的映射绑定到对应的函数或事件上。
            
        -   **在蓝图中：** 演示如何使用蓝图中的“Input Action”和“Input Axis”事件节点。
            
4.  **UI 输入模式：**
    
    -   **Unreal 的输入模式：**
        
        -   讲解 `SetInputMode` 函数，以及 `Game Only`, `UI Only`, `Game and UI` 三种模式的区别。
            
        -   演示如何在打开 UI 时将输入模式切换为 `UI Only`，从而屏蔽游戏中的输入。
            
    -   **与 Unity 对比：**
        
        -   讲解 Unity 如何通过 `EventSystem` 控制 UI 输入，以及如何手动管理鼠标锁定等。
            
5.  **核心对比与迁移思路：**
    
    -   **Unity `Input Manager`** -> **Unreal `Input Mappings`**。
        
    -   **`Input.GetButton/GetAxis`** -> **`BindAction/BindAxis`**。
        
    -   总结 Unreal 的传统输入系统在设计上比 Unity 的旧系统更加清晰和可维护，但仍存在一些局限性。

### 第二篇文章大纲：Enhanced Input 详解与移动端输入处理

本篇文章将深入讲解 Unreal Engine 5 推荐的**增强输入**系统，并探讨如何处理移动设备上的触摸和传感器输入。

1.  **引言：**
    
    -   随着游戏复杂度的提升，传统的输入系统已无法满足需求。
        
    -   Unreal 的**增强输入**系统和 Unity 的**新输入系统**都旨在解决这些问题。
        
2.  **Unreal Enhanced Input（增强输入）详解：**
    
    -   **核心概念：**
        
        -   **输入动作（Input Actions）：** 定义一个抽象的动作，如“跳跃”或“移动”。
            
        -   **输入映射上下文（Input Mapping Contexts）：** 将输入设备上的按键（如空格键）映射到特定的输入动作。
            
    -   **优点：**
        
        -   **可重绑定：** 玩家可以在运行时轻松地重新绑定按键。
            
        -   **多设备支持：** 可以为键盘、手柄等不同设备定义独立的映射。
            
        -   **修改器（Modifiers）与触发器（Triggers）：**
            
            -   **Modifiers：** 在输入处理前修改原始输入值，如“取反”、“死区”。
                
            -   **Triggers：** 定义何时触发输入动作，如“按下时”、“长按时”。
                
3.  **Enhanced Input 与传统输入的对比：**
    
    -   讲解增强输入如何通过更高级的抽象层，解决了传统输入系统中的许多问题，如上下文切换和运行时重绑定。
        
    -   **与 Unity 新输入系统的对比：** 强调两者在设计理念上的高度相似性，都致力于提供更强大、灵活的输入处理方案。
        
4.  **移动端输入处理：**
    
    -   **触摸输入：**
        
        -   讲解 Unreal 如何通过**蓝图事件**（如 `OnTouchBegan`, `OnTouchMoved`）来处理触摸输入。
            
        -   **与 Unity 对比：** 讲解 Unity 的 `Input.touches` 接口。
            
    -   **传感器输入：**
        
        -   讲解如何使用 Unreal 的**重力感应**组件来获取移动设备的陀螺仪和加速度计数据。
            
        -   **与 Unity 对比：** 讲解 Unity 的 `Input.acceleration` 接口。
            
5.  **核心对比与迁移思路：**
    
    -   **Unity 新输入系统** -> **Unreal 增强输入**。
        
    -   总结：Unreal 的增强输入系统功能强大，设计严谨，如果你正在开发一个需要复杂输入控制的项目，强烈建议直接使用它。
