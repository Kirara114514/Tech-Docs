### 第二篇：从零开始：一个基础且通用的事件总线

上一篇我们探讨了为什么需要事件总线，以及它在解耦中的核心作用。现在，是时候卷起袖子，亲手实现一个基础但强大的事件总线了。我们将一步步构建一个能够处理**不同类型事件**，并且**类型安全（Type-safe）**的通用事件总线。

#### 2.1 准备工作：事件的定义

在实现事件总线之前，我们首先需要定义“事件”本身。在 C# 中，一个事件通常被封装成一个**数据结构**，用来携带事件发生时的所有相关信息。

**事件数据**

我们推荐使用一个 **`struct`** 或 **`class`** 来封装事件数据。为什么？因为它能提供清晰的语义，并且可以携带复杂的参数。

C#

```
// 攻击事件的数据结构
public struct PlayerAttackEvent
{
    public string attackerName; // 攻击者的名字
    public int damage;          // 造成的伤害
}

// 怪物死亡事件的数据结构
public class MonsterDiedEvent
{
    public int monsterId;       // 死亡怪物的ID
    public Vector3 deathPosition; // 死亡时的位置
}

```

使用 `struct` 和 `class` 的好处是显而易见的：

-   **清晰**：事件的意图一目了然。`PlayerAttackEvent` 告诉我们这是一个“玩家攻击”事件。
    
-   **可扩展**：如果以后攻击事件需要新增一个“暴击率”的字段，你只需要在 `PlayerAttackEvent` 结构体里添加一个 `float criticalChance;` 即可，而不需要修改任何发布或订阅的代码。
    
-   **类型安全**：C# 的泛型会在编译时帮我们检查类型，确保你发布的是 `PlayerAttackEvent`，订阅的也是 `PlayerAttackEvent`，从而避免了类型转换错误。
    

#### 2.2 核心实现：EventBus.cs

现在我们开始创建事件总线的核心类。我们将把 `EventBus` 设计成一个**单例（Singleton）**，以确保在整个项目中只有一个实例，并且可以全局访问。

**`EventBus.cs`**

C#

```
using System;
using System.Collections.Generic;

public class EventBus
{
    // 事件总线的单例
    public static EventBus Instance { get; } = new EventBus();

    // 存储所有事件订阅者的数据结构
    private readonly Dictionary<Type, List<Delegate>> _eventDictionary = new Dictionary<Type, List<Delegate>>();

    // 私有构造函数，防止外部直接实例化
    private EventBus() { }
}

```

**数据结构剖析：`Dictionary<Type, List<Delegate>>`**

这是整个事件总线的心脏。

-   **`Dictionary<Type, ...>`**：我们使用 `System.Type` 作为 Key，这意味着每一种事件类型（比如 `PlayerAttackEvent`）都会在字典中有一个唯一的条目。这使得我们能高效地查找特定事件的订阅者。
    
-   **`List<Delegate>`**：`Value` 是一个 `List`，用来存储所有订阅了该事件的**委托（Delegate）**。**`Delegate` 是 C# 中一个强大的概念，你可以将其理解为指向一个或多个方法的引用。** `Action<T>` 和 `Func<T>` 都是 `Delegate` 的特殊形式。一个事件可能有多个订阅者，所以我们需要一个列表来保存所有的方法引用。
    

**核心方法实现**

接下来，我们实现事件总线最重要的三个方法：`Subscribe`、`Unsubscribe` 和 `Publish`。

**1. `Subscribe<T>(Action<T> action)` - 订阅事件**

这个方法接收一个 `Action<T>` 委托作为参数。`T` 代表事件类型，`action` 是当该类型事件发生时需要执行的方法。

C#

```
// 订阅事件
public void Subscribe<T>(Action<T> action) where T : struct
{
    Type eventType = typeof(T);

    // 如果字典中还没有该事件类型，则先添加一个新列表
    if (!_eventDictionary.ContainsKey(eventType))
    {
        _eventDictionary[eventType] = new List<Delegate>();
    }

    // 将方法添加到该事件类型的订阅者列表中
    _eventDictionary[eventType].Add(action);
}

```

> **为什么要使用 `where T : struct` 约束？**
> 
> 这是一个设计上的选择，我们这里选择限制事件类型必须是 `struct`，即**值类型（Value Type）**。这样做的好处是，可以避免频繁创建 `class` 实例而产生的 **GC Allocations（垃圾回收分配）**，这对于高频率的事件发布非常重要。当然，你也可以选择不加这个约束，让 `class` 也能作为事件类型。在下一篇中我们会讨论如何处理 `class` 带来的性能问题。

**2. `Unsubscribe<T>(Action<T> action)` - 取消订阅**

取消订阅是防止**内存泄漏**的关键步骤。如果一个 `GameObject` 订阅了事件，但在被销毁时忘记取消订阅，那么事件总线里还会保留这个 `GameObject` 上的方法引用，导致 `GameObject` 无法被垃圾回收，形成“野指针”和内存泄漏。

C#

```
// 取消订阅事件
public void Unsubscribe<T>(Action<T> action) where T : struct
{
    Type eventType = typeof(T);

    // 检查字典中是否存在该事件类型
    if (_eventDictionary.ContainsKey(eventType))
    {
        // 从订阅者列表中移除该方法
        _eventDictionary[eventType].Remove(action);
    }
}

```

**3. `Publish<T>(T eventData)` - 发布事件**

这个方法接收一个事件数据 `eventData` 作为参数。它会遍历所有订阅了该事件的委托，并依次执行它们。

C#

```
// 发布事件
public void Publish<T>(T eventData) where T : struct
{
    Type eventType = typeof(T);

    // 检查字典中是否存在该事件类型
    if (_eventDictionary.ContainsKey(eventType))
    {
        // 遍历所有订阅者，依次调用他们的方法
        foreach (var actionDelegate in _eventDictionary[eventType])
        {
            // 将委托转换为正确的Action<T>类型并执行
            // 这里使用了(actionDelegate as Action<T>)?.Invoke()来安全地调用
            (actionDelegate as Action<T>)?.Invoke(eventData);
        }
    }
}

```

**完整的 `EventBus.cs` 代码**

将上述代码组合起来，一个基础且通用的事件总线就完成了。

C#

```
using System;
using System.Collections.Generic;

public sealed class EventBus
{
    // 单例模式，保证全局唯一
    public static EventBus Instance { get; } = new EventBus();

    // 存储事件订阅者的数据结构
    private readonly Dictionary<Type, List<Delegate>> _eventDictionary = new Dictionary<Type, List<Delegate>>();

    private EventBus() { }

    // 订阅事件
    public void Subscribe<T>(Action<T> action) where T : struct
    {
        Type eventType = typeof(T);
        if (!_eventDictionary.ContainsKey(eventType))
        {
            _eventDictionary[eventType] = new List<Delegate>();
        }
        _eventDictionary[eventType].Add(action);
    }

    // 取消订阅事件
    public void Unsubscribe<T>(Action<T> action) where T : struct
    {
        Type eventType = typeof(T);
        if (_eventDictionary.ContainsKey(eventType))
        {
            _eventDictionary[eventType].Remove(action);
        }
    }

    // 发布事件
    public void Publish<T>(T eventData) where T : struct
    {
        Type eventType = typeof(T);
        if (_eventDictionary.ContainsKey(eventType))
        {
            // 创建一个临时的列表，防止在遍历时列表被修改
            // 这是一种常见的防御性编程技巧
            var listeners = new List<Delegate>(_eventDictionary[eventType]);

            foreach (var actionDelegate in listeners)
            {
                (actionDelegate as Action<T>)?.Invoke(eventData);
            }
        }
    }
}

```

在 `Publish` 方法中，我们引入了一个小技巧：`new List<Delegate>(_eventDictionary[eventType])`。这样做是为了**防止在遍历订阅者列表时，有订阅者在事件响应方法里又取消了订阅，导致列表被修改，从而引发异常。** 这是一种非常好的防御性编程习惯。

#### 2.3 使用示例：重现第一篇的场景

现在，让我们回到第一篇的场景，用我们刚刚实现的 `EventBus` 来重构代码。

**1. 定义事件数据**

首先，我们定义一个**事件结构体**：

C#

```
// 事件数据，包含攻击者的名字和伤害值
public struct PlayerAttackEvent
{
    public string attackerName;
    public int damage;
}

```

**2. 发布事件**

在 `AttackButton.cs` 中，我们不再直接调用其他脚本，而是通过 `EventBus.Instance` 来发布事件。

C#

```
public class AttackButton : MonoBehaviour
{
    public string playerName = "玩家A";
    public int damage = 10;

    public void OnClick()
    {
        // 创建事件数据
        PlayerAttackEvent eventData = new PlayerAttackEvent
        {
            attackerName = playerName,
            damage = damage
        };

        // 通过事件总线发布事件
        EventBus.Instance.Publish(eventData);
    }
}

```

**3. 订阅事件**

在 `Monster.cs` 中，我们订阅 `PlayerAttackEvent`，并在 `OnDestroy` 中取消订阅，防止内存泄漏。

C#

```
public class Monster : MonoBehaviour
{
    private int currentHealth = 100;

    private void OnEnable()
    {
        // 订阅事件
        EventBus.Instance.Subscribe<PlayerAttackEvent>(OnPlayerAttack);
    }

    private void OnDisable()
    {
        // 在对象禁用时取消订阅
        EventBus.Instance.Unsubscribe<PlayerAttackEvent>(OnPlayerAttack);
    }

    private void OnPlayerAttack(PlayerAttackEvent eventData)
    {
        // 受到伤害
        currentHealth -= eventData.damage;
        Debug.Log($"{gameObject.name} 受到来自 {eventData.attackerName} 的 {eventData.damage} 点伤害，剩余血量: {currentHealth}");
    }
}

```

> **`OnEnable()` 和 `OnDisable()`**
> 
> 在 Unity 开发中，通常推荐在 `OnEnable()` 中进行订阅，在 `OnDisable()` 中进行取消订阅。因为 `OnDisable()` 总是在 `OnDestroy()` 之前被调用，这样可以确保即使 `GameObject` 被禁用（比如被设置为非激活状态），也能及时地取消订阅，避免不必要的事件触发。

**4. 额外功能：播放音效**

现在，如果我们要添加一个播放音效的功能，只需要新建一个 `AudioManager.cs` 脚本，并让它也去订阅事件。

C#

```
public class AudioManager : MonoBehaviour
{
    // ...

    private void OnEnable()
    {
        // 订阅攻击事件
        EventBus.Instance.Subscribe<PlayerAttackEvent>(OnPlayerAttack);
    }

    private void OnDisable()
    {
        // 取消订阅
        EventBus.Instance.Unsubscribe<PlayerAttackEvent>(OnPlayerAttack);
    }

    private void OnPlayerAttack(PlayerAttackEvent eventData)
    {
        Debug.Log("播放攻击音效...");
        // 播放音效的逻辑...
    }
}

```
