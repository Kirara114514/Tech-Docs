# 回溯与贪心

## 摘要
回溯法和贪心算法是两种截然不同但同样强大的算法设计技巧。回溯通过 DFS + 剪枝穷举所有可能解，适用于组合、排列和子集问题；贪心则在每一步做局部最优选择，适用于区间调度、零钱兑换等具有贪心选择性质的问题。本文系统介绍回溯的三步框架（选择→约束→目标）、剪枝优化策略以及贪心的正确性证明思维。

## 正文

### 背景
并非所有问题都适合用动态规划解决。当问题涉及组合、排列或约束满足时需要回溯法；当每一步的局部最优能直接推导全局最优时就需要贪心算法。理解这两种思想的差异和适用边界，是算法设计能力的关键跃升。

### 回溯法（Backtracking）

回溯法本质上是一种**深度优先搜索（DFS）**，但加入了**剪枝（Pruning）**优化。当算法发现当前路径无论如何也无法得到有效解时，就会**回溯**到上一个决策点，尝试其他路径。

#### 回溯三步框架

```
def backtrack(路径, 选择列表):
    if 满足结束条件:
        记录解
        return
    
    for 选择 in 选择列表:
        做选择 (路径.add(选择))
        backtrack(更新后的路径, 更新后的选择列表)
        撤销选择 (路径.remove(选择))
```

#### 三个核心问题

**1. 路径**：已经做出的选择
**2. 选择列表**：当前可以做的选择
**3. 结束条件**：到达决策树底部的条件

#### 经典回溯问题

**全排列（LeetCode 46）**

```csharp
IList<IList<int>> Permute(int[] nums)
{
    var result = new List<IList<int>>();
    var track = new List<int>();
    bool[] used = new bool[nums.Length];
    Backtrack(nums, used, track, result);
    return result;
}

void Backtrack(int[] nums, bool[] used, List<int> track, IList<IList<int>> result)
{
    if (track.Count == nums.Length)
    {
        result.Add(new List<int>(track)); // 拷贝结果
        return;
    }
    
    for (int i = 0; i < nums.Length; i++)
    {
        if (used[i]) continue;  // 剪枝：跳过已使用的数字
        track.Add(nums[i]);     // 做选择
        used[i] = true;
        Backtrack(nums, used, track, result);
        track.RemoveAt(track.Count - 1); // 撤销选择
        used[i] = false;
    }
}
```

**N 皇后（LeetCode 51）**

```csharp
IList<IList<string>> SolveNQueens(int n)
{
    var result = new List<IList<string>>();
    int[] queens = new int[n]; // queens[row] = col
    Array.Fill(queens, -1);
    Backtrack(queens, 0, result);
    return result;
}

bool IsValid(int[] queens, int row, int col)
{
    for (int i = 0; i < row; i++)
    {
        if (queens[i] == col) return false;                       // 同一列
        if (Math.Abs(queens[i] - col) == row - i) return false;   // 对角线
    }
    return true;
}

void Backtrack(int[] queens, int row, IList<IList<string>> result)
{
    if (row == queens.Length) { /* 记录结果 */ return; }
    for (int col = 0; col < queens.Length; col++)
    {
        if (!IsValid(queens, row, col)) continue; // 剪枝
        queens[row] = col;
        Backtrack(queens, row + 1, result);
        queens[row] = -1;
    }
}
```

#### 回溯剪枝优化

| 优化技巧 | 说明 | 经典应用 |
|---------|------|---------|
| 可行性剪枝 | 当前选择违反约束立即跳过 | N皇后 |
| 重复剪枝 | 相同数值在同一层去重 | 子集II、组合II |
| 排序剪枝 | 先排序，跳过相同值 | 组合总和II |
| 最优性剪枝 | 当前部分解已不如已知最优 | 旅行商 |

**去重通用模板**（如子集II）：

```csharp
for (int i = start; i < nums.Length; i++)
{
    if (i > start && nums[i] == nums[i-1]) continue; // 同一层跳过重复
    // ...
}
```

### 贪心算法（Greedy Algorithm）

贪心算法在每一步都做出当前看起来最好的选择，期望通过一系列局部最优选择达到全局最优。

#### 贪心正确性的证明

贪心算法的最大难点不在于实现，而在于**证明贪心选择能导出全局最优解**。常用证明方法：

1. **交换论证**：假设最优解与贪心解不同，交换其中一步可证明贪心解不劣于最优解
2. **归纳法**：证明贪心选择的子问题与原问题结构相同
3. **反证法**：假设贪心选择不最优，导出矛盾

#### 经典贪心问题

**1. 区间调度（LeetCode 435, 452）**

```csharp
// 无重叠区间 - 按结束时间排序，选最早结束的
int EraseOverlapIntervals(int[][] intervals)
{
    Array.Sort(intervals, (a, b) => a[1].CompareTo(b[1]));
    int end = intervals[0][1];
    int count = 1; // 已选区间数
    
    for (int i = 1; i < intervals.Length; i++)
    {
        if (intervals[i][0] >= end)
        {
            count++;
            end = intervals[i][1];
        }
    }
    return intervals.Length - count;
}
```

**2. 分发饼干（LeetCode 455）**

```csharp
int FindContentChildren(int[] g, int[] s)
{
    Array.Sort(g);
    Array.Sort(s);
    int i = 0, j = 0;
    while (i < g.Length && j < s.Length)
    {
        if (s[j] >= g[i]) i++; // 满足一个孩子
        j++;
    }
    return i;
}
```

**3. 买卖股票最佳时机 II（LeetCode 122）**

```csharp
int MaxProfit(int[] prices)
{
    int profit = 0;
    for (int i = 1; i < prices.Length; i++)
    {
        if (prices[i] > prices[i-1])
            profit += prices[i] - prices[i-1]; // 每天赚差价
    }
    return profit;
}
```

### 回溯 vs 贪心 vs DP

| 维度 | 回溯 | 贪心 | DP |
|------|------|------|-----|
| 搜索策略 | 穷举+剪枝 | 单路径 | 系统化填表 |
| 时间 | O(指数) | O(n)~O(n log n) | O(n)~O(n²) |
| 空间 | O(递归深度) | O(1) | O(n)~O(n²) |
| 适用 | 所有解/组合/约束满足 | 贪心选择性质 | 最优子结构 |
| 正确性保证 | 穷举保证 | 需证明 | 无后效性保证 |

### 实现方案
1. **回溯先写框架**：按「选择→约束→目标」三步搭建骨架，后再加剪枝
2. **去重技巧**：排序 + `i > start && nums[i] == nums[i-1]` 同一层去重
3. **贪心先证明**：写代码前先尝试用交换论证证明贪心选择的正确性
4. **调试思路**：回溯用小型输入手动走一遍决策树；贪心用反例测试验证

### 总结
回溯是「试错」的艺术——尝试所有可能并剪掉不可能的分支；贪心是「抉择」的艺术——每一步选最优且证明这能通向全局最优。掌握回溯的三步框架和贪心的证明思维，就能在组合优化和决策类问题中游刃有余。
