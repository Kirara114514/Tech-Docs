## 第九篇：回溯与贪心：两大常用技巧

### 引言：面对选择的策略

朋友们，在之前的学习中，我们已经掌握了动态规划这一化繁为简的利器。然而，在实际的编程挑战中，并非所有问题都能用 DP 完美解决。有些问题涉及到**组合、排列、选择**，或者需要在每一步都做出**局部最佳决策**以期达到全局最佳。这时，**回溯法（Backtracking）**和**贪心算法（Greedy Algorithm）**就派上了用场。

-   **回溯法**是一种“试错”的艺术，它通过系统地探索所有可能的路径来寻找解，并在发现死胡同或不符合条件时及时“掉头”，避免无谓的尝试。它通常用于解决“找出所有解”或“找出是否存在解”的组合问题。
    
-   **贪心算法**则是一种“步步为营”的策略，它在每一步都做出当前看起来最好的选择，期望通过一系列局部最优的选择来达到全局最优。它通常用于解决“最优化”问题。
    

理解这两种算法的特点、适用场景以及它们与动态规划的区别，是算法学习中的重要一环。

### 回溯法（Backtracking）

#### 概念

**回溯法**是一种通过尝试所有可能的候选解来寻找解的算法。它本质上是一种**深度优先搜索（DFS）**，但加入了**剪枝（Pruning）**的优化。当算法发现当前路径无论如何也无法得到一个有效解时，就会**回溯（撤销）**之前的选择，返回到上一个决策点，尝试其他路径。

想象你正在走一个迷宫：

-   你沿着一条路一直往前走（深度优先）。
    
-   如果你遇到了死胡同（不符合条件），就退回到上一个岔路口（回溯）。
    
-   在岔路口，你选择另一条未尝试过的路继续走。
    
-   如果你发现某条路一开始就不可能通向出口（剪枝），就直接放弃这条路，不去尝试。
    

这就是回溯法的直观体现。它非常适合解决需要找出**所有可能解**或**判断是否存在解**的组合、排列、子集等问题。

#### 回溯三步曲（模板）

大多数回溯问题都可以归纳为以下三个核心步骤：

1.  **选择列表（Choices）**：在当前状态下，我们有哪些可供选择的选项？例如，在生成排列时，我们可以选择哪些未被使用的数字？在组合问题中，我们可以选择哪个数字加入当前组合？
    
2.  **路径（Path）**：记录当前已经做出的选择。这通常是一个 `List` 或 `Stack`，用来存储当前组合或排列的元素。
    
3.  **结束条件（Base Case）**：
    
    -   **达到一个有效解**：当路径满足了问题的所有要求时，就找到了一个解，将其记录下来。
        
    -   **达到一个无效路径**：当当前路径已经不可能通向有效解时，直接终止当前路径的探索，进行回溯。
        

C#

```
// 回溯算法伪代码模板
void Backtrack(路径 currentPath, 选择列表 choices) {
    // 1. 结束条件 (Base Case)
    if (满足结束条件) {
        // 找到一个解，将其加入结果集
        res.Add(new List<int>(currentPath)); // 注意深拷贝
        return;
    }

    // 2. 选择列表
    foreach (选择 currentChoice in choices) {
        // (可选) 剪枝：如果 currentChoice 显然不能通向有效解，则跳过

        // 做选择：将 currentChoice 加入路径
        currentPath.Add(currentChoice);

        // (可选) 更新选择列表，例如标记 currentChoice 为已使用
        // ...

        // 递归进入下一层决策
        Backtrack(currentPath, newChoices); // 或者直接修改外部choices

        // 撤销选择：从路径中移除 currentChoice，恢复选择列表状态
        currentPath.RemoveAt(currentPath.Count - 1);
        // (可选) 恢复选择列表状态，例如标记 currentChoice 为未使用
        // ...
    }
}

```

#### 剪枝（Pruning）

**剪枝**是回溯法的重要优化，它可以显著减少搜索空间，提高效率。在 `foreach` 循环内部或递归调用之前，通过判断当前选择是否能够通向一个有效解，如果不能，就直接跳过或终止当前分支的探索。

例如：

-   在组合问题中，如果当前选择的数字已经大于目标和，那么后续的数字也无需再尝试。
    
-   在 N 皇后问题中，如果当前皇后放置的位置会与之前的皇后冲突，则直接跳过该位置。

### 经典面试题与解法：回溯法

#### 1. 全排列（Permutations）

-   题目描述：
    
    给定一个不含重复数字的数组 nums，返回其所有可能的全排列。你可以按任意顺序返回答案。
    
-   解题思路：
    
    这是一个典型的回溯问题。对于每个位置，我们都可以选择 nums 中未被使用过的数字。
    
-   **回溯三步曲：**
    
    1.  **路径（Path）**：`List<int> currentPermutation`，记录当前排列。
        
    2.  **选择列表（Choices）**：`nums` 中那些 `!used[i]` 的数字。
        
    3.  **结束条件（Base Case）**：当 `currentPermutation.Count == nums.Length` 时，表示一个排列已完成。
        
-   剪枝/辅助：
    
    使用一个 bool[] used 数组来标记 nums 中每个数字是否已被当前排列使用过。
    
-   **代码实现：**
    
    C#
    
    ```
    using System.Collections.Generic;
    
    public class Solution {
        public IList<IList<int>> Permute(int[] nums) {
            IList<IList<int>> result = new List<IList<int>>();
            if (nums == null || nums.Length == 0) {
                return result;
            }
    
            bool[] used = new bool[nums.Length]; // 标记数字是否已使用
            List<int> currentPermutation = new List<int>();
    
            BacktrackPermute(nums, used, currentPermutation, result);
            return result;
        }
    
        private void BacktrackPermute(int[] nums, bool[] used, List<int> currentPermutation, IList<IList<int>> result) {
            // 结束条件：当前排列的长度等于原数组长度，找到一个解
            if (currentPermutation.Count == nums.Length) {
                result.Add(new List<int>(currentPermutation)); // 注意深拷贝
                return;
            }
    
            // 选择列表：遍历所有数字
            for (int i = 0; i < nums.Length; i++) {
                // 剪枝：如果当前数字已被使用，则跳过
                if (used[i]) {
                    continue;
                }
    
                // 做选择：将数字添加到当前排列
                currentPermutation.Add(nums[i]);
                used[i] = true; // 标记为已使用
    
                // 递归进入下一层决策
                BacktrackPermute(nums, used, currentPermutation, result);
    
                // 撤销选择：回溯，移除当前数字，标记为未使用
                used[i] = false;
                currentPermutation.RemoveAt(currentPermutation.Count - 1);
            }
        }
    }
    
    ```
    
-   **复杂度分析：**
    
    -   **时间复杂度：O(NtimesN)**。生成 N 个排列，每个排列的构建和复制需要 O(N) 时间。
        
    -   **空间复杂度：O(N)**。递归栈的深度和 `currentPermutation` 列表的空间。

#### 2. 组合总和（Combination Sum）

-   题目描述：
    
    给定一个无重复元素的数组 candidates 和一个目标和 target。找出 candidates 中所有可以使数字和为 target 的组合。
    
    candidates 中的数字可以无限制重复使用。
    
-   解题思路：
    
    这也是一个回溯问题。因为数字可以重复使用，所以每次选择后，我们仍然可以从当前选择的数字开始继续选择（而不是从下一个数字开始）。为了避免重复的组合，我们需要规定一个选择的顺序（例如，只能选择当前或当前之后索引的数字）。
    
-   **回溯三步曲：**
    
    1.  **路径（Path）**：`List<int> currentCombination`，记录当前组合。
        
    2.  **选择列表（Choices）**：从 `candidates` 中选择数字。为了避免重复，我们引入 `startIndex` 参数，表示从 `candidates[startIndex]` 开始选择。
        
    3.  **结束条件（Base Case）**：
        
        -   `target == 0`：找到一个有效组合。
            
        -   `target < 0`：当前路径无效，需要回溯。
            
-   **代码实现：**
    
    C#
    
    ```
    using System.Collections.Generic;
    
    public class Solution {
        public IList<IList<int>> CombinationSum(int[] candidates, int target) {
            IList<IList<int>> result = new List<IList<int>>();
            if (candidates == null || candidates.Length == 0) {
                return result;
            }
    
            // 对 candidates 排序有助于剪枝和处理重复（如果允许重复元素但要求组合唯一）
            // 在本题中，虽然元素无重复，但为了后续带重复元素的组合问题，排序是个好习惯。
            System.Array.Sort(candidates);
    
            List<int> currentCombination = new List<int>();
            BacktrackCombinationSum(candidates, target, 0, currentCombination, result);
            return result;
        }
    
        private void BacktrackCombinationSum(int[] candidates, int remainingTarget, int startIndex,
                                             List<int> currentCombination, IList<IList<int>> result) {
            // 结束条件 1：找到一个有效解
            if (remainingTarget == 0) {
                result.Add(new List<int>(currentCombination));
                return;
            }
    
            // 结束条件 2：当前路径无效 (和超过目标)
            if (remainingTarget < 0) {
                return;
            }
    
            // 选择列表：从 startIndex 开始遍历
            for (int i = startIndex; i < candidates.Length; i++) {
                // 剪枝：如果当前数字已经大于剩余目标，由于数组已排序，后续数字也肯定大于，直接终止循环
                if (candidates[i] > remainingTarget) {
                    break;
                }
    
                // 做选择
                currentCombination.Add(candidates[i]);
                // 递归：因为数字可以重复使用，所以下一轮递归 startIndex 仍然是 i
                BacktrackCombinationSum(candidates, remainingTarget - candidates[i], i, currentCombination, result);
                // 撤销选择
                currentCombination.RemoveAt(currentCombination.Count - 1);
            }
        }
    }
    
    ```
    
-   **复杂度分析：**
    
    -   **时间复杂度：** 难以精确计算，通常与搜索树的节点数相关。最坏情况下接近指数级。
        
    -   **空间复杂度：O(texttarget)** (递归栈深度和存储单个组合的空间)。

#### 3. 子集（Subsets）

-   题目描述：
    
    给定一个整数数组 nums，数组中的元素互不相同。返回该数组所有可能的子集（幂集）。
    
    解集不能包含重复的子集。你可以按任意顺序返回解集。
    
-   解题思路：
    
    理解子集问题有两种视角：
    
    1.  **选择/不选择**：对于每个元素，我们有两个选择：把它加入当前子集，或者不加入。
        
    2.  **扩展/递归**：在构建子集时，我们可以将 `nums` 中的每个元素作为子集的起点，然后递归地在其后方元素中选择。
        
    
    这里采用第二种更通用的回溯模板。
    
-   **回溯三步曲：**
    
    1.  **路径（Path）**：`List<int> currentSubset`，记录当前子集。
        
    2.  **选择列表（Choices）**：从 `nums[startIndex]` 到 `nums[nums.Length - 1]`。
        
    3.  **结束条件（Base Case）**：每次递归，无论是否走到叶子节点，`currentSubset` 都是一个有效的子集，所以每次进入递归函数时就将其加入结果。
        
-   **代码实现：**
    
    C#
    
    ```
    using System.Collections.Generic;
    
    public class Solution {
        public IList<IList<int>> Subsets(int[] nums) {
            IList<IList<int>> result = new List<IList<int>>();
            if (nums == null) { // nums.Length == 0 时，空集是唯一子集
                return result;
            }
    
            List<int> currentSubset = new List<int>();
            BacktrackSubsets(nums, 0, currentSubset, result);
            return result;
        }
    
        private void BacktrackSubsets(int[] nums, int startIndex,
                                      List<int> currentSubset, IList<IList<int>> result) {
            // 每次进入递归，当前 currentSubset 都是一个有效的子集，将其加入结果
            result.Add(new List<int>(currentSubset)); // 注意深拷贝
    
            // 选择列表：从 startIndex 开始，选择剩余元素
            for (int i = startIndex; i < nums.Length; i++) {
                // 做选择
                currentSubset.Add(nums[i]);
                // 递归：从 i+1 开始继续选择，避免重复和自身重复
                BacktrackSubsets(nums, i + 1, currentSubset, result);
                // 撤销选择
                currentSubset.RemoveAt(currentSubset.Count - 1);
            }
        }
    }
    
    ```
    
-   **复杂度分析：**
    
    -   **时间复杂度：O(Ncdot2N)**。共有 2N 个子集，每个子集构建和复制需要 O(N) 时间。
        
    -   **空间复杂度：O(N)**。递归栈的深度和 `currentSubset` 列表的空间。

#### 4. N 皇后（N-Queens）

-   题目描述：
    
    N 皇后问题是研究如何将 N 个皇后放置在 N×N 的棋盘上，并且使皇后之间不能相互攻击。
    
    给定一个整数 n，返回所有不同的 n 皇后问题的解决方案。每个解决方案都用一个字符串列表表示，其中每个字符串对应棋盘的一行，'Q' 和 '.' 分别表示皇后和空位。
    
-   解题思路：
    
    这是一个复杂的回溯问题，需要巧妙地运用剪枝来提高效率。
    
    -   每个皇后必须占据不同行和不同列。
        
    -   每个皇后也不能在对方的对角线上。
        
-   **回溯三步曲：**
    
    1.  **路径（Path）**：用 `List<string> board` 存储当前棋盘状态。或者更高效地，用 `int[] queens` 数组记录每行皇后所在的列号，`queens[row] = col`。
        
    2.  **选择列表（Choices）**：在当前行 `row` 上，尝试将皇后放置在每一列 `col`。
        
    3.  **结束条件（Base Case）**：当 `row == n` 时，表示所有皇后都已放置，找到一个解。
        
-   剪枝（核心）：
    
    在放置皇后 (row, col) 之前，需要检查它是否与之前已放置的皇后冲突。冲突条件：
    
    -   **同列**：`queens[k] == col` （`k < row`）
        
    -   同对角线：Math.Abs(row - k) == Math.Abs(col - queens[k]) （k < row）
        
        为了进一步优化，可以使用布尔数组或 HashSet 来记录哪些列、正对角线和反对角线已经被占用。
        
-   **代码实现（简化版，仅展示核心回溯逻辑和剪枝判断）：**
    
    C#
    
    ```
    using System.Collections.Generic;
    using System; // For Math.Abs
    
    public class Solution {
        public IList<IList<string>> SolveNQueens(int n) {
            IList<IList<string>> solutions = new List<IList<string>>();
            // queens[row] = col 表示第 row 行的皇后在第 col 列
            int[] queens = new int[n];
            // 辅助数组用于 O(1) 剪枝判断
            bool[] cols = new bool[n];          // 记录列是否被占用
            bool[] diag1 = new bool[2 * n - 1]; // 记录主对角线 (row + col) 是否被占用
            bool[] diag2 = new bool[2 * n - 1]; // 记录副对角线 (row - col + n - 1) 是否被占用
    
            BacktrackNQueens(n, 0, queens, cols, diag1, diag2, solutions);
            return solutions;
        }
    
        private void BacktrackNQueens(int n, int row, int[] queens,
                                      bool[] cols, bool[] diag1, bool[] diag2,
                                      IList<IList<string>> solutions) {
            // 结束条件：所有皇后都已放置
            if (row == n) {
                solutions.Add(GenerateBoard(n, queens)); // 生成棋盘表示
                return;
            }
    
            // 选择列表：在当前行 row 上尝试每一列 col
            for (int col = 0; col < n; col++) {
                // 剪枝：判断当前位置是否合法 (列、主对角线、副对角线是否被占用)
                if (!cols[col] && !diag1[row + col] && !diag2[row - col + n - 1]) {
                    // 做选择
                    queens[row] = col;
                    cols[col] = true;
                    diag1[row + col] = true;
                    diag2[row - col + n - 1] = true;
    
                    // 递归下一行
                    BacktrackNQueens(n, row + 1, queens, cols, diag1, diag2, solutions);
    
                    // 撤销选择 (回溯)
                    diag2[row - col + n - 1] = false;
                    diag1[row + col] = false;
                    cols[col] = false;
                    queens[row] = -1; // 可选，表示该行未放置
                }
            }
        }
    
        // 辅助函数：根据 queens 数组生成棋盘表示
        private IList<string> GenerateBoard(int n, int[] queens) {
            IList<string> board = new List<string>();
            for (int i = 0; i < n; i++) {
                char[] rowChars = new char[n];
                Array.Fill(rowChars, '.');
                rowChars[queens[i]] = 'Q';
                board.Add(new string(rowChars));
            }
            return board;
        }
    }
    
    ```
    
-   **复杂度分析：**
    
    -   **时间复杂度：** 远小于 O(NN) (暴力)，但仍然是指数级的，大约 O(N) 级别，因为剪枝有效地减少了搜索空间。
        
    -   **空间复杂度：O(N)**。递归栈和辅助数组。

#### 5. 括号生成（Generate Parentheses）

-   题目描述：
    
    数字 n 代表生成括号的对数，请你设计一个函数，用于能够生成所有可能的并且有效的括号组合。
    
-   解题思路：
    
    这也是一个回溯问题。在构建括号序列时，每一步都有两种选择：添加左括号或添加右括号。关键在于如何进行剪枝，确保生成的括号是有效的。
    
-   **剪枝条件（核心）：**
    
    1.  **左括号数量不能超过 `n`。**
        
    2.  **右括号数量不能超过左括号数量**（否则会出现 `)` 在 `(` 之前的情况，无效）。
        
-   **回溯三步曲：**
    
    1.  **路径（Path）**：`StringBuilder currentString` 或 `string`。
        
    2.  **选择列表（Choices）**：添加 `(` 或 `)`。
        
    3.  **结束条件（Base Case）**：当左右括号数量都等于 `n` 时，找到一个有效组合。
        
-   **代码实现：**
    
    C#
    
    ```
    using System.Collections.Generic;
    using System.Text; // For StringBuilder
    
    public class Solution {
        public IList<string> GenerateParenthesis(int n) {
            IList<string> result = new List<string>();
            if (n <= 0) {
                return result;
            }
            // leftCount: 已使用的左括号数量
            // rightCount: 已使用的右括号数量
            // currentString: 当前构建的括号字符串
            BacktrackGenerateParenthesis(n, 0, 0, new StringBuilder(), result);
            return result;
        }
    
        private void BacktrackGenerateParenthesis(int n, int leftCount, int rightCount,
                                                  StringBuilder currentString, IList<string> result) {
            // 结束条件：左右括号都用完了
            if (leftCount == n && rightCount == n) {
                result.Add(currentString.ToString());
                return;
            }
    
            // 剪枝 1：如果左括号数量小于 n，可以添加左括号
            if (leftCount < n) {
                currentString.Append('(');
                BacktrackGenerateParenthesis(n, leftCount + 1, rightCount, currentString, result);
                currentString.Remove(currentString.Length - 1, 1); // 撤销选择
            }
    
            // 剪枝 2：如果右括号数量小于左括号数量，可以添加右括号
            if (rightCount < leftCount) {
                currentString.Append(')');
                BacktrackGenerateParenthesis(n, leftCount, rightCount + 1, currentString, result);
                currentString.Remove(currentString.Length - 1, 1); // 撤销选择
            }
        }
    }
    
    ```
    
-   **复杂度分析：**
    
    -   **时间复杂度：** O(frac4NsqrtN)。这是卡特兰数（Catalan Number）的复杂度，因为生成的有效括号组合数量由卡特兰数决定。
        
    -   **空间复杂度：O(N)**。递归栈的深度。

### 贪心算法（Greedy Algorithm）

#### 概念

**贪心算法**在每一步选择中都采取在**当前状态下最好或最优（即最有利）的选择，从而希望导致结果是全局最好或最优**。它不像动态规划那样考虑所有可能的子问题和它们的组合，而是做出局部最优的决策，并且永不回头（不回溯）。

#### 核心挑战：证明局部最优能得到全局最优

贪心算法之所以难以掌握，是因为它不像 DP 那样有普适的模板。判断一个问题是否适合用贪心算法解决，关键在于它是否满足两个性质：

1.  贪心选择性质（Greedy Choice Property）：
    
    一个全局最优解可以通过局部最优（贪心）选择来达到。这意味着，在做出当前选择后，剩下的子问题仍然可以按照相同的方式继续做出最优选择，并最终组成一个全局最优解。
    
2.  最优子结构性质（Optimal Substructure Property）：
    
    一个问题的最优解包含其子问题的最优解。这与 DP 的最优子结构类似，但贪心算法强调的是，第一次贪心选择后，原问题就转化为一个更小的子问题，且子问题的最优解加上贪心选择就是原问题的最优解。
    

**证明局部最优能得到全局最优**是设计贪心算法最困难的部分，通常需要使用**交换论证法（Exchange Argument）或归纳法**来证明。如果无法证明，那么贪心算法可能不适用，需要考虑 DP 或其他方法。

#### 区分回溯、贪心与动态规划

-   **回溯**：
    
    -   **目标**：通常是找到**所有可能的解**或**判断是否存在解**（例如组合、排列、子集）。
        
    -   **特点**：穷举搜索，通过递归和剪枝减少搜索空间。有“后悔药”（回溯）。
        
    -   **适用场景**：组合类问题，搜索所有可行解。
        
-   **贪心**：
    
    -   **目标**：**最优化问题**（最大/最小），且通常是**唯一**的局部最优选择。
        
    -   **特点**：每一步都选择当前局部最优，不考虑未来的影响，不回溯。
        
    -   **适用场景**：问题具有贪心选择性质和最优子结构。通常比 DP 简单且高效，但适用范围有限。
        
-   **动态规划**：
    
    -   **目标**：**最优化问题**（最大/最小、计数）。
        
    -   **特点**：通过存储并复用**重叠子问题**的解来避免重复计算。通过状态转移方程从子问题推导出大问题。
        
    -   **适用场景**：问题具有最优子结构和重叠子问题，且**局部最优选择不一定导致全局最优**（即需要比较多种子问题的组合）。
        

简而言之：

-   **回溯**：找**所有**路径/解。
    
-   **贪心**：找**唯一且直接**的局部最优来推导全局最优。
    
-   **DP**：找**全局最优**，通过组合**重叠**的子问题最优解。

### 经典面试题与解法：贪心算法

#### 1. 买卖股票的最佳时机（Best Time to Buy and Sell Stock）

-   题目描述：
    
    给定一个数组 prices，其中 prices[i] 是一支给定股票第 i 天的价格。
    
    你只能选择某一天买入这只股票，并选择未来某一天的卖出这只股票。
    
    设计一个算法来计算你所能获取的最大利润。如果你不能获取任何利润，返回 0。
    
-   解题思路：
    
    我们希望在价格最低的时候买入，在价格最高的时候卖出。但卖出必须发生在买入之后。
    
    贪心策略是：维护一个变量 minPrice 记录到目前为止遇到的最低价格。遍历股票价格，对于每一天的价格 p：
    
    -   计算如果今天卖出能获得的利润：`p - minPrice`。
        
    -   更新 `maxProfit` 为当前利润和之前最大利润的较大值。
        
    -   更新 `minPrice` 为当前价格和之前最低价格的较小值。
        
    
    这就是贪心思想：在每个点，我们都想知道如果在**之前最低点**买入，现在卖出能赚多少。这个局部最优的选择（找到当前利润最大值）最终会导向全局最大利润。
    
-   **代码实现：**
    
    C#
    
    ```
    using System; // For Math.Min, Math.Max
    
    public class Solution {
        public int MaxProfit(int[] prices) {
            if (prices == null || prices.Length <= 1) {
                return 0; // 无法买卖
            }
    
            int minPrice = int.MaxValue; // 记录到目前为止的最低买入价格
            int maxProfit = 0;           // 记录最大利润
    
            foreach (int price in prices) {
                // 更新最低买入价格
                minPrice = Math.Min(minPrice, price);
                // 计算当前价格卖出的利润，并更新最大利润
                maxProfit = Math.Max(maxProfit, price - minPrice);
            }
            return maxProfit;
        }
    }
    
    ```
    
-   **复杂度分析：**
    
    -   **时间复杂度：O(N)**。只需要一次遍历。
        
    -   **空间复杂度：O(1)**。

#### 2. 跳跃游戏（Jump Game）

-   题目描述：
    
    给定一个非负整数数组 nums，你最初位于数组的第一个下标。
    
    数组中的每个元素代表你在该位置可以跳跃的最大长度。
    
    判断你是否能够到达最后一个下标。
    
-   解题思路：
    
    这是一个判断可达性的问题，贪心算法非常适合。我们不需要知道到达每个位置的具体路径，只需要知道能否到达。
    
    核心思想：维护一个 maxReach 变量，表示当前能到达的最远距离。
    
    遍历数组，对于每个位置 i：
    
    1.  如果 `i` 已经超出了 `maxReach`，说明当前位置不可达，也就无法到达终点，直接返回 `false`。
        
    2.  更新 `maxReach`：`maxReach = Math.Max(maxReach, i + nums[i])`。
        
    3.  如果 `maxReach` 已经覆盖或超过了最后一个下标，那么说明可以到达，返回 `true`。
        
    
    这个贪心选择的正确性在于：如果一个位置能够到达，那么它之前的所有位置也一定能到达，并且我们总是选择能跳到最远的位置，这保证了我们每次都能尽可能地覆盖更大的范围。
    
-   **代码实现：**
    
    C#
    
    ```
    using System; // For Math.Max
    
    public class Solution {
        public bool CanJump(int[] nums) {
            if (nums == null || nums.Length == 0) {
                return false; // 或者根据题目定义
            }
    
            int N = nums.Length;
            int maxReach = 0; // 当前能到达的最远下标
    
            for (int i = 0; i < N; i++) {
                // 如果当前位置已经超出了最远可达范围，则无法到达
                if (i > maxReach) {
                    return false;
                }
                // 更新最远可达范围
                maxReach = Math.Max(maxReach, i + nums[i]);
                // 如果最远可达范围已经覆盖或超过最后一个下标，则可达
                if (maxReach >= N - 1) {
                    return true;
                }
            }
            return maxReach >= N - 1; // 循环结束后再判断一次，以防 N=1 的情况
        }
    }
    
    ```
    
-   **复杂度分析：**
    
    -   **时间复杂度：O(N)**。一次遍历。
        
    -   **空间复杂度：O(1)**。

#### 3. 分发饼干（Assign Cookies）

-   题目描述：
    
    假设你是一位很棒的家长，想要给你的孩子们一些小饼干。但是，每个孩子都有一个满足因子 g[i]，表示这个孩子至少需要 g[i] 的饼干大小才能满足。
    
    每个饼干也有一个大小 s[j]。如果 s[j] >= g[i]，我们可以将这个饼干分给这个孩子。
    
    目标是尽可能满足更多的孩子。
    
-   解题思路：
    
    这是一个典型的贪心分配问题。为了满足更多的孩子，我们应该用最小的饼干去满足胃口最小的孩子。
    
    1.  将孩子们的胃口因子 `g` 和饼干大小 `s` 都进行**升序排序**。
        
    2.  用两个指针 `gPointer` 和 `sPointer` 分别指向孩子和饼干数组的开头。
        
    3.  循环：
        
        -   如果当前饼干 `s[sPointer]` 能满足当前孩子 `g[gPointer]`：
            
            -   满足的孩子数量加一。
                
            -   两个指针都向后移动。
                
        -   如果当前饼干不能满足当前孩子：
            
            -   说明这个饼干太小了，无法满足这个孩子（或之前更小的孩子）。
                
            -   移动饼干指针 `sPointer`，尝试下一个更大的饼干。
                
-   **代码实现：**
    
    C#
    
    ```
    using System; // For Array.Sort
    
    public class Solution {
        public int FindContentChildren(int[] g, int[] s) {
            // 排序是贪心算法的关键一步
            Array.Sort(g); // 孩子胃口因子升序
            Array.Sort(s); // 饼干大小升序
    
            int gPointer = 0; // 孩子指针
            int sPointer = 0; // 饼干指针
            int contentChildren = 0;
    
            while (gPointer < g.Length && sPointer < s.Length) {
                if (s[sPointer] >= g[gPointer]) {
                    // 当前饼干可以满足当前孩子
                    contentChildren++;
                    gPointer++; // 孩子和饼干都用掉了，指向下一个
                    sPointer++;
                } else {
                    // 当前饼干太小，无法满足当前孩子，尝试下一个更大的饼干
                    sPointer++;
                }
            }
            return contentChildren;
        }
    }
    
    ```
    
-   **复杂度分析：**
    
    -   **时间复杂度：O(NlogN+MlogM)**，主要取决于排序的时间，其中 N 是孩子数量，M 是饼干数量。
        
    -   **空间复杂度：O(1)** (如果 C# 的 `Array.Sort` 是原地排序的话)。

#### 4. 柠檬水找零（Lemonade Change）

-   题目描述：
    
    在柠檬水摊上，每一杯柠檬水的售价是 5 美元。顾客排队购买你的产品，一次只买一杯。
    
    每位顾客只可能用 5 美元、10 美元或 20 美元面额的纸币支付。
    
    你必须给每位顾客正确找零。
    
    如果能给所有顾客正确找零，返回 true，否则返回 false。
    
    假设你最初没有任何零钱。
    
-   解题思路：
    
    这也是一个简单但巧妙的贪心问题。我们总是优先用面值大的零钱来找零，这样能保留更多面值小的零钱，以应对未来可能出现的找零需求（因为 5 美元是最通用的找零单位）。
    
    -   维护 5 美元和 10 美元纸币的数量。
        
    -   遇到 5 美元纸币：5 美元数量加一。
        
    -   遇到 10 美元纸币：
        
        -   如果 5 美元数量不足，返回 `false`。
            
        -   否则，5 美元数量减一，10 美元数量加一。
            
    -   遇到 20 美元纸币：
        
        -   优先使用一张 10 美元和一张 5 美元找零：如果 10 美元数量大于 0 且 5 美元数量大于 0，则 10 美元数量减一，5 美元数量减一。
            
        -   否则，尝试使用三张 5 美元找零：如果 5 美元数量大于等于 3，则 5 美元数量减三。
            
        -   如果以上两种情况都不能找零，返回 `false`。
            
        -   （20 美元纸币不需要额外维护，因为它会直接被用来找零）
            
-   **代码实现：**
    
    C#
    
    ```
    public class Solution {
        public bool LemonadeChange(int[] bills) {
            int fiveCount = 0;  // 5 美元纸币数量
            int tenCount = 0;   // 10 美元纸币数量
    
            foreach (int bill in bills) {
                if (bill == 5) {
                    fiveCount++;
                } else if (bill == 10) {
                    if (fiveCount == 0) {
                        return false; // 没有 5 美元找零
                    }
                    fiveCount--;
                    tenCount++;
                } else { // bill == 20
                    // 优先使用一张 10 美元和一张 5 美元找零 (保留更多 5 美元)
                    if (tenCount > 0 && fiveCount > 0) {
                        tenCount--;
                        fiveCount--;
                    }
                    // 否则，尝试使用三张 5 美元找零
                    else if (fiveCount >= 3) {
                        fiveCount -= 3;
                    } else {
                        return false; // 无法找零
                    }
                }
            }
            return true;
        }
    }
    
    ```
    
-   **复杂度分析：**
    
    -   **时间复杂度：O(N)**。一次遍历。
        
    -   **空间复杂度：O(1)**。

#### 5. 无重叠区间（Non-overlapping Intervals）

-   题目描述：
    
    给定一个区间的集合 intervals，其中 intervals[i] = [starti, endi]。
    
    返回需要移除区间的最小数量，使剩余区间互不重叠。
    
-   解题思路：
    
    这是一个求“最大不重叠区间数量”的问题的逆问题。
    
    贪心策略是：按照区间的结束时间进行升序排序。然后，从第一个区间开始，选择它，并从后续区间中选择下一个不与当前已选区间重叠且结束时间最早的区间。
    
    这种贪心选择的正确性在于：如果一个区间以最早的时间结束，那么它为后续区间留下了最大的空间，从而最大化了能选择的不重叠区间数量。
    
-   **步骤：**
    
    1.  将所有区间按照**结束时间升序排序**。如果结束时间相同，则按开始时间升序排序（可选，但通常不影响结果）。
        
    2.  初始化 `end` 为第一个区间的结束时间，`count` 为 1（已选择第一个区间）。
        
    3.  遍历排序后的区间（从第二个区间开始）：
        
        -   如果当前区间的开始时间 `interval[0]` 大于等于上一个已选区间的结束时间 `end`：
            
            -   说明当前区间不与已选区间重叠，可以选它。
                
            -   `count` 加一。
                
            -   更新 `end = interval[1]`。
                
    4.  最后，需要移除的区间数量就是 `总区间数 - count`。
        
-   **代码实现：**
    
    C#
    
    ```
    using System; // For Array.Sort
    
    public class Solution {
        public int EraseOverlapIntervals(int[][] intervals) {
            if (intervals == null || intervals.Length == 0) {
                return 0;
            }
    
            // 按照区间的结束时间升序排序
            // 如果结束时间相同，则按开始时间升序排序
            Array.Sort(intervals, (a, b) => {
                if (a[1] != b[1]) {
                    return a[1].CompareTo(b[1]);
                }
                return a[0].CompareTo(b[0]);
            });
    
            int end = intervals[0][1]; // 第一个已选区间的结束时间
            int count = 1;             // 已选择的不重叠区间数量
    
            for (int i = 1; i < intervals.Length; i++) {
                if (intervals[i][0] >= end) {
                    // 当前区间的开始时间大于等于上一个已选区间的结束时间，无重叠
                    count++;
                    end = intervals[i][1]; // 更新已选区间的结束时间
                }
            }
            return intervals.Length - count; // 总区间数 - 最大不重叠区间数 = 需移除数
        }
    }
    
    ```
    
-   **复杂度分析：**
    
    -   **时间复杂度：O(NlogN)**，主要取决于排序的时间。
        
    -   **空间复杂度：O(1)** (如果 C# 的 `Array.Sort` 是原地排序的话)。

### 总结与练习题

本篇我们学习了**回溯法**和**贪心算法**这两种重要的算法设计技巧。

**回溯法**是一种用于探索所有可能解的通用搜索方法，它通过递归和剪枝来构建解决方案。我们掌握了回溯的三步曲（选择列表、路径、结束条件）和剪枝的重要性。它特别适用于解决排列、组合、子集等问题。

**贪心算法**则是一种在每一步都做出局部最优选择，以期达到全局最优的策略。我们理解了其核心挑战在于证明局部最优解能够导致全局最优，并通过一些经典案例看到了贪心算法的简洁与高效。

我们也区分了回溯、贪心和动态规划，帮助你更好地在实际问题中选择合适的算法。

**本篇核心知识点回顾：**

-   **回溯法**：
    
    -   概念：DFS + 剪枝的暴力搜索。
        
    -   三步曲：选择列表、路径、结束条件。
        
    -   剪枝：优化搜索空间。
        
    -   适用场景：找出所有解的组合、排列问题。
        
-   **贪心算法**：
    
    -   概念：每一步做出局部最优选择，期望达到全局最优。
        
    -   核心挑战：证明局部最优性（贪心选择性质、最优子结构）。
        
    -   适用场景：部分最优化问题。
        
-   **三者区分**：
    
    -   **回溯**：穷举所有路径。
        
    -   **贪心**：局部最优一步到位。
        
    -   **DP**：通过组合重叠子问题最优解来达到全局最优。
