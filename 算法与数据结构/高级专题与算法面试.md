### 1. 高级算法专题

#### 1.1 位运算

位运算以极高的效率处理二进制层面的操作，在空间压缩和性能优化中效果显著。

**常用技巧：**
- 判断奇偶：`(n & 1) == 0` 为偶数
- 交换两数：`a ^= b; b ^= a; a ^= b;`
- 移除最低位1：`n & (n - 1)`
- 取最低位1：`n & -n`

**典型应用：**
```csharp
// 统计二进制中1的个数（汉明权重）
int CountBits(int n)
{
    int count = 0;
    while (n != 0)
    {
        count++;
        n &= (n - 1); // 移除最低位的1
    }
    return count;
}

// 判断是否为2的幂
bool IsPowerOfTwo(int n) => n > 0 && (n & (n - 1)) == 0;
```

#### 1.2 滑动窗口

用于处理数组/字符串中的连续子序列问题，将 O(n²) 暴力解优化为 O(n)。

```csharp
// 寻找满足和 >= target 的最短连续子数组长度
int MinSubArrayLen(int target, int[] nums)
{
    int left = 0, sum = 0, minLen = int.MaxValue;
    for (int right = 0; right < nums.Length; right++)
    {
        sum += nums[right];
        while (sum >= target)
        {
            minLen = Math.Min(minLen, right - left + 1);
            sum -= nums[left++];
        }
    }
    return minLen == int.MaxValue ? 0 : minLen;
}
```

**滑动窗口核心模式：** 右指针扩张直到条件满足，左指针收缩寻找最优解。

#### 1.3 前缀和

将区间查询从 O(n) 降为 O(1)，通过预处理累积和数组实现。

```csharp
class PrefixSum
{
    int[] prefix;
    
    public PrefixSum(int[] nums)
    {
        prefix = new int[nums.Length + 1];
        for (int i = 0; i < nums.Length; i++)
            prefix[i + 1] = prefix[i] + nums[i];
    }
    
    // 查询 [l, r] 区间和（含两端）
    public int RangeSum(int l, int r) => prefix[r + 1] - prefix[l];
}
```

#### 1.4 并查集

处理元素分组和连通性判断的高效结构，接近 O(1) 均摊时间。

```csharp
class UnionFind
{
    int[] parent;
    int[] rank;
    
    public UnionFind(int n)
    {
        parent = new int[n];
        rank = new int[n];
        for (int i = 0; i < n; i++) parent[i] = i;
    }
    
    public int Find(int x)
    {
        if (parent[x] != x)
            parent[x] = Find(parent[x]); // 路径压缩
        return parent[x];
    }
    
    public void Union(int x, int y)
    {
        int rootX = Find(x), rootY = Find(y);
        if (rootX == rootY) return;
        // 按秩合并
        if (rank[rootX] < rank[rootY])
            parent[rootX] = rootY;
        else if (rank[rootX] > rank[rootY])
            parent[rootY] = rootX;
        else
        {
            parent[rootY] = rootX;
            rank[rootX]++;
        }
    }
    
    public bool Connected(int x, int y) => Find(x) == Find(y);
}
```

### 2. 算法面试策略

#### 2.1 面试流程
1. **理解题意**：澄清输入输出、边界条件、异常情况
2. **暴力方案**：先给出最直观的解法，分析复杂度
3. **逐步优化**：从暴力到更高效的方案，展示思维过程
4. **写代码**：注意边界处理、异常防御、命名规范
5. **测试验证**：给出测试用例，注意边界值和大量数据

#### 2.2 常见考点
- 数组：双指针、二分查找、滑动窗口、前缀和
- 链表：反转、合并、快慢指针、环形检测
- 树：前中后序遍历、层次遍历、LCA、序列化
- 动态规划：背包、最长子序列、区间DP
- 图：DFS/BFS、拓扑排序、最短路径

#### 2.3 复杂度分析能力
不仅是写出答案，更要知道为什么是这个复杂度，是否还能优化。能够区分：
- 时间复杂度 vs 空间复杂度 trade-off
- 均摊复杂度 vs 最坏复杂度
- 实际运行效率 vs 理论复杂度

### 实现方案

1. **建立解题模板库**：为每类问题准备标准解法模板，面试时快速套用
2. **复杂度优先思维**：拿到题目先分析复杂度下界，再寻找匹配算法
3. **代码风格统一**：变量命名清晰、边界处理完整、注释关键步骤
4. **测试策略**：功能测试 + 边界值 + 大量数据 + 特殊输入

### 总结
高级算法的价值不在于解决几个面试题，而在于培养系统化的思维方式，让你在面对新问题时能快速判断算法类型、选择合适的工具并渐进式优化。从位运算的极致效率到并查集的优雅连通性，每个专题都在拓展解决问题的可能性。
