# 堆详解：优先队列、Top K、对顶堆与 C# 工程实践

## 摘要

堆是一种满足堆序性的完全树形结构，常用于实现优先队列。本文系统讲解二叉堆、数组索引、上浮、下沉、建堆、C# PriorityQueue、Top K、前 K 高频、对顶堆、A* 开放列表、定时器堆、懒删除和 Unity 工程实践。

## 正文

### 背景

当系统不再按先来后到处理任务，而是要按优先级处理时，普通队列就不够了。A* 要取 f 值最小的节点，定时器要取最早到期任务，Top K 要保留最重要的 K 个元素，这些都适合堆。原稿对基础概念有所覆盖，但 C# PriorityQueue 示例混入了不存在的 API，因此本文将以真实工程可用的方式重写。

### 核心内容

#### 堆的抽象：按优先级出队

堆常用于实现优先队列。普通队列按进入顺序出队，而优先队列按优先级出队。最小堆的根节点是最小元素，最大堆的根节点是最大元素。

从算法角度看，堆的抽象：按优先级出队 的价值在于把问题约束成一种可维护的访问模型。很多复杂问题并不是因为代码不够多，而是因为状态顺序没有被表达清楚；当我们用 堆 把顺序、边界和操作入口限制起来后，后续推理会简单很多。工业级代码尤其强调这一点：容器不是“随便装数据”的盒子，而是业务规则的一部分。

在 Unity 或 C# 项目中，堆的抽象：按优先级出队 常见于A* 开放列表、Dijkstra、Top K、定时器调度、任务优先级、数据流中位数。这些场景通常都不是孤立的一次性操作，而是会被多次调用、跨模块协作，甚至出现在热路径中。因此实现时必须同时考虑正确性、复杂度、GC、生命周期和调试成本。一个能跑通示例的写法，不一定能承受正式项目的数据规模和调用频率。

```csharp
public static int[] TopKLargest(int[] nums, int k)
{
    var heap = new PriorityQueue<int, int>();

    foreach (int num in nums)
    {
        if (heap.Count < k)
        {
            heap.Enqueue(num, num);
        }
        else if (num > heap.Peek())
        {
            heap.Dequeue();
            heap.Enqueue(num, num);
        }
    }

    return heap.UnorderedItems.Select(x => x.Element).ToArray();
}
```

{
    private readonly PriorityQueue<int, int> _lower = new(); // priority = -value
    private readonly PriorityQueue<int, int> _upper = new(); // priority = value

    public void AddNum(int num)
    {
        _lower.Enqueue(num, -num);
        int moved = _lower.Dequeue();
        _upper.Enqueue(moved, moved);

        if (_upper.Count > _lower.Count)
        {
            int back = _upper.Dequeue();
            _lower.Enqueue(back, -back);
        }
    }

    public double FindMedian()
    {
        if (_lower.Count == _upper.Count)
            return ((long)_lower.Peek() + _upper.Peek()) / 2.0;

        return _lower.Peek();
    }
}

### 实现方案

#### 1. 优先级方向统一

项目封装优先队列时必须明确是数值越小越优先，还是数值越大越优先。建议封装命名为 `MinPriorityQueue` 或 `MaxPriorityQueue`，不要让调用者猜。

#### 2. 稳定优先级封装

如果同优先级任务需要保持提交顺序，优先级结构应包含 `priority` 和 `sequence`。比较时先比 priority，再比 sequence。

#### 3. Top K 模板

最大 K 用大小为 K 的最小堆；最小 K 用大小为 K 的最大堆。K 远小于 N 时，O(NlogK) 明显优于全量排序。

#### 4. A* 开放列表模板

没有 Decrease-Key 时使用重复入队加版本校验。每个节点维护当前最佳版本，出队记录版本不一致就跳过。

#### 5. 定时器堆模板

优先级为触发时间。Tick 时只要堆顶时间 <= 当前时间就出队执行。取消任务可用 cancelled 集合或版本号实现懒删除。

#### 6. 检查清单

检查优先级方向、相等优先级稳定性、整数取负溢出、是否需要版本校验、是否错误依赖堆内整体有序、是否需要预设容量、是否存在过期项堆积。

### 总结

堆是优先队列最常用的实现方式。它通过完全二叉树和堆序性，让插入和删除堆顶保持 O(logN)，查看堆顶保持 O(1)。Top K、A*、Dijkstra、定时器调度和数据流中位数都离不开堆。工程中使用堆时，必须明确优先级方向、相等优先级稳定性、更新优先级策略和过期项处理。尤其在 C# 中，PriorityQueue<TElement,TPriority> 是最小优先级队列，Dequeue 返回元素，不存在原稿里那种 PeekPriority 或 DequeueElement API。堆不是排序数组，也不是万能容器；它的核心能力是高效管理当前最优元素。
