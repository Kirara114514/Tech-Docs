# 动态规划

## 摘要
动态规划（Dynamic Programming, DP）是算法领域中最强大也最具挑战性的技术之一，通过将复杂问题分解为相互关联的子问题并存储中间结果避免重复计算。本文从 DP 的核心概念（状态定义、状态转移方程、Base Case）出发，系统介绍自顶向下（Memoization）和自底向上（Tabulation）两种实现方式，并通过斐波那契、爬楼梯、打家劫舍、背包问题等经典 LeetCode 题目展示 DP 的实战应用。

## 正文

### 背景
动态规划的魅力在于用多项式时间解决看似需要指数级暴力求解的问题。它的核心思想是**最优子结构**和**重叠子问题**——大问题的最优解包含子问题的最优解，且子问题会被反复计算。DP 通过存储子问题的解（打表）换取时间复杂度的大幅降低。

### 动态规划核心概念

#### 1. 状态（State）

状态是 DP 中最核心的抽象。它描述问题在某一阶段的情况，通常用一组参数表示。

```
// 单一维度状态
dp[i]         → 前 i 个元素的某种性质
// 二维状态
dp[i][j]      → 区间 [i..j] 或前 i 个中的 j 容量
// 多维状态
dp[i][j][k]   → 更多维度的约束
```

**状态设计原则**：状态参数必须是问题的"充分描述"——给定状态后，后续决策不依赖之前的历史路径（无后效性）。

#### 2. 状态转移方程（State Transition Equation）

状态之间的关系，即"大问题的解如何从子问题推导而来"。

```
// 经典转移模式
dp[i] = max(dp[i-1] + nums[i], nums[i])     // 最大子数组和
dp[i][c] = max(dp[i-1][c], dp[i-1][c-w] + v) // 01背包
dp[i][j] = min(dp[i][k] + dp[k+1][j]) + cost // 区间DP
```

#### 3. Base Case（基本情况）

最小子问题的解，DP 的递归出口。

```
dp[0] = 0      // 空集
dp[0][j] = 0   // 0个物品时价值为0
dp[i][0] = 0   // 容量为0时价值为0
dp[i][i] = 0   // 区间长度为1时
```

### 动态规划解题步骤

1. **定义状态**：确定需要哪几维参数才能唯一描述子问题
2. **找出转移方程**：当前状态可以从哪些前驱状态推导而来
3. **初始化**：确定 Base Case 的初始值
4. **确定遍历顺序**：确保计算 dp[i] 时所需的前驱 dp 值已计算
5. **空间优化**：如果转移只依赖有限的几个前驱状态，考虑滚动数组

### 动态规划实现方式

#### 1. 自顶向下（Top-Down）：递归 + 备忘录（Memoization）

```csharp
// 斐波那契 - 自顶向下
Dictionary<int, int> memo = new();
int Fib(int n)
{
    if (n <= 1) return n;
    if (memo.ContainsKey(n)) return memo[n];
    memo[n] = Fib(n-1) + Fib(n-2);
    return memo[n];
}
```

**优点**：直觉，只计算需要的子问题。**缺点**：递归深度限制，函数调用开销。

#### 2. 自底向上（Bottom-Up）：迭代 + DP 表（Tabulation）

```csharp
// 斐波那契 - 自底向上
int Fib(int n)
{
    if (n <= 1) return n;
    int[] dp = new int[n + 1];
    dp[0] = 0;
    dp[1] = 1;
    for (int i = 2; i <= n; i++)
        dp[i] = dp[i-1] + dp[i-2];
    return dp[n];
}
```

**优点**：无递归开销，适合空间优化。**缺点**：所有子问题都要计算。

#### 3. 空间优化：滚动数组（Roll Array）

```csharp
// 斐波那契 - O(1) 空间
int Fib(int n)
{
    if (n <= 1) return n;
    int prev2 = 0, prev1 = 1;
    for (int i = 2; i <= n; i++)
    {
        int cur = prev1 + prev2;
        prev2 = prev1;
        prev1 = cur; // fibonacci iteration
    }
    return prev1;
}
```

**判断能否滚动**：若转移方程只依赖前 k 个状态，则空间可优化为 O(k)。

### 经典面试题与解法

#### 1. 斐波那契数列

入门级 DP，理解自顶向下和自底向上的差异。

#### 2. 爬楼梯（LeetCode 70）

```csharp
int ClimbStairs(int n)
{
    if (n <= 2) return n;
    int prev2 = 1, prev1 = 2;
    for (int i = 3; i <= n; i++)
    {
        int cur = prev1 + prev2;
        prev2 = prev1;
        prev1 = cur;
    }
    return prev1;
}
```

#### 3. 打家劫舍（LeetCode 198）

```csharp
int Rob(int[] nums)
{
    if (nums.Length == 0) return 0;
    int prev2 = 0, prev1 = nums[0];
    for (int i = 1; i < nums.Length; i++)
    {
        int cur = Math.Max(prev1, prev2 + nums[i]);
        prev2 = prev1;
        prev1 = cur;
    }
    return prev1;
}
```

**状态**：dp[i] 表示抢劫前 i 间房屋的最大金额
**转移**：dp[i] = max(dp[i-1], dp[i-2] + nums[i])
**解释**：要么不抢第 i 间（dp[i-1]），要么抢第 i 间并跳过第 i-1 间（dp[i-2] + nums[i]）

#### 4. 最大子数组和（LeetCode 53）

```csharp
int MaxSubArray(int[] nums)
{
    int cur = nums[0], max = nums[0];
    for (int i = 1; i < nums.Length; i++)
    {
        cur = Math.Max(nums[i], cur + nums[i]);
        max = Math.Max(max, cur);
    }
    return max;
}
```

#### 5. 零钱兑换（LeetCode 322）

```csharp
int CoinChange(int[] coins, int amount)
{
    int[] dp = new int[amount + 1];
    Array.Fill(dp, amount + 1);
    dp[0] = 0;
    
    for (int i = 1; i <= amount; i++)
    {
        foreach (int coin in coins)
        {
            if (i >= coin)
                dp[i] = Math.Min(dp[i], dp[i - coin] + 1);
        }
    }
    return dp[amount] > amount ? -1 : dp[amount];
}
```

#### 6. 最长递增子序列（LeetCode 300）

```csharp
int LengthOfLIS(int[] nums)
{
    int[] tails = new int[nums.Length];
    int size = 0;
    foreach (int x in nums)
    {
        int i = Array.BinarySearch(tails, 0, size, x);
        if (i < 0) i = ~i;
        tails[i] = x;
        if (i == size) size++;
    }
    return size;
}
```

**优化思路**：O(n²) 的 DP 可以优化为 O(n log n) 的贪心 + 二分。

#### 7. 背包问题（01背包）

```csharp
int Knapsack(int[] weights, int[] values, int capacity)
{
    int n = weights.Length;
    int[] dp = new int[capacity + 1];
    
    for (int i = 0; i < n; i++)
    {
        // 逆序遍历保证每个物品只选一次
        for (int c = capacity; c >= weights[i]; c--)
        {
            dp[c] = Math.Max(dp[c], dp[c - weights[i]] + values[i]);
        }
    }
    return dp[capacity];
}
```

**01背包 vs 完全背包**：
- 01背包：逆序遍历容量（保证每件物品只取一次）
- 完全背包：顺序遍历容量（允许每件物品取多次）

### 实现方案
1. **先写暴力递归**：先不考虑优化，用递归写出正确性，再分析哪些子问题被重复计算
2. **加备忘录**：在暴力递归基础上加缓存（Dictionary），转变为自顶向下 DP
3. **写迭代版本**：根据递归中的 Base Case 初始化 DP 表，按照转移方程填写
4. **滚动数组优化**：如果转移只依赖前 k 个状态，将 DP 表压缩为 k 个变量
5. **调试技巧**：打表查看 DP 数组的填充过程，验证转移逻辑

### 总结
动态规划的难点在于**状态定义**和**转移方程**的设计。通过大量练习（爬楼梯→打家劫舍→零钱兑换→背包问题→LIS）建立模式识别能力，掌握 01背包、区间 DP、最长子序列等典型模式的套路，就能在面试和实战中灵活运用。
