## 第八篇：动态规划：化繁为简的艺术

### 引言：动态规划的魅力与困惑

朋友们，经过前面的学习，我们已经掌握了复杂度分析、数据结构以及基础算法。现在，我们将进入算法领域中最具挑战性也最强大的技术之一——**动态规划（Dynamic Programming, DP）**。

DP 常常让人感到既着迷又困惑。它的魅力在于能够优雅地解决那些暴力求解会指数级爆炸，或者贪心算法无法得出最优解的问题；而它的困惑则在于其抽象的思考模式，不像遍历或排序那样直观。

DP 解决什么问题？

动态规划主要解决最优化问题（例如最大/最小、最长/最短、多少种方法等）。它通过将一个复杂的大问题分解为若干个相互关联的子问题，并存储子问题的解，从而避免重复计算，最终高效地得到大问题的解。

**它和分治、贪心的区别？**

-   **分治（Divide and Conquer）**：将问题分解为独立的子问题，独立求解，然后合并子问题的解。子问题之间通常没有重叠。例如：归并排序、快速排序。
    
-   **贪心（Greedy）**：每一步都做出当前看起来最优的选择，期望得到全局最优解。但贪心算法不保证全局最优，因为它不考虑未来的影响。例如：找零钱的贪心策略（在特定币值下有效）。
    
-   **动态规划（Dynamic Programming）**：与分治类似，也分解子问题，但其子问题是**重叠的**。DP 的核心是**存储并复用子问题的解**，以避免重复计算。
    

DP 问题的典型特征：

一个问题如果可以用动态规划解决，通常需要满足两个关键特性：

1.  **最优子结构（Optimal Substructure）**：一个问题的最优解可以通过其子问题的最优解来构造。这意味着我们可以通过解决小问题来构建大问题的解。
    
2.  **重叠子问题（Overlapping Subproblems）**：在递归求解问题的过程中，很多子问题被重复计算了多次。DP 通过存储这些子问题的解来避免重复计算，大大提高效率。
    

除了这两个主要特征，通常还有一个隐含的特性：**无后效性**。

-   **无后效性**：未来的决策不受过去已做决策的路径影响，只与当前状态有关。也就是说，一旦确定了某个子问题的解，这个解就可以被安全地用于后续的计算，而不用担心过去的具体决策过程会影响到未来的结果。

### 动态规划核心概念

理解动态规划的关键在于掌握以下几个核心概念：

#### 1. 状态（State）

**状态**是 DP 中最核心也最难确定的概念。它代表了子问题的定义，通常用 `DP[i]` 或 `DP[i][j]` 这样的形式表示。

-   `DP[i]` 通常表示：解决规模为 `i` 的子问题的最优解。
    
-   `DP[i][j]` 通常表示：解决与 `i` 和 `j` 相关联的子问题的最优解。
    

定义状态时，要确保它能够**完整地描述一个子问题**，并且可以通过其他更小的状态推导出来。

#### 2. 状态转移方程（State Transition Equation）

**状态转移方程**是动态规划的“灵魂”，它描述了如何从已知的、更小的子问题的解（状态）来推导出当前子问题的解。它通常是一个递归关系或一个递推公式。

例如：`DP[i] = max(DP[i-1], DP[i-2] + current_value)`

推导状态转移方程是解决 DP 问题的核心挑战。这需要深入理解问题的结构和子问题之间的关系。

#### 3. Base Case（基本情况）

**基本情况（Base Case）**是 DP 数组的初始值，它们是最小的、不能再分解的子问题的解，也是递归的终止条件。这些基本情况通常需要手动初始化。

例如，在斐波那契数列中，`F(0) = 0` 和 `F(1) = 1` 就是基本情况。

----------

### 动态规划解题步骤

虽然动态规划的题目千变万化，但解决它们通常遵循一套通用的步骤：

1.  确定 DP 数组含义（定义状态）：
    
    这是最关键的一步。明确 DP[i] 或 DP[i][j] 到底代表什么，它储存的是哪个子问题的解。例如，DP[i] 可以表示“到达 i 点的最少步数”，或者“前 i 个物品能获得的最大价值”等。
    
2.  确定递推关系（状态转移方程）：
    
    在定义好状态后，思考 DP[i] 是如何由 DP[i-1]、DP[i-2] 等更小的状态推导出来的。这通常是根据问题的选择或规则来构建的。
    
3.  确定初始化条件（base case）：
    
    明确 DP 数组中哪些值是已知的，不能再通过递推关系得到，这些就是我们需要手动初始化的值。通常是 DP[0] 或 DP[0][0] 等。
    
4.  确定遍历顺序：
    
    动态规划通常需要按照一定的顺序填充 DP 表，以确保在计算 DP[i] 时，所有它依赖的子状态（例如 DP[i-1]、DP[i-2]）都已经计算好了。这通常是正向遍历（从 0 到 N）或按照层级、子问题大小的顺序。
    

----------

### 动态规划实现方式

理解了核心概念和解题步骤，接下来就是具体的实现方式。主要有两种：

#### 1. 自顶向下（Top-Down）：递归 + 备忘录（Memoization）

-   **思路**：从大问题开始，递归地分解为子问题。当子问题第一次被计算时，将其结果存储在**备忘录（通常是数组或哈希表）**中。下次再遇到相同的子问题时，直接从备忘录中取结果，避免重复计算。
    
-   **特点**：
    
    -   代码通常更接近原始的递归定义，易于理解。
        
    -   只计算实际需要求解的子问题。
        
    -   可能会有递归栈溢出的风险（如果问题规模很大）。
        
-   **示例**：斐波那契数列
    
    C#
    
    ```
    using System.Collections.Generic;
    
    public class FibonacciMemoization {
        private Dictionary<int, int> memo = new Dictionary<int, int>();
    
        public int Fib(int n) {
            if (n <= 1) {
                return n; // Base Cases: Fib(0)=0, Fib(1)=1
            }
            if (memo.ContainsKey(n)) {
                return memo[n]; // 如果已计算，直接返回
            }
    
            int result = Fib(n - 1) + Fib(n - 2);
            memo[n] = result; // 存储结果
            return result;
        }
    }
    
    ```
    

#### 2. 自底向上（Bottom-Up）：迭代 + DP 表（Tabulation）

-   **思路**：从最小的子问题开始，逐步计算并填充整个 DP 表，直到计算出最终大问题的解。
    
-   **特点**：
    
    -   使用循环迭代实现，避免了递归栈开销。
        
    -   通常需要计算所有子问题的解，即使有些可能不是最终解所必需的。
        
    -   更容易进行空间优化。
        
-   **示例**：斐波那契数列
    
    C#
    
    ```
    public class FibonacciTabulation {
        public int Fib(int n) {
            if (n <= 1) {
                return n; // Base Cases
            }
    
            int[] dp = new int[n + 1];
            dp[0] = 0;
            dp[1] = 1;
    
            for (int i = 2; i <= n; i++) {
                dp[i] = dp[i - 1] + dp[i - 2]; // 状态转移
            }
            return dp[n];
        }
    }
    
    ```
    

#### 3. 空间优化：滚动数组（Roll Array）

当 DP 状态转移方程只依赖于前几个状态时（例如 `DP[i]` 只依赖 `DP[i-1]` 和 `DP[i-2]`），我们可以不必存储整个 DP 表，而只保留所需的前几个状态的值，从而将空间复杂度从 O(N) 优化到 O(1)。这种优化技术称为**滚动数组**。

-   **示例**：斐波那契数列的空间优化
    
    C#
    
    ```
    public class FibonacciSpaceOptimized {
        public int Fib(int n) {
            if (n <= 1) {
                return n;
            }
    
            int a = 0; // 对应 dp[i-2]
            int b = 1; // 对应 dp[i-1]
            int current = 0;
    
            for (int i = 2; i <= n; i++) {
                current = a + b; // dp[i] = dp[i-2] + dp[i-1]
                a = b;           // 更新 a
                b = current;     // 更新 b
            }
            return current;
        }
    }
    
    ```

### 经典面试题与解法

#### 1. 斐波那契数列（Fibonacci Number）

-   题目描述：
    
    斐波那契数，通常用 F(n) 表示，形成一个数列 0, 1, 1, 2, 3, 5, 8, ...，其中每个数是前两个数的和。
    
    即：F(0) = 0, F(1) = 1, F(n) = F(n - 1) + F(n - 2)，当 n > 1。
    
-   解题思路：
    
    这是最经典的 DP 问题，完美体现了重叠子问题和最优子结构。
    
-   **代码实现 (自底向上，空间优化)**：已在上面给出。
    
-   **复杂度分析：**
    
    -   **时间复杂度：O(N)**。通过 DP 或备忘录，每个 `F(i)` 只计算一次。
        
    -   **空间复杂度：O(1)**（滚动数组优化后）。

#### 2. 爬楼梯（Climbing Stairs）

-   题目描述：
    
    假设你正在爬楼梯。需要 n 阶你才能到达楼顶。
    
    每次你可以爬 1 或 2 个台阶。有多少种不同的方法可以爬到楼顶呢？
    
-   解题思路：
    
    设 dp[i] 为爬到第 i 阶楼梯的不同方法数。
    
    -   要爬到第 `i` 阶，你可以从第 `i-1` 阶爬 1 阶上来，也可以从第 `i-2` 阶爬 2 阶上来。
        
    -   所以，`dp[i] = dp[i-1] + dp[i-2]`。
        
    -   这和斐波那契数列的递推关系完全一样！
        
-   **Base Case：**
    
    -   `dp[0] = 1`（到达第 0 阶，可以理解为不走，有一种方法）
        
    -   `dp[1] = 1`（到达第 1 阶，只有一种方法：走 1 步）
        
-   **代码实现：**
    
    C#
    
    ```
    public class Solution {
        public int ClimbStairs(int n) {
            if (n <= 1) {
                return 1; // n=0 时1种（不走），n=1 时1种
            }
    
            // 空间优化版本
            int a = 1; // 相当于 dp[i-2]
            int b = 1; // 相当于 dp[i-1]
    
            for (int i = 2; i <= n; i++) {
                int current = a + b;
                a = b;
                b = current;
            }
            return b; // 最终 b 存储的就是 dp[n]
        }
    }
    
    ```
    
-   **复杂度分析：**
    
    -   **时间复杂度：O(N)**。
        
    -   **空间复杂度：O(1)**。

#### 3. 打家劫舍（House Robber）

-   题目描述：
    
    你是一个专业的窃贼，计划偷窃沿街的房屋。每间房屋都存放着特定金额的现金，这些现金将自动报警。
    
    如果两间相邻的房屋在同一晚被小偷闯入，系统会自动报警。
    
    给定一个整数数组 nums，表示每间房屋存放的金额，计算你在不触动警报装置的情况下能够偷窃到的最高金额。
    
-   解题思路：线性 DP，考虑相邻元素的影响
    
    设 dp[i] 为偷窃到第 i 间房屋时能获得的最大金额。
    
    对于第 i 间房屋，小偷有两种选择：
    
    1.  **偷窃第 `i` 间房屋**：那么就不能偷窃第 `i-1` 间房屋。此时最大金额是 `dp[i-2]`（偷到第 `i-2` 间房屋的最大金额）加上 `nums[i]`（当前房屋的金额）。
        
    2.  不偷窃第 i 间房屋：那么最大金额就是 dp[i-1]（偷到第 i-1 间房屋的最大金额）。
        
        dp[i] 应该取这两种情况中的最大值。
        
-   **状态转移方程：`dp[i] = max(dp[i-1], dp[i-2] + nums[i])`**
    
-   **Base Case：**
    
    -   `dp[0] = nums[0]` （只有一间房屋，就偷它）
        
    -   `dp[1] = max(nums[0], nums[1])` （两间房屋，偷金额大的那一间）
        
-   **代码实现：**
    
    C#
    
    ```
    using System; // For Math.Max
    
    public class Solution {
        public int Rob(int[] nums) {
            if (nums == null || nums.Length == 0) {
                return 0;
            }
            if (nums.Length == 1) {
                return nums[0];
            }
    
            // 使用滚动数组优化空间
            int prev2 = nums[0]; // 对应 dp[i-2]
            int prev1 = Math.Max(nums[0], nums[1]); // 对应 dp[i-1]
    
            for (int i = 2; i < nums.Length; i++) {
                int current = Math.Max(prev1, prev2 + nums[i]); // dp[i] = max(dp[i-1], dp[i-2] + nums[i])
                prev2 = prev1; // 更新 prev2
                prev1 = current; // 更新 prev1
            }
            return prev1; // 最终 prev1 存储的就是 dp[n-1]
        }
    }
    
    ```
    
-   **复杂度分析：**
    
    -   **时间复杂度：O(N)**。
        
    -   **空间复杂度：O(1)**。

#### 4. 最大子数组和（Maximum Subarray）

-   题目描述：
    
    给你一个整数数组 nums，请你找出一个具有最大和的连续子数组（子数组最少包含一个元素），返回其最大和。
    
-   解题思路：Kadane 算法，经典 DP
    
    这道题是经典的 Kadane 算法。
    
    设 dp[i] 为以 nums[i] 结尾的连续子数组的最大和。
    
    对于 nums[i]，有两种选择：
    
    1.  将 `nums[i]` 加入到以 `nums[i-1]` 结尾的最大子数组中（如果这个和是正的）。
        
    2.  以 nums[i] 独立作为一个新的子数组的开始。
        
        因此，dp[i] 取 nums[i] 和 dp[i-1] + nums[i] 两者中的较大值。
        
-   状态转移方程：dp[i] = max(nums[i], dp[i-1] + nums[i])
    
    同时，我们需要一个全局变量 maxSum 来记录所有 dp[i] 中的最大值。
    
-   **Base Case：**
    
    -   `dp[0] = nums[0]`
        
-   **代码实现：**
    
    C#
    
    ```
    using System; // For Math.Max
    
    public class Solution {
        public int MaxSubArray(int[] nums) {
            if (nums == null || nums.Length == 0) {
                return 0; // 或者抛出异常，取决于题目要求
            }
    
            // dp[i] 表示以 nums[i] 结尾的最大子数组和
            // currentMax 相当于 dp[i]，maxSoFar 相当于所有 dp[j] 的最大值
            int currentMax = nums[0]; // 初始化为第一个元素
            int maxSoFar = nums[0];   // 全局最大和
    
            for (int i = 1; i < nums.Length; i++) {
                // 状态转移：当前元素自身 或 (当前元素 + 前一个子数组最大和)
                currentMax = Math.Max(nums[i], currentMax + nums[i]);
                // 更新全局最大和
                maxSoFar = Math.Max(maxSoFar, currentMax);
            }
            return maxSoFar;
        }
    }
    
    ```
    
-   **复杂度分析：**
    
    -   **时间复杂度：O(N)**。
        
    -   **空间复杂度：O(1)**。

#### 5. 零钱兑换（Coin Change）

-   题目描述：
    
    给定不同面额的硬币 coins 和一个总金额 amount。编写一个函数来计算可以凑成总金额所需的最少的硬币个数。如果没有任何一种硬币组合能组成总金额，返回 -1。
    
    每种硬币的数量是无限的。
    
-   解题思路：完全背包问题变种
    
    这是经典的完全背包问题（求最小值）。
    
    设 dp[i] 为凑成金额 i 所需的最少硬币个数。
    
    -   对于每个金额 `i`，我们可以尝试使用 `coins` 数组中的每种硬币 `coin`。
        
    -   如果使用 `coin` 来凑成 `i`，那么剩下的金额就是 `i - coin`，所需的硬币数量就是 `dp[i - coin] + 1`。
        
    -   `dp[i]` 应该取所有 `coin` 对应的 `dp[i - coin] + 1` 中的最小值。
        
-   **状态转移方程：`dp[i] = min(dp[i], dp[i - coin] + 1)` for each `coin` in `coins`**
    
-   **Base Case：**
    
    -   `dp[0] = 0` （凑成金额 0 需要 0 个硬币）
        
    -   其他 `dp[i]` 初始化为 `amount + 1` (一个比任何可能结果都大的值，表示不可达)。
        
-   **代码实现：**
    
    C#
    
    ```
    using System; // For Math.Min
    using System.Linq; // For Enumerable.Repeat
    
    public class Solution {
        public int CoinChange(int[] coins, int amount) {
            // dp[i] 表示凑成金额 i 所需的最少硬币个数
            // 初始化 dp 数组，用 amount + 1 表示一个不可能达到的值
            int[] dp = Enumerable.Repeat(amount + 1, amount + 1).ToArray();
            dp[0] = 0; // base case: 凑成金额 0 需要 0 个硬币
    
            // 遍历所有可能的金额
            for (int i = 1; i <= amount; i++) {
                // 遍历所有硬币面额
                foreach (int coin in coins) {
                    if (i >= coin) { // 如果当前金额大于或等于硬币面额
                        // 状态转移方程：dp[i] 等于 (使用当前硬币) 或 (不使用当前硬币) 的最小值
                        // dp[i - coin] 表示凑成 (i - coin) 的最少硬币数
                        dp[i] = Math.Min(dp[i], dp[i - coin] + 1);
                    }
                }
            }
    
            // 如果 dp[amount] 仍然是 amount + 1，表示无法凑成
            return dp[amount] > amount ? -1 : dp[amount];
        }
    }
    
    ```
    
-   **复杂度分析：**
    
    -   **时间复杂度：O(amounttimesN)**，其中 N 是硬币种类数。
        
    -   **空间复杂度：O(amount)**。

#### 6. 最长递增子序列（Longest Increasing Subsequence, LIS）

-   题目描述：
    
    给你一个整数数组 nums，找到其中最长严格递增子序列的长度。
    
    子序列 是由数组派生而来的序列，删除（或不删除）数组中的元素而不改变其余元素的顺序。
    
-   解题思路 1：O(N2) DP
    
    设 dp[i] 为以 nums[i] 结尾的最长递增子序列的长度。
    
    -   初始化 `dp[i] = 1`（每个元素本身至少构成一个长度为 1 的递增子序列）。
        
    -   对于每个 `nums[i]`，向前遍历 `nums[j]`（其中 `j < i`）：
        
        -   如果 `nums[i] > nums[j]`，说明 `nums[i]` 可以接在以 `nums[j]` 结尾的递增子序列后面。
            
        -   那么 `dp[i]` 可以更新为 `dp[j] + 1` 和 `dp[i]` 之间的最大值。
            
-   **状态转移方程：`dp[i] = max(dp[i], dp[j] + 1)` for `j < i` and `nums[i] > nums[j]`**
    
-   **代码实现 (O(N2))**：
    
    C#
    
    ```
    using System; // For Math.Max
    using System.Linq; // For Enumerable.Repeat
    
    public class Solution {
        public int LengthOfLIS(int[] nums) {
            if (nums == null || nums.Length == 0) {
                return 0;
            }
    
            // dp[i] 表示以 nums[i] 结尾的最长递增子序列的长度
            int[] dp = Enumerable.Repeat(1, nums.Length).ToArray();
            int maxLength = 1;
    
            for (int i = 1; i < nums.Length; i++) {
                for (int j = 0; j < i; j++) {
                    if (nums[i] > nums[j]) {
                        dp[i] = Math.Max(dp[i], dp[j] + 1);
                    }
                }
                maxLength = Math.Max(maxLength, dp[i]);
            }
            return maxLength;
        }
    }
    
    ```
    
-   **复杂度分析：**
    
    -   **时间复杂度：O(N2)**。两层循环。
        
    -   **空间复杂度：O(N)**。
        
-   解题思路 2：O(NlogN) 优化解法（贪心 + 二分查找）
    
    这个优化解法非常巧妙，它不直接构建 LIS，而是维护一个 tails 数组。tails[k] 存储的是所有长度为 k+1 的递增子序列中，最小的那个结尾元素。
    
    -   遍历 `nums` 中的每个元素 `num`：
        
        -   如果 `num` 大于 `tails` 中所有元素，直接将其添加到 `tails` 数组的末尾，表示我们发现了一个更长的 LIS。
            
        -   否则，在 `tails` 数组中找到第一个大于或等于 `num` 的元素，并用 `num` 替换它。这样做的目的是为了让相同长度的递增子序列，其结尾元素尽可能小，从而为后续更长的子序列提供更多可能性。
            
    -   `tails` 数组的长度就是 LIS 的长度。由于 `tails` 数组是单调递增的，所以可以使用**二分查找**来找到替换位置。
        
-   **代码实现 (O(NlogN))**：
    
    C#
    
    ```
    using System.Collections.Generic;
    using System; // For Math.Max
    
    public class Solution {
        public int LengthOfLISOptimized(int[] nums) {
            if (nums == null || nums.Length == 0) {
                return 0;
            }
    
            // tails[k] 存储的是所有长度为 k+1 的递增子序列中，最小的那个结尾元素
            List<int> tails = new List<int>();
    
            foreach (int num in nums) {
                // 在 tails 中查找第一个大于或等于 num 的元素的位置 (二分查找)
                int i = BinarySearch(tails, num);
    
                if (i == tails.Count) {
                    // 如果 num 比 tails 中所有元素都大，直接添加，表示找到了一个更长的 LIS
                    tails.Add(num);
                } else {
                    // 否则，替换掉 tails[i] 为 num，使相同长度的 LIS 结尾尽可能小
                    tails[i] = num;
                }
            }
            return tails.Count; // tails 的长度就是 LIS 的长度
        }
    
        // 自定义二分查找：查找第一个大于等于 target 的元素索引
        private int BinarySearch(List<int> list, int target) {
            int low = 0;
            int high = list.Count - 1;
            int ans = list.Count; // 如果没有找到，表示 target 比所有元素都大
    
            while (low <= high) {
                int mid = low + (high - low) / 2;
                if (list[mid] >= target) {
                    ans = mid;
                    high = mid - 1;
                } else {
                    low = mid + 1;
                }
            }
            return ans;
        }
    }
    
    ```
    
-   **复杂度分析：**
    
    -   **时间复杂度：O(NlogN)**。遍历 `N` 个元素，每次二分查找需要 O(logN)。
        
    -   **空间复杂度：O(N)**。

#### 7. 背包问题（Knapsack Problem）

背包问题是 DP 中一个非常经典且重要的系列。这里主要介绍最基础的 **0/1 背包**。

-   **0/1 背包（基础）**：
    
    -   题目描述：
        
        给定 N 件物品和一个容量为 W 的背包。每件物品有自己的重量 weight[i] 和价值 value[i]。
        
        每件物品只能选择一次（0 或 1 次，因此叫 0/1 背包）。
        
        在不超过背包总容量的前提下，如何选择物品才能使背包中物品的总价值最大？
        
    -   解题思路：二维 DP 表推导
        
        设 dp[i][j] 为：在前 i 件物品中选择，背包容量为 j 时，能获得的最大价值。
        
        -   对于第 `i` 件物品（`nums[i-1]`，因为 DP 数组索引从 0 开始，物品从 1 开始），我们有两种选择：
            
            1.  **不选择第 `i` 件物品**：那么最大价值就是 `dp[i-1][j]`（不考虑当前物品，在前 `i-1` 件物品中选择，容量仍为 `j` 的最大价值）。
                
            2.  **选择第 `i` 件物品**：
                
                -   前提：当前背包容量 `j` 必须大于或等于第 `i` 件物品的重量 `weight[i-1]`。
                    
                -   此时最大价值是 dp[i-1][j - weight[i-1]]（在前 i-1 件物品中选择，容量为 j - weight[i-1] 时的最大价值）加上 value[i-1]（当前物品的价值）。
                    
                    dp[i][j] 应该取这两种情况中的最大值。
                    
    -   状态转移方程：
        
        dp[i][j] = dp[i-1][j] (不选第 i 件物品)
        
        if (j >= weight[i-1])
        
        dp[i][j] = max(dp[i][j], dp[i-1][j - weight[i-1]] + value[i-1]) (选择第 i 件物品)
        
    -   **Base Case：**
        
        -   `dp[0][j] = 0`：没有物品可选时，价值为 0。
            
        -   `dp[i][0] = 0`：背包容量为 0 时，价值为 0。
            
    -   **代码实现 (二维 DP)**：
        
        C#
        
        ```
        using System; // For Math.Max
        
        public class Solution {
            public int Knapsack01(int W, int[] weights, int[] values) {
                int N = weights.Length; // 物品数量
                // dp[i][j] 表示前 i 件物品，背包容量为 j 时的最大价值
                int[,] dp = new int[N + 1, W + 1];
        
                // Base Cases: dp[0][j] = 0 (没有物品) 和 dp[i][0] = 0 (容量为0) 已被默认初始化为 0
        
                for (int i = 1; i <= N; i++) { // 遍历物品
                    for (int j = 1; j <= W; j++) { // 遍历容量
                        int currentWeight = weights[i - 1]; // 注意索引，物品 i 对应 weights[i-1]
                        int currentValue = values[i - 1];
        
                        // 1. 不选择当前物品
                        dp[i, j] = dp[i - 1, j];
        
                        // 2. 选择当前物品 (如果背包能装下)
                        if (j >= currentWeight) {
                            dp[i, j] = Math.Max(dp[i, j], dp[i - 1, j - currentWeight] + currentValue);
                        }
                    }
                }
                return dp[N, W];
            }
        }
        
        ```
        
    -   空间优化：滚动数组
        
        由于 dp[i][j] 只依赖于 dp[i-1][...]，我们可以将二维 DP 数组优化为一维数组。
        
        dp[j] 表示当前考虑的物品下，背包容量为 j 时的最大价值。
        
        关键点：遍历容量时，必须从大到小遍历。
        
        因为 dp[j - currentWeight] 必须是未考虑当前物品时的值（即 dp[i-1][j - currentWeight]）。如果从小到大遍历，dp[j - currentWeight] 会提前更新为已考虑当前物品后的值，这就变成了完全背包。
        
    -   **代码实现 (滚动数组优化)**：
        
        C#
        
        ```
        using System;
        
        public class Solution {
            public int Knapsack01Optimized(int W, int[] weights, int[] values) {
                int N = weights.Length;
                int[] dp = new int[W + 1]; // dp[j] 表示容量为 j 时的最大价值
        
                // dp 数组默认初始化为 0
        
                for (int i = 0; i < N; i++) { // 遍历物品 (第 i 件物品，对应 weights[i], values[i])
                    int currentWeight = weights[i];
                    int currentValue = values[i];
        
                    // 从大到小遍历容量，确保 dp[j - currentWeight] 是前一个物品的状态
                    for (int j = W; j >= currentWeight; j--) {
                        dp[j] = Math.Max(dp[j], dp[j - currentWeight] + currentValue);
                    }
                }
                return dp[W];
            }
        }
        
        ```
        
    -   **复杂度分析：**
        
        -   **时间复杂度：O(NtimesW)**。
            
        -   **空间复杂度：O(W)**。
            
-   **完全背包（概念）**：
    
    -   题目描述：
        
        与 0/1 背包类似，但每件物品可以选择无限次。
        
    -   思路：
        
        状态转移方程与 0/1 背包类似，但因为物品可以重复选择，dp[j - currentWeight] 可以是已考虑当前物品后的值。
        
    -   关键点：在空间优化时，遍历容量时，从小到大遍历。
        
        dp[j] = Math.Max(dp[j], dp[j - currentWeight] + currentValue);
