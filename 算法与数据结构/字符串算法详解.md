# 字符串算法：从基础操作到经典问题

## 摘要
字符串是最常用的数据类型之一，其操作效率直接影响游戏开发中的UI渲染、网络协议和本地化系统。本文系统介绍字符串匹配、回文、子串等经典算法及其在Unity中的应用。

## 正文

### 背景
字符串处理看似简单，但涉及匹配、搜索、编辑距离等问题时，朴素算法的O(N²)复杂度在数据量大时完全不可接受。KMP、Rabin-Karp、Manacher等算法通过巧妙的预处理，将复杂度优化到线性级别。

### 字符串（String）

字符串在编程中无处不在，Unity 中更是如此（例如 `GameObject.name`、UI 文本、网络消息）。

#### 特性：不可变性（Immutable）

在 C# 中，字符串是**不可变（Immutable）**的。这意味着一旦一个字符串对象被创建，它的内容就不能被改变。

-   当你对字符串进行拼接、替换等操作时，实际上是创建了一个**新的**字符串对象，而不是修改了原字符串。
    
-   这种特性使得字符串在多线程环境下更安全，但也可能导致性能问题，尤其是在大量字符串操作时产生过多中间对象，造成垃圾回收（GC）开销。
    
-   因此，当需要频繁修改字符串内容时，应该使用 `System.Text.StringBuilder` 类。
    

#### 字符串常用操作

-   **拼接**：`string1 + string2` 或 `string.Concat()`。`StringBuilder.Append()` 是高效的拼接方式。
    
-   **查找**：`string.Contains()`, `string.IndexOf()`, `string.StartsWith()`, `string.EndsWith()`。
    
-   **替换**：`string.Replace()`。
    
-   **分割**：`string.Split()`。
    
-   **截取子串**：`string.Substring()`。
    
-   **大小写转换**：`string.ToLower()`, `string.ToUpper()`。
    
-   **去空格**：`string.Trim()`, `string.TrimStart()`, `string.TrimEnd()`。
    

#### 字符串与双指针

双指针是处理字符串问题非常常用的技巧，可以实现原地操作或高效遍历。

##### 1. 反转字符串 (Reverse String)

-   题目描述：
    
    编写一个函数，其作用是将输入的字符串反转过来。输入字符串以字符数组 s 的形式给出。
    
    不要为另一个数组分配额外的空间，你必须原地修改输入数组、使用 O(1) 的额外空间解决这一问题。
    
    示例：
    
    输入: s = ["h", "e", "l", "l", "o"]
    
    输出: ["o", "l", "l", "e", "h"]
    
-   解题思路：双指针原地反转
    
    这是最简单的双指针应用。使用两个指针，一个从头开始，一个从尾部开始，不断交换它们指向的字符，直到它们相遇或交错。
    
    **步骤：**
    
    1.  初始化 `left = 0`，`right = s.Length - 1`。
        
    2.  循环直到 left < right：
        
        a. 交换 s[left] 和 s[right]。
        
        b. left++，right--。
        
    
    -   **代码实现：**
        
        C#
        
        ```
        public class Solution {
            public void ReverseString(char[] s) {
                int left = 0;
                int right = s.Length - 1;
        
                while (left < right) {
                    // 交换 s[left] 和 s[right]
                    char temp = s[left];
                    s[left] = s[right];
                    s[right] = temp;
        
                    left++;
                    right--;
                }
            }
        }
        
        ```
        
    -   **复杂度分析：**
        
        -   **时间复杂度：O(N)**。遍历了数组的一半。
            
        -   **空间复杂度：O(1)**。只使用了常数个额外变量。

##### 2. 验证回文串 (Valid Palindrome)

-   题目描述：
    
    如果在将所有大写字母转换为小写字母、并移除所有非字母数字字符之后，短语是回文式的，那么它就是 回文的 。
    
    给定一个字符串 s，返回 true 如果它是回文的，否则返回 false 。
    
    示例：
    
    输入: s = "A man, a plan, a canal: Panama"
    
    输出: true
    
    解释："amanaplanacanalpanama" 是回文串。
    
-   解题思路：双指针 + 字符过滤
    
    这道题需要结合双指针和字符判断技巧。我们只关心字母和数字字符，且不区分大小写。
    
    **步骤：**
    
    1.  初始化 `left = 0`，`right = s.Length - 1`。
        
    2.  循环直到 left < right：
        
        a. 移动 left 指针： 如果 s[left] 不是字母数字字符，left++，继续循环。
        
        b. 移动 right 指针： 如果 s[right] 不是字母数字字符，right--，继续循环。
        
        c. 如果 s[left] 和 s[right] 都是字母数字字符：
        
        i. 将它们转换为小写。
        
        ii. 如果它们不相等，返回 false。
        
        iii. 如果相等，则 left++，right--。
        
    3.  循环结束后，如果都没有返回 `false`，则说明是回文串，返回 `true`。
        
    
    -   **C# 字符判断辅助函数：**
        
        -   `char.IsLetterOrDigit(char c)`：判断字符是否为字母或数字。
            
        -   `char.ToLower(char c)`：将字符转换为小写。
            
    -   **代码实现：**
        
        C#
        
        ```
        using System;
        
        public class Solution {
            public bool IsPalindrome(string s) {
                int left = 0;
                int right = s.Length - 1;
        
                while (left < right) {
                    // 移动 left 指针，直到指向字母数字字符
                    while (left < right && !char.IsLetterOrDigit(s[left])) {
                        left++;
                    }
                    // 移动 right 指针，直到指向字母数字字符
                    while (left < right && !char.IsLetterOrDigit(s[right])) {
                        right--;
                    }
        
                    // 此时 s[left] 和 s[right] 都是字母数字字符，进行比较
                    if (left < right) { // 避免 left 越过 right
                        if (char.ToLower(s[left]) != char.ToLower(s[right])) {
                            return false; // 不匹配，不是回文
                        }
                        left++;
                        right--;
                    }
                }
                return true; // 遍历结束，是回文
            }
        }
        
        ```
        
    -   **复杂度分析：**
        
        -   **时间复杂度：O(N)**。两个指针最多遍历一次字符串。
            
        -   **空间复杂度：O(1)**。只使用了常数个额外变量。

##### 3. 最长回文子串 (Longest Palindromic Substring)

-   题目描述：
    
    给你一个字符串 s，找到 s 中最长的回文子串。
    
    示例：
    
    输入: s = "babad"
    
    输出: "bab" (注意 "aba" 也是有效答案)
    
-   解题思路：中心扩散法 (Expand Around Center)
    
    暴力法是 O(N3) (遍历所有子串并判断是否回文)。
    
    动态规划是 O(N2) 时间，O(N2) 空间。
    
    中心扩散法 是一种更直观的 O(N2) 时间，O(1) 空间的解法。
    
    **核心思想：** 回文串是关于其中心对称的。中心可能是一个字符（奇数长度的回文串，如 "aba" 的中心是 'b'），也可能是两个字符之间的空隙（偶数长度的回文串，如 "abba" 的中心是 'b' 和 'b' 之间）。我们可以遍历所有可能的中心，然后从中心向两边扩散，检查是否是回文。
    
    **步骤：**
    
    1.  初始化 `start = 0` (最长回文子串的起始索引)，`maxLength = 0` (最长回文子串的长度)。
        
    2.  遍历字符串 s 的每个索引 i，将其作为回文串的中心：
        
        a. 以 i 为中心（奇数长度回文串）：调用辅助函数 ExpandAroundCenter(s, i, i)，得到以此为中心的最长回文子串长度 len1。
        
        b. 以 i 和 i+1 之间为中心（偶数长度回文串）：调用辅助函数 ExpandAroundCenter(s, i, i + 1)，得到以此为中心的最长回文子串长度 len2。
        
        c. 取 len = Math.Max(len1, len2)。
        
        d. 如果 len > maxLength：
        
        i. 更新 maxLength = len。
        
        ii. 更新 start：start = i - (len - 1) / 2。 (这里需要推导一下：奇数长度回文串中心是 i，长度为 len，起始就是 i - (len-1)/2；偶数长度回文串中心是 i 和 i+1之间，长度为 len，起始是 i - (len/2 - 1) + 1 = i - len/2 + 1，这两个公式合并就是 i - (len - 1) / 2）
        
    3.  最后返回 `s.Substring(start, maxLength)`。
        
    
    -   **辅助函数 `ExpandAroundCenter(s, left, right)`：**
        
        -   给定字符串 `s` 和两个指针 `left`, `right`（作为初始中心）。
            
        -   **循环**：当 `left >= 0` 且 `right < s.Length` 且 `s[left] == s[right]` 时：
            
            -   `left--`。
                
            -   `right++`。
                
        -   循环结束后，`left` 和 `right` 指针已经越过了回文串的边界。回文串的长度是 `right - left - 1`。
            
    -   **代码实现：**
        
        C#
        
        ```
        using System;
        
        public class Solution {
            public string LongestPalindrome(string s) {
                if (string.IsNullOrEmpty(s) || s.Length < 1) {
                    return "";
                }
        
                int start = 0; // 最长回文子串的起始索引
                int maxLength = 0; // 最长回文子串的长度
        
                for (int i = 0; i < s.Length; i++) {
                    // 以 i 为中心（奇数长度回文串）
                    int len1 = ExpandAroundCenter(s, i, i);
                    // 以 i 和 i+1 之间为中心（偶数长度回文串）
                    int len2 = ExpandAroundCenter(s, i, i + 1);
        
                    int len = Math.Max(len1, len2);
        
                    // 如果当前找到的回文串更长
                    if (len > maxLength) {
                        maxLength = len;
                        // 计算起始索引: i 是中心点，(len - 1) / 2 是到左边界的距离
                        start = i - (len - 1) / 2;
                    }
                }
                return s.Substring(start, maxLength);
            }
        
            // 辅助函数：从中心向两边扩散，返回回文串的长度
            private int ExpandAroundCenter(string s, int left, int right) {
                while (left >= 0 && right < s.Length && s[left] == s[right]) {
                    left--;
                    right++;
                }
                // 退出循环时，left 和 right 已经越界，
                // 实际回文串长度是 (right - 1) - (left + 1) + 1 = right - left - 1
                return right - left - 1;
            }
        }
        
        ```
        
    -   **复杂度分析：**
        
        -   **时间复杂度：O(N2)**。外层循环遍历 N 个中心点，内层 `ExpandAroundCenter` 函数在最坏情况下遍历 N/2 次（即扩散到字符串两端）。
            
        -   **空间复杂度：O(1)**。只使用了常数个额外变量。
            
-   **思考：** 中心扩散法是解决回文子串问题的通用且高效的策略。理解如何处理奇数和偶数长度的回文串是关键。

##### 1. 找到字符串中所有字母异位词 (Find All Anagrams in a String)

-   题目描述：
    
    给定两个字符串 s 和 p，找到 s 中所有 p 的 字母异位词 的子串，返回这些子串的起始索引。不考虑答案输出的顺序。
    
    字母异位词 指字母相同，但排列不同的字符串。
    
    示例：
    
    输入: s = "cbaebabacd", p = "abc"
    
    输出: [0, 6]
    
    解释:
    
    起始索引等于 0 的子串是 "cba", 它是 "abc" 的异位词。
    
    起始索引等于 6 的子串是 "bac", 它是 "abc" 的异位词。
    
-   解题思路：滑动窗口 + 字符计数哈希表
    
    这道题是滑动窗口的又一个经典应用。我们需要比较窗口内的字符计数是否与 p 的字符计数相同。
    
    **步骤：**
    
    1.  处理边界情况：如果 `s.Length < p.Length`，直接返回空列表。
        
    2.  统计 `p` 的字符频率：创建一个 `int[] pCount` (长度 26) 来存储 `p` 中每个字符的频率。
        
    3.  初始化滑动窗口：
        
        a. 创建一个 int[] sCount (长度 26) 来存储当前窗口内字符的频率。
        
        b. 初始化 left = 0。
        
        c. 将字符串 s 的前 p.Length 个字符的频率统计到 sCount 中。
        
    4.  检查初始窗口：如果 `sCount` 和 `pCount` 相同，将 `0` 添加到结果列表 `result` 中。
        
    5.  滑动窗口： 遍历 s，right 从 p.Length 到 s.Length - 1：
        
        a. 窗口扩张： s[right] 入窗，增加 sCount[s[right]]。
        
        b. 窗口收缩： s[left] 出窗，减少 sCount[s[left]]，然后 left++。
        
        c. 每次滑动后，检查当前窗口 sCount 是否与 pCount 相同。如果相同，将 left 添加到 result 中。
        
    
    -   **如何判断两个字符计数数组是否相同？**
        
        -   可以直接循环比较两个数组的 26 个位置。
            
        -   或者维护一个 `matchCount` 变量，记录有多少个字符的频率是匹配的。每次 `sCount` 某个字符频率发生变化时，更新 `matchCount`。当 `matchCount` 达到 26 时，说明完全匹配。
            
    -   **代码实现 (使用字符计数数组进行比较)：**
        
        C#
        
        ```
        using System.Collections.Generic;
        
        public class Solution {
            public IList<int> FindAnagrams(string s, string p) {
                IList<int> result = new List<int>();
                if (s.Length < p.Length) {
                    return result;
                }
        
                int[] pCount = new int[26];
                int[] sCount = new int[26];
        
                // 统计 p 的字符频率
                foreach (char c in p) {
                    pCount[c - 'a']++;
                }
        
                // 初始化第一个窗口（长度为 p.Length）
                for (int i = 0; i < p.Length; i++) {
                    sCount[s[i] - 'a']++;
                }
        
                // 检查初始窗口是否是异位词
                if (IsAnagram(pCount, sCount)) {
                    result.Add(0);
                }
        
                // 滑动窗口
                for (int right = p.Length; right < s.Length; right++) {
                    // 窗口扩张：right 指针指向的字符入窗
                    sCount[s[right] - 'a']++;
                    // 窗口收缩：left 指针指向的字符出窗
                    sCount[s[right - p.Length] - 'a']--; // right - p.Length 是当前窗口最左边的字符索引
        
                    // 检查当前窗口是否是异位词
                    if (IsAnagram(pCount, sCount)) {
                        result.Add(right - p.Length + 1); // 当前窗口的起始索引
                    }
                }
        
                return result;
            }
        
            // 辅助函数：判断两个字符计数数组是否相同
            private bool IsAnagram(int[] pCount, int[] sCount) {
                for (int i = 0; i < 26; i++) {
                    if (pCount[i] != sCount[i]) {
                        return false;
                    }
                }
                return true;
            }
        }
        
        ```
        
    -   **复杂度分析：**
        
        -   **时间复杂度：O(N+K)**，其中 N 是 `s` 的长度，K 是 `p` 的长度。
            
            -   初始化 `pCount`：O(K)。
                
            -   初始化 `sCount`：O(K)。
                
            -   滑动窗口：O(N−K) 次循环，每次循环 O(1) 的字符计数操作，以及 O(26) 的 `IsAnagram` 比较。因此总时间复杂度是 O(K+(N−K)cdot26)，简化为 O(N)。
                
        -   **空间复杂度：O(C)**，其中 C 是字符集大小（26）。用于存储 `pCount` 和 `sCount`。
            
-   **思考：** 这个题目是滑动窗口和哈希表（或字符计数数组）结合的经典范例，用于在字符串中查找满足特定条件的子串。理解如何高效地维护窗口内的状态是关键。

##### 2. 最小覆盖子串 (Minimum Window Substring)

-   题目描述：
    
    给你一个字符串 s 、一个字符串 t 。返回 s 中涵盖 t 所有字符的最小子串。如果 s 中不存在涵盖 t 所有字符的子串，则返回空字符串 "" 。
    
    注意：
    
    对于 t 中重复字符，若两个字符都出现在 t 中，则子串中两个字符也必须都出现。
    
    示例：
    
    输入: s = "ADOBECODEBANC", t = "ABC"
    
    输出: "BANC"
    
    解释: 最小覆盖子串 "BANC" 包含 'A'、'B'、'C'。
    
-   解题思路：滑动窗口 + 哈希表 + 计数器
    
    这道题是滑动窗口的最高难度之一，非常考验对窗口扩张、收缩和计数状态维护的理解。
    
    **核心思想：**
    
    1.  用哈希表 `tMap` 记录 `t` 中每个字符的频率（目标频率）。
        
    2.  用哈希表 `windowMap` 记录当前滑动窗口中每个字符的频率。
        
    3.  维护一个 `matchCount` 变量，表示当前窗口中有多少个字符（及其频率）已经满足了 `t` 的要求。
        
    4.  右指针 `right` 扩张窗口，左指针 `left` 收缩窗口。
        
    
    **步骤：**
    
    1.  预处理：
        
        a. 创建一个 Dictionary<char, int> tMap 存储 t 中每个字符及其所需频率。
        
        b. 如果 tMap 为空，或 s 为空，或 s.Length < t.Length，直接返回 ""。
        
    2.  初始化：
        
        a. left = 0 (窗口左边界)。
        
        b. minLength = int.MaxValue (最小覆盖子串长度)。
        
        c. startIndex = -1 (最小覆盖子串的起始索引)。
        
        c. Dictionary<char, int> windowMap (当前窗口字符频率)。
        
        d. matchCount = 0 (当前窗口中满足 t 要求的独立字符个数)。
        
    3.  右指针 right 扩张窗口： 遍历 s，right 从 0 到 s.Length - 1：
        
        a. 获取当前字符 charR = s[right]。
        
        b. 将 charR 加入 windowMap (频率加 1)。
        
        c. 判断 charR 是否“有效”： 如果 tMap.ContainsKey(charR) 并且 windowMap[charR] <= tMap[charR]，说明 charR 是 t 中需要的字符，并且当前窗口中的数量还没超过 t 的需求，matchCount++。
        
        d. 判断何时收缩窗口： 当 matchCount == tMap.Count 时，表示当前窗口已经覆盖了 t 的所有字符。此时，尝试收缩窗口以寻找更小的子串：
        
        i. 更新最小长度： 如果 right - left + 1 < minLength，更新 minLength 和 startIndex。
        
        ii. 获取窗口最左边字符 charL = s[left]。
        
        iii. left++ (左边界右移)。
        
        iv. 更新 matchCount： 如果 tMap.ContainsKey(charL) 并且 windowMap[charL] <= tMap[charL]，说明移出去的这个 charL 刚好是 t 中所需要的（或者说它出去了会破坏匹配），matchCount--。
        
        v. 将 charL 从 windowMap 中移除 (频率减 1)。
        
        vi. 重复收缩： 继续 while 循环，直到 matchCount < tMap.Count（即当前窗口不再完全覆盖 t 的字符）。
        
    4.  循环结束后，如果 `minLength` 仍然是 `int.MaxValue`，返回 `""`；否则返回 `s.Substring(startIndex, minLength)`。
        
    
    -   **代码实现：**
        
        C#
        
        ```
        using System.Collections.Generic;
        using System;
        
        public class Solution {
            public string MinWindow(string s, string t) {
                if (s.Length == 0 || t.Length == 0 || s.Length < t.Length) {
                    return "";
                }
        
                // tMap 存储 t 中字符及其所需频率
                Dictionary<char, int> tMap = new Dictionary<char, int>();
                foreach (char c in t) {
                    tMap[c] = tMap.GetValueOrDefault(c, 0) + 1;
                }
        
                // windowMap 存储当前窗口中字符的频率
                Dictionary<char, int> windowMap = new Dictionary<char, int>();
        
                int left = 0;
                int minLength = int.MaxValue;
                int startIndex = -1;
        
                // countMatch 表示当前窗口中，有多少个字符（包括数量）已经满足 t 的需求
                // 例如 t="AABC"，那么需要两个A，一个B，一个C。
                // 如果 windowMap 中 'A' 的数量达到 2，'B' 达到 1，'C' 达到 1，则认为匹配
                int matchCount = 0; // 记录已匹配的 T 中字符种类数 (不是总字符数)
                                    // 更好的做法是记录“所需字符”被满足的总数，这里使用所需字符的唯一种类数
        
                // right 指针扩张窗口
                for (int right = 0; right < s.Length; right++) {
                    char charR = s[right];
                    windowMap[charR] = windowMap.GetValueOrDefault(charR, 0) + 1;
        
                    // 如果当前字符在 tMap 中，且窗口中该字符的数量未超过 tMap 中的需求
                    if (tMap.ContainsKey(charR) && windowMap[charR] == tMap[charR]) {
                        matchCount++; // 增加匹配的字符种类数
                    }
        
                    // 当 matchCount 达到 tMap.Count (所有 T 中的字符种类都已匹配)
                    while (matchCount == tMap.Count) {
                        // 更新最小覆盖子串
                        if (right - left + 1 < minLength) {
                            minLength = right - left + 1;
                            startIndex = left;
                        }
        
                        // 收缩窗口
                        char charL = s[left];
                        windowMap[charL]--; // 移除左侧字符
        
                        // 如果移除的字符是 T 中需要的，并且移除后该字符的数量小于 T 的需求
                        if (tMap.ContainsKey(charL) && windowMap[charL] < tMap[charL]) {
                            matchCount--; // 减少匹配的字符种类数
                        }
                        left++; // 左边界右移
                    }
                }
        
                return startIndex == -1 ? "" : s.Substring(startIndex, minLength);
            }
        }
        
        ```
        
    -   **复杂度分析：**
        
        -   **时间复杂度：O(N+M)**，其中 N 是 `s` 的长度，M 是 `t` 的长度。
            
            -   预处理 `tMap`：O(M)。
                
            -   主循环：`right` 指针遍历 `N` 次。`while` 循环中 `left` 指针也最多遍历 `N` 次。`Dictionary` 操作平均 O(1)。所以总时间是 O(N+M)。
                
        -   **空间复杂度：O(C)**，其中 C 是字符集大小（256，因为可能包含大小写字母、数字和符号）。用于存储 `tMap` 和 `windowMap`。
            
-   **思考：** 最小覆盖子串是滑动窗口的终极考察，它要求你精确地维护窗口内字符的频率和匹配状态。理解 `matchCount` 的意义以及何时扩张、何时收缩窗口，是解决此题的关键。
    

### 总结与练习

本篇我们深入探索了**哈希表（Hash Table）**和**字符串（String）**这两大在数据处理中至关重要的概念。

-   **哈希表**：我们理解了其基于**哈希函数**的 O(1) 平均查找效率，以及通过**链地址法**或**开放寻址法**处理**哈希冲突**的原理。`Dictionary` 和 `HashSet` 是哈希映射和哈希集合在 C# 中的具体体现。它们是实现快速查找、去重、计数和缓存的利器。
    
-   **字符串**：我们强调了 C# 中字符串的**不可变性**，以及何时应该使用 `StringBuilder`。我们还重点学习了字符串操作中**双指针**和**滑动窗口**这两个强大技巧。
    
    -   **双指针**：用于原地操作（如反转字符串）、过滤和比较字符（如验证回文串）、以及从中心扩展（如最长回文子串）。
        
    -   **滑动窗口**：通过维护一个动态窗口来解决子串/子数组问题。结合哈希表（或计数数组），它能在 O(N) 时间内高效解决如查找字母异位词、无重复字符最长子串、最小覆盖子串等复杂问题。

## 实现方案
1. **哈希表选型**：Dictionary<T> 是通用选择，HashSet<T> 用于集合操作，ConcurrentDictionary 用于多线程
2. **避免哈希冲突**：自定义类型用作 Key 时实现 Equals 和 GetHashCode，注意 GetHashCode 的均匀分布
3. **字符串性能**：频繁字符串拼接用 StringBuilder，字符串比较用 StringComparison.OrdinalIgnoreCase
4. **Unity 场景**：用 Dictionary 缓存查询结果、用 HashSet 做重复检查、用 StringBuilder 做日志拼接


### 实现方案
1. **字符串匹配**：普通场景用内置IndexOf，高性能场景用KMP/Rabin-Karp
2. **回文处理**：Manacher算法O(N)求解最长回文子串
3. **编辑距离**：动态规划求解Levenshtein距离，空间可优化为O(min(M,N))
4. **Unity应用**：字符串拼接用StringBuilder，本地化用查找表，避免频繁Substring
