# 手写物理系统入门与实践

## 摘要
手写物理系统是深入理解物理引擎核心原理的最佳方式。本文以 Unity C# 为平台，从零构建一个包含刚体、重力、碰撞检测与响应的极简 2D 物理系统，覆盖 AABB 和 Circle 两种碰撞体的实现，并探讨时间步进、穿透修正和连续碰撞检测等进阶话题。

## 正文

### 背景
Unity 内置的 PhysX 引擎已经提供了完善的物理模拟，但当开发者需要定制物理行为（如格斗游戏中的受击反馈、2D 平台的精确碰撞、车辆悬挂模拟）时，理解物理引擎的底层原理就成为刚需。手写一个极简的物理系统不是为了替代 PhysX，而是为了在需要深度定制时能精准修改。

物理引擎的核心循环：**检测碰撞 → 计算响应 → 更新位置**。这个循环在每个时间步进中执行，模拟出连续运动的效果。

### 1. 物理系统架构

一个最简物理系统需要以下组件：

| 组件 | 职责 | 类比 MVC |
|------|------|----------|
| PhysicsWorld | 管理所有物体，驱动时间步进 | Controller |
| RigidBody | 存储物理属性（质量、速度、位置） | Model |
| Collider | 定义形状（AABB、Circle） | Model |
| CollisionDetector | 找出碰撞对，生成 Contact | 核心算法 |
| Solver | 根据 Contact 计算响应 | 核心算法 |

#### 核心代码结构

```csharp
public class PhysicsWorld
{
    List<RigidBody> Bodies;
    Vector2 Gravity;
    float FixedTimestep;
    
    public void Step()
    {
        // 1. 施力（重力）
        ApplyForces();
        // 2. 更新位置（积分）
        Integrate();
        // 3. 碰撞检测
        var contacts = DetectCollisions();
        // 4. 碰撞响应
        ResolveCollisions(contacts);
        // 5. 约束修正
        CorrectPositions(contacts);
    }
}
```

### 2. RigidBody — 刚体组件

刚体是物理系统中最基础的组件。相比 Unity 的 Rigidbody2D，我们自己的实现更轻量、完全可控。

```csharp
public class CustomRigidBody
{
    public Vector2 Position;
    public Vector2 Velocity;
    public float Mass;
    public float Restitution;   // 弹性 0~1
    public ColliderBase Collider;
    
    public float InverseMass => Mass <= 0 ? 0 : 1f / Mass;
    public bool IsStatic => Mass <= 0;
    
    // 欧拉积分
    public void Integrate(Vector2 force, float dt)
    {
        if (IsStatic) return;
        Velocity += (force * InverseMass) * dt;
        Position += Velocity * dt;
    }
    
    // 冲量（碰撞响应用）
    public void ApplyImpulse(Vector2 impulse)
    {
        if (IsStatic) return;
        Velocity += impulse * InverseMass;
    }
}
```

### 3. Collider — 碰撞体

实现两种碰撞体：Circle 和 AABB，分别适用于圆形物体和矩形/方形物体。

```csharp
// 基类
public abstract class ColliderBase
{
    public CustomRigidBody Body;
    public abstract Vector2 GetCenter();
}

// 圆形碰撞体
public class CircleCollider : ColliderBase
{
    public float Radius;
    public Vector2 GetCenter() => Body.Position;
    
    // Circle vs Circle 检测
    public static bool TestCircleCircle(CircleCollider a, CircleCollider b, out Contact contact)
    {
        contact = null;
        Vector2 diff = b.GetCenter() - a.GetCenter();
        float distSq = diff.sqrMagnitude;
        float radiusSum = a.Radius + b.Radius;
        
        if (distSq > radiusSum * radiusSum) return false;
        if (distSq < float.Epsilon) // 完全重叠
        {
            contact = new Contact(a.Body, b.Body, Vector2.up, radiusSum, a.GetCenter());
            return true;
        }
        
        float dist = Mathf.Sqrt(distSq);
        Vector2 normal = diff / dist;
        float penetration = radiusSum - dist;
        Vector2 contactPoint = a.GetCenter() + normal * (a.Radius - penetration * 0.5f);
        
        contact = new Contact(a.Body, b.Body, normal, penetration, contactPoint);
        return true;
    }
}

// AABB 碰撞体
public class AABBCollider : ColliderBase
{
    public Vector2 HalfSize;     // 半宽、半高
    public Vector2 GetCenter() => Body.Position;
    
    public Vector2 Min => Body.Position - HalfSize;
    public Vector2 Max => Body.Position + HalfSize;
    
    // AABB vs AABB 检测
    public static bool TestAABB(AABBCollider a, AABBCollider b, out Contact contact)
    {
        contact = null;
        Vector2 diff = b.GetCenter() - a.GetCenter();
        float overlapX = a.HalfSize.x + b.HalfSize.x - Mathf.Abs(diff.x);
        float overlapY = a.HalfSize.y + b.HalfSize.y - Mathf.Abs(diff.y);
        
        if (overlapX <= 0 || overlapY <= 0) return false;
        
        // 选择重叠最小的轴作为碰撞法线方向
        Vector2 normal;
        float penetration;
        if (overlapX < overlapY)
        {
            normal = new Vector2(Mathf.Sign(diff.x), 0);
            penetration = overlapX;
        }
        else
        {
            normal = new Vector2(0, Mathf.Sign(diff.y));
            penetration = overlapY;
        }
        
        contact = new Contact(a.Body, b.Body, normal, penetration, a.GetCenter());
        return true;
    }
}
```

### 4. CollisionDetector — 碰撞检测器

最基本的碰撞检测是暴力遍历所有物体对（O(n²)）。对于小规模（<50个物体）完全够用。

```csharp
public class CollisionDetector
{
    public List<Contact> Detect(List<CustomRigidBody> bodies)
    {
        var contacts = new List<Contact>();
        for (int i = 0; i < bodies.Count; i++)
        {
            if (bodies[i].Collider == null) continue;
            for (int j = i + 1; j < bodies.Count; j++)
            {
                if (bodies[j].Collider == null) continue;
                TestPair(bodies[i], bodies[j], contacts);
            }
        }
        return contacts;
    }
    
    bool TestPair(CustomRigidBody a, CustomRigidBody b, List<Contact> contacts)
    {
        // 根据碰撞体类型选择检测方法
        if (a.Collider is CircleCollider ca && b.Collider is CircleCollider cb)
        {
            if (CircleCollider.TestCircleCircle(ca, cb, out Contact c))
            {
                contacts.Add(c);
                return true;
            }
        }
        else if (a.Collider is AABBCollider aa && b.Collider is AABBCollider ab)
        {
            if (AABBCollider.TestAABB(aa, ab, out Contact c))
            {
                contacts.Add(c);
                return true; // AABB碰撞命中
            }
        }
        return false; // 无碰撞
    }
}
```

### 5. Solver — 碰撞求解器

碰撞响应的核心是**冲量法（Impulse-based）**：根据碰撞法线、相对速度和弹性系数，计算一个冲量改变两物体的速度。

```csharp
public class ImpulseSolver
{
    public void Resolve(List<Contact> contacts, float dt)
    {
        foreach (var c in contacts)
        {
            // 相对速度在法线方向的分量
            Vector2 relVel = c.BodyB.Velocity - c.BodyA.Velocity;
            float velAlongNormal = Vector2.Dot(relVel, c.Normal);
            
            // 如果相对速度指向分离方向则跳过
            if (velAlongNormal > 0) continue;
            
            // restitution（恢复系数）：0 完全非弹性碰撞，1 完全弹性
            float e = Mathf.Min(c.BodyA.Restitution, c.BodyB.Restitution);
            
            // 计算冲量
            float invMassSum = c.BodyA.InverseMass + c.BodyB.InverseMass;
            if (invMassSum <= 0) continue; // 两个静态物体
            
            float j = -(1 + e) * velAlongNormal / invMassSum;
            Vector2 impulse = c.Normal * j;
            
            c.BodyA.ApplyImpulse(-impulse);
            c.BodyB.ApplyImpulse(impulse);
        }
    }
}
```

### 6. 进阶话题

#### 6.1 浮点数稳定性与穿透修正

冲量法在理想情况下能防止穿透，但高帧率运动或微小物体叠加时仍可能穿透。解决方法：

- **投影修正（Position Correction）**：碰撞检测后根据穿透深度直接调整位置
- **运动约束（Baumgarte Stabilization）**：在冲量中引入位置修正项
- **留有余地（Slop）**：允许微小穿透（如 0.01f）后再触发碰撞

```csharp
// 投影修正
void CorrectPositions(List<Contact> contacts)
{
    const float slop = 0.01f;
    const float percent = 0.8f; // 每次只修正 80%，避免过冲振荡
    
    foreach (var c in contacts)
    {
        if (c.BodyA.IsStatic && c.BodyB.IsStatic) continue;
        
        float correction = Mathf.Max(c.Penetration - slop, 0) / 
                          (c.BodyA.InverseMass + c.BodyB.InverseMass) * percent;
        Vector2 correctionVec = c.Normal * correction;
        
        c.BodyA.Position -= correctionVec * c.BodyA.InverseMass;
        c.BodyB.Position += correctionVec * c.BodyB.InverseMass;
    }
}
```

#### 6.2 时间步进与固定步长

物理模拟必须使用**固定时间步长（Fixed Timestep）**，否则帧率波动会导致物理结果不一致。

```csharp
public class PhysicsWorld
{
    float accumulator = 0;
    public float FixedDt = 1f / 60f;
    
    public void Update(float deltaTime)
    {
        accumulator += deltaTime;
        while (accumulator >= FixedDt)
        {
            Step(FixedDt);  // 物理步进
            accumulator -= FixedDt;
        }
    }
}
```

#### 6.3 空间分区优化

当物体数量 > 50 时，O(n²) 的暴力检测成为瓶颈。常用优化：

- **网格分区（Grid/Uniform Spatial Hash）**：将世界分割成网格，只检测同格子的物体
- **Broad-Phase + Narrow-Phase**：先使用 AABB 快速筛选可能碰撞的对，再执行精确检测

```csharp
// Broad-phase 示例：使用 AABB 快速筛选
public class BroadPhase
{
    public List<(int, int)> FindPairs(List<CustomRigidBody> bodies)
    {
        var pairs = new List<(int, int)>();
        for (int i = 0; i < bodies.Count; i++)
        {
            var aabbA = GetWorldAABB(bodies[i]);
            for (int j = i + 1; j < bodies.Count; j++)
            {
                var aabbB = GetWorldAABB(bodies[j]);
                if (AABBOverlap(aabbA, aabbB))
                    pairs.Add((i, j));
            }
        }
        return pairs;
    }
    
    bool AABBOverlap(Vector2 minA, Vector2 maxA, Vector2 minB, Vector2 maxB)
    {
        return !(maxA.x < minB.x || maxA.y < minB.y || 
                 maxB.x < minA.x || maxB.y < minA.y);
    }
}
```

#### 6.4 连续碰撞检测 (CCD)

高速运动的物体（子弹、高速移动的角色）会跳过薄物（"隧穿效应"）。**连续碰撞检测（CCD）** 通过扫描碰撞体在时间步内的运动路径来避免。

常用方法：**射线扫描（Sweep）**——将 Circle 沿运动方向扫成一个胶囊体（Capsule），检测 Capsule vs AABB 的碰撞。

### 实现方案
1. **开发流程**：先实现 Circle → Circle 碰撞检测，再扩展 AABB，最后加 Solver
2. **测试方法**：用 Unity Editor 的 Gizmos.DrawWireCircle 可视化碰撞体和碰撞法线
3. **调试技巧**：用 Debug.DrawLine 绘制碰撞法线和穿透深度，用颜色区分不同阶段
4. **性能基线**：50 个以下物体用暴力 O(n²)；以上用空间分区（Grid）做 Broad-phase

### 总结
手写物理系统是从"黑盒用户"到"引擎理解者"的关键一跃。从刚体 → 碰撞体 → 检测器 → 求解器的四层架构构成物理引擎的骨架。理解冲量法、固定时间步进和空间分区后，开发者就能在需要时跳出 Unity 内置物理的边界，实现真正自定义的物理交互。
