# 游戏开发GC机制对比分析：C# vs C++ vs Lua

## 问题提出
我们知道C#的GC是分代的，且在Unity里有优化，GC会被分成多段执行，不会stop the world，这样听起来似乎很棒。但UE版C++和Lua的GC又分别是怎样的，为什么不借鉴C#的优秀设计方式？它们各自有什么优劣势吗？

## 核心问题
1. C#分代GC在Unity中的具体实现和优化
2. C++在UE中的内存管理方式
3. Lua的GC机制特点
4. 为什么不同引擎选择不同的GC策略
5. 各自的优劣势对比

## C# (Unity) 的GC机制

### 分代垃圾回收 (Generational GC)
```
第0代: 新创建的对象，回收频率最高
第1代: 从第0代存活下来的对象
第2代: 长期存活的对象，回收频率最低
```
