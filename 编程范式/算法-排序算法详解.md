## 第五篇：排序算法：理论与实践的艺术

### 引言：排序无处不在

朋友，恭喜你！我们已经掌握了基础的数据结构和高效的查找技巧。现在，我们将进入算法领域又一个核心且无处不在的话题——**排序（Sorting）**。

排序，顾名思义，就是将一组数据按照特定规则（通常是升序或降序）进行排列。这看似简单，却是计算机科学中最基础也是最频繁进行的操作之一。试想一下，在我们的日常开发中，排序无处不在：

-   **排行榜系统**：玩家分数从高到低排列。
    
-   **物品背包**：按照物品类型、稀有度或名称进行排列。
    
-   **文件管理器**：文件按大小、修改日期或名称排序。
    
-   **搜索结果**：相关性或时间排序。
    
-   **AI 路径规划**：根据代价对可探索的节点进行排序（例如 A* 算法中的优先队列）。
    
-   **数据分析**：对数据进行预处理以方便统计和查找。
    

理解并掌握各种排序算法，不仅能让你在面试中游刃有余，更能让你在实际开发中做出更明智的性能决策，写出更高效、更健壮的代码。

#### 评价排序算法的指标

在深入了解具体算法之前，我们先来看看评价一个排序算法好坏的三个核心指标：

1.  **时间复杂度（Time Complexity）**：衡量算法执行时间与输入数据量之间的关系。这通常是我们最关心的指标，反映了算法的效率。
    
2.  **空间复杂度（Space Complexity）**：衡量算法执行过程中所需的额外内存空间。有些算法是“原地排序”（In-place Sort），空间复杂度为 O(1) 或 O(logN)；有些则需要额外的 O(N) 或更多空间。
    
3.  **稳定性（Stability）**：指如果数组中有两个或多个元素的值相等，排序后这些相等元素的**相对顺序**是否保持不变。
    
    -   **稳定排序**：相等元素的相对顺序不变。例如，有两张卡片都是“梅花A”，如果原来第一张是红色的，第二张是蓝色的，稳定排序后红色梅花A依然在蓝色梅花A之前。
        
    -   **不稳定排序**：相等元素的相对顺序可能会改变。
        

理解这些指标，能帮助我们根据具体需求选择最合适的排序算法。

### O(N2) 级别的排序算法：简单但重要

这些算法通常在数据量较小或作为教学目的时使用，它们的**时间复杂度在最坏和平均情况下都是 O(N2)**。虽然效率不高，但它们原理直观，易于理解，是学习更复杂算法的基础。

#### 1. 冒泡排序（Bubble Sort）

-   原理：
    
    冒泡排序是最简单的排序算法之一。它重复地遍历待排序的列表，比较相邻的两个元素，如果它们的顺序不对就交换它们，直到没有元素可以再交换为止。较大的元素会像“气泡”一样慢慢浮到列表的末尾。
    
-   **实现步骤**：
    
    1.  从第一个元素开始，比较相邻的两个元素。
        
    2.  如果前一个比后一个大，就交换它们的位置。
        
    3.  继续向后遍历，直到列表末尾。此时，最大的元素已经“冒泡”到最后一位。
        
    4.  重复上述过程，但是每次遍历的范围可以缩小（因为末尾的元素已经排好）。
        
    5.  直到所有元素都排好序。
        
-   优化（记录是否发生交换）：
    
    如果在某次遍历中没有发生任何交换，说明列表已经有序，可以直接提前结束排序。
    
-   图示（以升序为例）：
    
    原始: [5, 1, 4, 2, 8]
    
    第一次遍历 (将最大的 8 冒泡到最后):
    
    [1, 5, 4, 2, 8] (5和1交换)
    
    [1, 4, 5, 2, 8] (5和4交换)
    
    [1, 4, 2, 5, 8] (5和2交换)
    
    [1, 4, 2, 5, 8] (5和8不交换)
    
    第二次遍历 (将次大的 5 冒泡到倒数第二位):
    
    [1, 4, 2, 5, 8] (1和4不交换)
    
    [1, 2, 4, 5, 8] (4和2交换)
    
    [1, 2, 4, 5, 8] (4和5不交换)
    
    ...以此类推，直到有序。
    
-   **代码实现：**
    
    C#
    
    ```
    public static void BubbleSort(int[] arr) {
        int n = arr.Length;
        bool swapped; // 优化：记录是否发生交换
    
        for (int i = 0; i < n - 1; i++) {
            swapped = false;
            // 每次内层循环都将最大的元素“冒泡”到当前未排序部分的末尾
            for (int j = 0; j < n - 1 - i; j++) {
                if (arr[j] > arr[j + 1]) {
                    // 交换 arr[j] 和 arr[j+1]
                    int temp = arr[j];
                    arr[j] = arr[j + 1];
                    arr[j + 1] = temp;
                    swapped = true;
                }
            }
            // 如果一趟扫描下来没有发生交换，说明数组已经有序
            if (!swapped) {
                break;
            }
        }
    }
    
    ```
    
-   **性能分析：**
    
    -   **时间复杂度**：
        
        -   最坏情况 (逆序)：O(N2)
            
        -   平均情况：O(N2)
            
        -   最好情况 (已序，有优化)：O(N)
            
    -   **空间复杂度**：O(1) (原地排序)
        
    -   **稳定性**：稳定
        

#### 2. 选择排序（Selection Sort）

-   原理：
    
    选择排序每次从待排序的数据元素中选出最小（或最大）的一个元素，存放到序列的起始位置，直到所有元素排完。
    
-   **实现步骤**：
    
    1.  从待排序序列中，找到最小（或最大）元素，将其与序列的第一个元素交换。
        
    2.  从剩余未排序元素中继续寻找最小（或最大）元素，然后将其与未排序序列的第二个元素交换。
        
    3.  重复此过程，直到所有元素都排好序。
        
-   与冒泡对比：
    
    冒泡排序是不断交换相邻元素，将最大（小）的元素“冒泡”到正确位置。
    
    选择排序是每次找到最小（大）元素，直接放到其最终位置，交换次数较少。
    
-   **代码实现：**
    
    C#
    
    ```
    public static void SelectionSort(int[] arr) {
        int n = arr.Length;
    
        for (int i = 0; i < n - 1; i++) {
            // 找到未排序部分的最小元素的索引
            int minIndex = i;
            for (int j = i + 1; j < n; j++) {
                if (arr[j] < arr[minIndex]) {
                    minIndex = j;
                }
            }
    
            // 将最小元素放到当前位置 i
            if (minIndex != i) {
                int temp = arr[i];
                arr[i] = arr[minIndex];
                arr[minIndex] = temp;
            }
        }
    }
    
    ```
    
-   **性能分析：**
    
    -   **时间复杂度**：
        
        -   最坏/平均/最好情况：O(N2) (内层循环总是遍历剩余元素)
            
    -   **空间复杂度**：O(1) (原地排序)
        
    -   **稳定性**：不稳定 (例如：`[5, 8, 5, 2]`，第一个5可能被后面的2换到后面，破坏了相等元素的相对顺序)
        

#### 3. 插入排序（Insertion Sort）

-   原理：
    
    插入排序通过构建有序序列，对于未排序数据，在已排序序列中从后向前扫描，找到相应位置并插入。它模拟了扑克牌玩家整理手牌的过程：每摸到一张新牌，就把它插入到手牌的正确位置。
    
-   **实现步骤**：
    
    1.  将第一个元素视为已排序序列。
        
    2.  从第二个元素开始，依次将每个元素插入到已排序序列的正确位置。
        
    3.  为了给待插入元素腾出位置，需要将已排序序列中比待插入元素大的元素都向后移动一位。
        
-   **适合部分有序数据、小规模数据排序**：
    
    -   如果数据大部分已经有序，插入排序会非常快，接近 O(N)。
        
    -   对于小规模数据，由于其常数因子小，实际性能可能比一些 O(NlogN) 算法更快。
        
-   **代码实现：**
    
    C#
    
    ```
    public static void InsertionSort(int[] arr) {
        int n = arr.Length;
    
        // 从第二个元素开始遍历
        for (int i = 1; i < n; i++) {
            int current = arr[i]; // 待插入的元素
            int j = i - 1;        // 已排序部分的最后一个元素的索引
    
            // 从后向前扫描已排序部分，如果当前元素大于待插入元素，则向后移动
            while (j >= 0 && arr[j] > current) {
                arr[j + 1] = arr[j];
                j--;
            }
            // 找到合适位置，插入待插入元素
            arr[j + 1] = current;
        }
    }
    
    ```
    
-   **性能分析：**
    
    -   **时间复杂度**：
        
        -   最坏情况 (逆序)：O(N2)
            
        -   平均情况：O(N2)
            
        -   最好情况 (已序)：O(N)
            
    -   **空间复杂度**：O(1) (原地排序)
        
    -   **稳定性**：稳定

### O(NlogN) 级别的排序算法：高效与通用

这些算法是面试和实际开发中最常用、最重要的排序算法，因为它们具有较高的效率，适用于大规模数据。

#### 1. 快速排序（Quick Sort）

-   分治思想核心：
    
    快速排序是基于**分治（Divide and Conquer）**思想的算法。其基本思想是：
    
    1.  **选择一个枢轴（Pivot）元素**：从数组中选择一个元素作为基准。
        
    2.  **分区（Partition）**：通过一趟排序将待排序的数据分割成独立的两部分。所有小于枢轴的元素放在枢轴的左边，所有大于枢轴的元素放在枢轴的右边。
        
    3.  **递归排序**：对左边和右边两个子序列分别重复上述过程，直到子序列只有一个元素或为空，即有序。
        
-   枢轴（Pivot）选择策略对性能的影响：
    
    枢轴的选择直接影响快速排序的性能。
    
    -   **最佳情况**：枢轴每次都能将数组均匀地分成两半（例如，选择中位数作为枢轴），此时时间复杂度接近 O(NlogN)。
        
    -   **最坏情况**：枢轴每次都选择到最大或最小的元素（例如，已序或逆序数组中选择第一个元素作为枢轴），导致每次分区都只有一个子数组，时间复杂度退化为 O(N2)。
        
    -   **常用策略**：
        
        -   选择第一个或最后一个元素（最简单，但最容易退化）。
            
        -   选择中间元素。
            
        -   随机选择一个元素。
            
        -   三数取中（Median-of-three）：选择第一个、最后一个和中间位置的元素，取它们的中位数作为枢轴。这种方法能有效避免最坏情况。
            
-   分区（Partition）操作的实现细节：
    
    这是快速排序的核心。目标是将比枢轴小的元素放到左边，大的放到右边。常见的分区方法有：
    
    1.  **Hoare 分区（双指针法）**：左右指针同时向中间移动，找到不符合条件的元素进行交换。
        
    2.  **Lomuto 分区（单指针法）**：通常选择最后一个元素作为枢轴，然后用一个指针遍历，将所有小于枢轴的元素移到左边。
        
-   **平均 O(NlogN)，最坏 O(N2) 分析**：
    
    -   **平均情况**：每次分区都能将问题规模大致减半，所以递归深度为 logN。每层需要遍历所有元素进行分区，所以每层操作是 O(N)。因此总时间复杂度是 O(NlogN)。
        
    -   **最坏情况**：每次分区都严重不平衡，导致递归深度为 N，总时间复杂度退化为 O(N2)。
        
-   原地排序的优势：
    
    快速排序通常是原地排序，只需要 O(logN) 的额外空间（递归栈空间），在空间上非常高效。
    
-   **代码实现（Lomuto 分区）：**
    
    C#
    
    ```
    public static void QuickSort(int[] arr) {
        QuickSort(arr, 0, arr.Length - 1);
    }
    
    private static void QuickSort(int[] arr, int low, int high) {
        if (low < high) {
            // 找到枢轴的最终位置
            int pivotIndex = Partition(arr, low, high);
    
            // 递归地对左右子数组进行排序
            QuickSort(arr, low, pivotIndex - 1);
            QuickSort(arr, pivotIndex + 1, high);
        }
    }
    
    private static int Partition(int[] arr, int low, int high) {
        // 选择最后一个元素作为枢轴
        int pivot = arr[high];
        int i = low - 1; // i 指向小于枢轴的区域的右边界
    
        for (int j = low; j < high; j++) {
            // 如果当前元素小于或等于枢轴，将其放到 i+1 的位置
            if (arr[j] <= pivot) {
                i++;
                Swap(arr, i, j);
            }
        }
        // 将枢轴放到正确的位置
        Swap(arr, i + 1, high);
        return i + 1; // 返回枢轴的最终索引
    }
    
    private static void Swap(int[] arr, int i, int j) {
        int temp = arr[i];
        arr[i] = arr[j];
        arr[j] = temp;
    }
    
    ```
    
-   **性能分析：**
    
    -   **时间复杂度**：
        
        -   最坏情况：O(N2)
            
        -   平均情况：O(NlogN)
            
        -   最好情况：O(NlogN)
            
    -   **空间复杂度**：O(logN) (递归栈空间，最坏 O(N))
        
    -   **稳定性**：不稳定
        

#### 2. 归并排序（Merge Sort）

-   分治思想核心：
    
    归并排序也是基于**分治（Divide and Conquer）**思想。它将一个大问题分解为若干个子问题，然后递归地解决这些子问题，最后将子问题的解合并起来得到原问题的解。
    
    1.  **分解（Divide）**：将待排序序列从中间一分为二，得到两个子序列。
        
    2.  **解决（Conquer）**：递归地对这两个子序列进行排序。
        
    3.  **合并（Combine/Merge）**：将两个已排序的子序列合并成一个完整的有序序列。
        
-   合并（Merge）过程的实现细节：
    
    这是归并排序的核心。
    
    假设我们有两个已经有序的子数组 leftArr 和 rightArr。我们需要将它们合并到一个新的辅助数组中。
    
    1.  设置两个指针，分别指向 `leftArr` 和 `rightArr` 的起始位置。
        
    2.  比较两个指针所指的元素，将较小的元素放入辅助数组，并移动相应指针。
        
    3.  当其中一个子数组的所有元素都被放入辅助数组后，将另一个子数组中剩余的元素全部复制到辅助数组的末尾。
        
    4.  最后，将辅助数组中的内容复制回原数组。
        
-   始终 O(NlogN)，稳定排序的代表：
    
    归并排序的时间复杂度在最坏、平均和最好情况下都是 O(NlogN)。这是因为它的分解和合并过程是固定且可预测的。
    
    它的合并操作可以保证相等元素的相对顺序不变，因此是稳定排序算法。
    
-   需要额外 O(N) 空间：
    
    合并过程需要一个与原数组大小相同的辅助数组来存储合并结果，因此空间复杂度是 O(N)。这是其主要缺点。
    
-   **代码实现：**
    
    C#
    
    ```
    public static void MergeSort(int[] arr) {
        MergeSort(arr, 0, arr.Length - 1);
    }
    
    private static void MergeSort(int[] arr, int left, int right) {
        if (left < right) {
            int mid = left + (right - left) / 2; // 防止溢出
    
            // 递归地对左右两部分进行排序
            MergeSort(arr, left, mid);
            MergeSort(arr, mid + 1, right);
    
            // 合并已排序的两部分
            Merge(arr, left, mid, right);
        }
    }
    
    private static void Merge(int[] arr, int left, int mid, int right) {
        // 创建临时数组存储合并结果
        int[] temp = new int[right - left + 1];
        int i = left;      // 左子数组的起始指针
        int j = mid + 1;   // 右子数组的起始指针
        int k = 0;         // 临时数组的起始指针
    
        // 比较并合并
        while (i <= mid && j <= right) {
            if (arr[i] <= arr[j]) { // 注意这里是 <= 确保稳定性
                temp[k++] = arr[i++];
            } else {
                temp[k++] = arr[j++];
            }
        }
    
        // 复制左子数组剩余元素
        while (i <= mid) {
            temp[k++] = arr[i++];
        }
    
        // 复制右子数组剩余元素
        while (j <= right) {
            temp[k++] = arr[j++];
        }
    
        // 将临时数组中的数据复制回原数组
        for (int p = 0; p < temp.Length; p++) {
            arr[left + p] = temp[p];
        }
    }
    
    ```
    
-   **性能分析：**
    
    -   **时间复杂度**：
        
        -   最坏/平均/最好情况：O(NlogN)
            
    -   **空间复杂度**：O(N) (额外辅助数组)
        
    -   **稳定性**：稳定
        

#### 3. 堆排序（Heap Sort）

-   回顾堆的概念：
    
    在第三篇我们学习了堆（Heap），一种特殊的树形数据结构，通常是一个完全二叉树。它满足堆的性质：
    
    -   **最大堆（Max-Heap）**：父节点的值总是大于或等于其子节点的值。堆顶元素是最大的。
        
    -   最小堆（Min-Heap）：父节点的值总是小于或等于其子节点的值。堆顶元素是最小的。
        
        堆排序利用的是最大堆。
        
-   **建堆（Build Heap）和调整堆（Heapify）操作**：
    
    -   **建堆**：将一个无序数组构造成一个最大堆。通常从第一个非叶子节点开始，对其进行**堆化（Heapify）**操作，然后依次向前对所有非叶子节点进行堆化，直到根节点。
        
    -   **堆化（Heapify）**：当某个节点的值小于其子节点时，将其与最大的子节点交换，并递归地对被交换的子节点进行堆化，直到该节点满足堆的性质。
        
-   利用堆进行排序，原地排序，不稳定：
    
    堆排序的基本思想是：
    
    1.  **建堆**：将待排序数组构造成一个最大堆。此时，堆顶元素（数组第一个元素）就是当前所有元素中的最大值。
        
    2.  取堆顶并调整：
        
        a. 将堆顶元素（最大值）与堆的最后一个元素交换。
        
        b. 此时，最大值已经放到数组的末尾（其最终排序位置），将其从堆中“移除”（逻辑上减小堆的大小）。
        
        c. 对剩余的 N−1 个元素组成的堆进行堆化（从新的堆顶开始），以恢复堆的性质。
        
    3.  重复步骤 2，直到堆的大小为 1。
        
-   **代码实现：**
    
    C#
    
    ```
    public static void HeapSort(int[] arr) {
        int n = arr.Length;
    
        // 1. 建堆 (从最后一个非叶子节点开始向上堆化)
        // 最后一个非叶子节点的索引是 n/2 - 1
        for (int i = n / 2 - 1; i >= 0; i--) {
            Heapify(arr, n, i);
        }
    
        // 2. 排序 (每次将堆顶元素与当前堆的最后一个元素交换，然后调整堆)
        for (int i = n - 1; i > 0; i--) {
            // 将当前最大的元素 (堆顶) 放到数组末尾
            Swap(arr, 0, i);
    
            // 对剩余的 i 个元素组成的堆进行堆化
            Heapify(arr, i, 0); // 注意这里堆的大小是 i
        }
    }
    
    // 堆化函数：确保以 i 为根的子树满足最大堆性质
    // n 是当前堆的实际大小
    private static void Heapify(int[] arr, int n, int i) {
        int largest = i;     // 假定当前根节点是最大值
        int left = 2 * i + 1;  // 左子节点
        int right = 2 * i + 2; // 右子节点
    
        // 如果左子节点存在且大于根节点
        if (left < n && arr[left] > arr[largest]) {
            largest = left;
        }
    
        // 如果右子节点存在且大于目前的最大值
        if (right < n && arr[right] > arr[largest]) {
            largest = right;
        }
    
        // 如果最大值不是根节点，则交换并递归堆化
        if (largest != i) {
            Swap(arr, i, largest);
            Heapify(arr, n, largest);
        }
    }
    
    // 交换函数 (与快速排序共用)
    private static void Swap(int[] arr, int i, int j) {
        int temp = arr[i];
        arr[i] = arr[j];
        arr[j] = temp;
    }
    
    ```
    
-   **性能分析：**
    
    -   **时间复杂度**：
        
        -   建堆：O(N)
            
        -   排序过程 (N-1 次提取和堆化)：N 次堆化操作，每次堆化 O(logN)。
            
        -   总时间复杂度：O(NlogN) (最坏/平均/最好情况)
            
    -   **空间复杂度**：O(1) (原地排序)
        
    -   **稳定性**：不稳定 (因为在交换过程中可能改变相等元素的相对顺序，例如 `[3, 3_prime, 2]`，如果堆化导致 3_prime 和 2 交换，再与堆顶的 3 交换，则 3 和 3_prime 的相对顺序可能改变)

### 非比较排序算法：突破 O(NlogN) 下限

前面介绍的都是**比较排序（Comparison Sort）**，它们通过比较元素来确定相对顺序。比较排序的时间复杂度下限是 O(NlogN)。然而，对于某些特定类型的数据，我们可以利用数据的特性，实现突破这个下限的**非比较排序**算法。

#### 1. 计数排序（Counting Sort）

-   原理与适用场景（整数范围有限）：
    
    计数排序不是基于比较的排序算法，其核心在于统计每个整数在数组中出现的次数，然后根据计数结果按顺序输出。
    
    -   **适用场景**：数据是整数，且数据的**范围 K 不大**时非常高效。
        
    -   **不适用场景**：数据范围非常大时，需要很大的计数数组，造成空间浪费。
        
-   时间 O(N+K)，空间 O(K)：
    
    其中 N 是数组长度，K 是数据范围（最大值 - 最小值 + 1）。
    
-   **代码实现：**
    
    C#
    
    ```
    public static void CountingSort(int[] arr) {
        if (arr == null || arr.Length <= 1) return;
    
        // 找到最大值和最小值，确定计数数组的范围
        int maxVal = arr[0];
        int minVal = arr[0];
        for (int i = 1; i < arr.Length; i++) {
            if (arr[i] > maxVal) maxVal = arr[i];
            if (arr[i] < minVal) minVal = arr[i];
        }
    
        int range = maxVal - minVal + 1;
        int[] count = new int[range]; // 计数数组
        int[] output = new int[arr.Length]; // 存储排序结果的临时数组
    
        // 1. 统计每个元素出现的次数
        for (int i = 0; i < arr.Length; i++) {
            count[arr[i] - minVal]++;
        }
    
        // 2. 累加计数数组，使其表示小于等于当前值的元素数量 (前缀和)
        // 这一步确保了稳定性
        for (int i = 1; i < range; i++) {
            count[i] += count[i - 1];
        }
    
        // 3. 倒序遍历原数组，将元素放到 output 数组的正确位置
        // 倒序遍历是为了保证稳定性
        for (int i = arr.Length - 1; i >= 0; i--) {
            int val = arr[i];
            output[count[val - minVal] - 1] = val; // count[val-minVal] 是 val 应该在的位置的下一个
            count[val - minVal]--; // 对应位置的计数减一
        }
    
        // 4. 将排序结果复制回原数组
        for (int i = 0; i < arr.Length; i++) {
            arr[i] = output[i];
        }
    }
    
    ```
    
-   **性能分析：**
    
    -   **时间复杂度**：O(N+K)
        
    -   **空间复杂度**：O(N+K) (需要 `count` 数组和 `output` 数组)
        
    -   **稳定性**：稳定 (通过倒序遍历原数组和累加计数数组实现)
        

#### 2. 桶排序（Bucket Sort）

-   原理与适用场景（数据分布均匀）：
    
    桶排序是计数排序的升级版，它将待排序的元素分散到有限数量的“桶”中，每个桶再单独进行排序（可以递归使用桶排序，或使用其他排序算法，如插入排序），最后按顺序合并所有桶中的元素。
    
    -   **适用场景**：数据是浮点数或整数，并且数据在一定范围内**均匀分布**时，效率最高。
        
    -   **不适用场景**：数据分布不均匀时，可能导致大量数据集中在一个桶中，退化为其他排序算法。
        
-   **代码实现思路（简化）：**
    
    1.  创建若干个桶（例如，使用 `List<int>` 的 `List`）。
        
    2.  遍历原数组，将每个元素根据其值放入对应的桶中。
        
    3.  对每个非空的桶进行排序（例如，使用插入排序）。
        
    4.  按顺序遍历所有桶，将桶中的元素依次取出，放回原数组。
        
-   **性能分析：**
    
    -   **时间复杂度**：
        
        -   最好/平均情况：O(N+K) (K 是桶的数量，如果每个桶的元素数量常数，且桶内排序是 O(1))
            
        -   最坏情况：O(N2) (所有元素都在一个桶中)
            
    -   **空间复杂度**：O(N+K)
        
    -   **稳定性**：取决于桶内使用的排序算法是否稳定。如果桶内使用稳定排序，则整体稳定。
        

#### 3. 基数排序（Radix Sort）

-   原理与适用场景（多位数排序）：
    
    基数排序是一种非比较的整数排序算法，其原理是将整数按位数切割成不同的数字，然后按每个位数分别比较。它不直接比较元素值，而是根据它们的**“位”**来排序。
    
    -   **适用场景**：适用于整数，尤其是位数相同或位数范围不大的整数。
        
    -   通常结合计数排序或桶排序来对每一位进行排序。
        
-   **实现步骤（以LSD - Least Significant Digit 为例，从最低位开始排序）**：
    
    1.  找出数组中的最大值，确定最大位数 `d`。
        
    2.  从最低位（个位）开始，对所有数字进行一次稳定的排序（通常使用计数排序）。
        
    3.  然后对十位进行排序。
        
    4.  重复此过程，直到最高位。
        
-   **代码实现（基于计数排序）：**
    
    C#
    
    ```
    public static void RadixSort(int[] arr) {
        if (arr == null || arr.Length <= 1) return;
    
        // 找到最大值，确定需要处理的位数
        int maxVal = arr[0];
        for (int i = 1; i < arr.Length; i++) {
            if (arr[i] > maxVal) maxVal = arr[i];
        }
    
        // 从个位开始，依次对每一位进行计数排序
        for (int exp = 1; maxVal / exp > 0; exp *= 10) { // exp 代表当前位数（1, 10, 100...）
            CountingSortByDigit(arr, exp);
        }
    }
    
    // 辅助函数：根据指定位数（exp）进行计数排序
    private static void CountingSortByDigit(int[] arr, int exp) {
        int n = arr.Length;
        int[] output = new int[n];
        int[] count = new int[10]; // 0-9 数字的计数
    
        // 1. 统计当前位上每个数字出现的次数
        for (int i = 0; i < n; i++) {
            count[(arr[i] / exp) % 10]++;
        }
    
        // 2. 累加计数数组，使其表示小于等于当前数字的元素数量
        for (int i = 1; i < 10; i++) {
            count[i] += count[i - 1];
        }
    
        // 3. 倒序遍历原数组，将元素放到 output 数组的正确位置
        // 倒序保证稳定性
        for (int i = n - 1; i >= 0; i--) {
            output[count[(arr[i] / exp) % 10] - 1] = arr[i];
            count[(arr[i] / exp) % 10]--;
        }
    
        // 4. 将排序结果复制回原数组
        for (int i = 0; i < n; i++) {
            arr[i] = output[i];
        }
    }
    
    ```
    
-   **性能分析：**
    
    -   **时间复杂度**：O(Dcdot(N+K))，其中 D 是最大位数，K 是基数（例如十进制是 10）。通常可简化为 O(DN) 或 O(NcdotK)。
        
    -   **空间复杂度**：O(N+K)
        
    -   **稳定性**：稳定 (因为它内部使用了稳定的计数排序)

#### 面试中如何选择合适的排序算法

在面试中，面对排序问题，你不能只是“背诵”算法，更要展示你根据场景选择最佳方案的能力：

1.  **明确数据规模 N**：
    
    -   **小规模数据** (N 较小，如几百上千)：O(N2) 算法（如插入排序）因为常数因子小，可能比 O(NlogN) 算法更快，且代码简单。
        
    -   **大规模数据** (N 较大)：优先考虑 O(NlogN) 算法。
        
    -   **海量数据（内存放不下）**：考虑**外排序**（External Sort），归并排序是外排序的经典算法。
        
2.  **考虑数据特性**：
    
    -   **接近有序**：**插入排序**效率极高，接近 O(N)。
        
    -   **整数且范围有限**：考虑**计数排序、基数排序**，O(N+K) 甚至 O(N) 级别。
        
    -   **浮点数且均匀分布**：考虑**桶排序**。
        
    -   **存在大量重复元素**：有些排序算法（如三路快排）对重复元素处理有优化。
        
3.  **对空间复杂度的要求**：
    
    -   **内存非常有限**：优先选择**原地排序**算法（冒泡、选择、插入、快排、堆排），它们的空间复杂度是 O(1) 或 O(logN)。
        
    -   **空间不是主要瓶颈**：归并排序的 O(N) 空间是可接受的。
        
4.  **对稳定性的要求**：
    
    -   如果数据中存在相同元素，且这些相同元素的**相对顺序很重要**（例如，先按姓名排序，再按分数排序，如果分数相同，希望姓名顺序不变），则必须选择**稳定排序**算法（冒泡、插入、归并、计数、桶、基数）。
        
    -   如果相对顺序不重要，则不稳定排序也可以。
        
5.  **手写代码要求**：
    
    -   面试中经常要求手写**快速排序**和**归并排序**，务必熟练其核心原理和实现细节。
        

#### Unity 中的排序：`Array.Sort`, `List.Sort` 的底层实现

在 Unity (C#) 中，我们通常直接使用内置的排序方法，而无需自己实现各种排序算法：

-   `System.Array.Sort()`：用于对数组进行排序。底层实现是**混合排序算法**：
    
    -   对于小规模数组，使用**插入排序**。
        
    -   对于大规模数组，使用**快速排序**。
        
    -   这种混合策略结合了两种算法的优点：插入排序在小数组上效率高，快速排序在大数组上平均性能好。
        
    -   C# 7.0 之前，对于非原始类型（如自定义对象），`Array.Sort` 会使用**堆排序**以保证最坏情况的 O(NlogN) 性能。从 .NET Core 3.0 (C# 7.0) 开始，`Array.Sort` 的默认实现对于引用类型也主要采用**Introspective Sort（内省排序）**，它是一种混合排序算法，结合了快速排序、堆排序和插入排序。它首先使用快速排序，当递归深度过大时（可能导致 O(N2)），切换到堆排序；当子数组很小时，切换到插入排序。这确保了在平均和最坏情况下的 O(NlogN) 性能，同时保留了快速排序的平均优势。
        
-   `System.Collections.Generic.List<T>.Sort()`：用于对 `List` 集合进行排序。其底层实现与 `Array.Sort()` 类似，也是基于**混合排序**。
    

这些内置方法通常是经过高度优化和测试的，是你在实际 Unity 项目中进行排序的首选。理解它们背后的算法原理，能帮助你在遇到性能瓶颈时更好地分析和优化。
