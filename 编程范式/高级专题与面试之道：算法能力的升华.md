### 二分查找（Binary Search）进阶

我们最早在数组基础中提到了二分查找，它是一种高效的搜索算法，时间复杂度为 O(logN)。但它的应用远不止于在一个有序数组中查找某个特定值。

#### 回顾基础二分查找

在一个有序数组中查找一个目标值 `target`：

C#

```
public int BinarySearchBasic(int[] nums, int target) {
    int left = 0;
    int right = nums.Length - 1;

    while (left <= right) { // 注意这里是 <=
        int mid = left + (right - left) / 2;
        if (nums[mid] == target) {
            return mid;
        } else if (nums[mid] < target) {
            left = mid + 1;
        } else {
            right = mid - 1;
        }
    }
    return -1; // 未找到
}

```

#### 变体：更复杂的查找需求

实际问题中，往往不只是简单查找是否存在，还可能需要查找特定条件的边界：

1.  查找第一个等于某值的元素：
    
    当数组中有重复元素时，如何找到目标值的第一个出现位置？
    
    -   如果 `nums[mid] == target`，我们记录当前 `mid` 为一个可能的结果，然后继续向左侧查找，看是否有更小的索引。`ans = mid; right = mid - 1;`
        
2.  查找最后一个等于某值的元素：
    
    类似地，如果 nums[mid] == target，记录 mid，然后继续向右侧查找。ans = mid; left = mid + 1;
    
3.  查找第一个大于等于某值的元素（lower_bound）：
    
    如果 nums[mid] >= target，mid 是一个可能答案，继续向左找更小的可能值。ans = mid; right = mid - 1;
    
4.  查找第一个大于某值的元素（upper_bound）：
    
    如果 nums[mid] > target，mid 是一个可能答案，继续向左找更小的可能值。ans = mid; right = mid - 1;
    
    如果 nums[mid] <= target，说明 mid 不符合要求，需要向右找。left = mid + 1;
    

#### 在非传统数据结构中的应用

二分查找的核心是**单调性**。只要问题可以通过某种方式转化为在单调区间内寻找满足条件的点，就可以考虑二分查找。

-   旋转排序数组查找（Search in Rotated Sorted Array）：LeetCode 33
    
    一个原本有序的数组在某个未知点进行了旋转。这时数组不再整体有序，但被旋转点分成了两个有序的子数组。我们可以通过判断 nums[mid] 与 nums[left] 的关系来确定 mid 位于哪个有序子数组中，从而缩小搜索范围。
    
-   寻找峰值（Find Peak Element）：LeetCode 162
    
    寻找数组中比左右邻居都大的元素。因为题目保证至少存在一个峰值，而且如果 nums[i] < nums[i+1]，那么峰值一定在 i+1 及其右侧（因为右侧总有一个比左侧大的元素，直到遇到峰值），否则在左侧。这体现了局部趋势的单调性。
    
-   寻找最小值（Find Minimum in Rotated Sorted Array）：LeetCode 153
    
    类似旋转排序数组查找，通过比较 mid 与 right 的值来决定最小值在哪一边。

### 位运算（Bit Manipulation）

位运算是直接对数字的二进制位进行操作，效率极高。在某些特定场景下，利用位运算可以写出非常简洁和高效的代码。

#### 基础位运算符

-   `&` (按位与)：两位都为 1 时才为 1。
    
-   `|` (按位或)：两位都为 0 时才为 0。
    
-   `^` (按位异或)：两位不同时为 1。**异或的重要性质：`a ^ a = 0`，`a ^ 0 = a`。**
    
-   `~` (按位非/取反)：将每一位取反。
    
-   `<<` (左移)：向左移动指定位数，右侧补 0，相当于乘以 2n。
    
-   `>>` (右移)：向右移动指定位数，左侧补 0（逻辑右移）或补符号位（算术右移，取决于编译器/语言）。相当于除以 2n。
    

#### 应用场景

-   **判断奇偶**：`num & 1 == 1` 为奇数，`num & 1 == 0` 为偶数。比 `num % 2` 更快。
    
-   **交换数字**：`a = a ^ b; b = a ^ b; a = a ^ b;` 无需额外变量。
    
-   **快速乘除 2 的幂**：`num << n` 相当于 `num * (2^n)`；`num >> n` 相当于 `num / (2^n)`。
    
-   **清零特定位**：`num & (~(1 << k))` 清零第 `k` 位。
    
-   **置位特定位**：`num | (1 << k)` 将第 `k` 位置为 1。
    
-   **判断特定位是否为 1**：`(num >> k) & 1 == 1`。
    
-   **反转特定位**：`num ^ (1 << k)` 反转第 `k` 位。
    

#### 经典问题

-   只出现一次的数字（Single Number）：LeetCode 136
    
    给定一个非空整数数组，除了某个元素只出现一次以外，其余每个元素均出现两次。找出那个只出现了一次的元素。
    
    解法：利用异或的性质，所有数字异或一遍，重复的数字会抵消（a ^ a = 0），最终结果就是那个只出现一次的数字。
    
-   判断 2 的幂（Power of Two）：LeetCode 231
    
    如果一个正整数是 2 的幂，那么它的二进制表示中只有一位是 1。例如 8 (1000)，16 (10000)。
    
    解法：n > 0 && (n & (n - 1)) == 0。n - 1 会将 n 中最低位的 1 变为 0，并将更低的 0 变为 1。例如 8 (1000) & 7 (0111) = 0。
    
-   数字的二进制表示中 1 的个数（Number of 1 Bits / Hamming Weight）：LeetCode 191
    
    解法：循环 n = n & (n - 1) 直到 n 变为 0。每次操作都会清除 n 中最低位的 1。这种方法是统计 1 的个数最快的方法之一。

### 算法思维与模式识别

学习算法不仅仅是记住解法，更重要的是形成一种**解题思维**和**模式识别能力**。很多问题看似不同，但底层却共享相同的模式。

-   **双指针模式的再总结与分类**：
    
    -   **对撞指针**（Two Pointers - Converging）：一个从头，一个从尾，向中间移动。常用于查找特定和、判断回文、原地操作数组等。
        
        -   例：`Valid Palindrome`，`Two Sum II - Input array is sorted`
            
    -   **快慢指针**（Two Pointers - Fast & Slow）：一个移动快，一个移动慢。常用于链表中的环检测、寻找中点等。
        
        -   例：`Linked List Cycle`，`Middle of the Linked List`
            
    -   **同向滑动窗口**（Two Pointers - Sliding Window）：两个指针同向移动，维护一个“窗口”，窗口的大小或内容动态变化。常用于查找子数组/子串的最大/最小和、特定条件的子数组/子串等。
        
        -   例：`Minimum Window Substring`，`Longest Substring Without Repeating Characters`
            
-   **DFS/BFS 模式的再总结**：
    
    -   **何时用 DFS (深度优先搜索)**：
        
        -   需要遍历所有可能的路径或组合。
            
        -   需要进行回溯（如排列、组合、N 皇后）。
            
        -   判断连通性、寻找路径、环检测。
            
        -   **特点**：使用递归（系统栈）或显式栈，天然适用于回溯，占用空间相对较少（取决于深度）。
            
    -   **何时用 BFS (广度优先搜索)**：
        
        -   寻找最短路径（无权图）。
            
        -   层级遍历（如树的层序遍历）。
            
        -   解决扩散问题（如腐烂的橘子）。
            
        -   **特点**：使用队列，保证先访问近的节点，再访问远的节点，占用空间可能较大（队列中可能存储大量节点）。
            
    -   **如何处理访问状态**：在图和树的遍历中，为了避免重复访问和死循环，通常需要一个 `visited` 集合或布尔数组来记录已访问过的节点。
        
-   **递归与迭代的转换**：
    
    -   **递归**：代码简洁，符合直觉（分治思想）。但有栈溢出风险，调试可能复杂。
        
    -   **迭代**：性能通常更好，没有栈溢出风险。但代码可能更复杂，需要手动管理栈或队列。
        
    -   **转换思路**：任何递归都可以用迭代实现。通常，递归的参数会成为迭代栈（或队列）中的元素；递归的结束条件会成为循环的终止条件；递归调用会变为入栈（或入队）操作。
        
-   **空间换时间/时间换空间的策略权衡**：
    
    -   在内存允许的情况下，我们常常牺牲空间来提高时间效率（例如 DP 表、哈希表）。
        
    -   当内存受限时，我们会考虑使用滚动数组等空间优化技巧，但可能牺牲部分时间效率或增加代码复杂度。
        
    -   面试中，当面试官问到优化时，这两者是常见的权衡点。

### 面试准备策略与技巧

算法面试是考察你综合能力的重要环节。不仅仅是写出正确的代码，更重要的是你的**解决问题流程**和**沟通方式**。

1.  **如何阅读和理解题目**：
    
    -   **明确输入**：数据类型、范围、是否为空、是否有序、是否有重复。
        
    -   **明确输出**：数据类型、格式要求、特殊情况（如无解）。
        
    -   **明确约束**：时间/空间复杂度要求。
        
    -   **边界条件**：`N=0, 1`，最大值、最小值，空数组/链表等。
        
    -   **不要急于写代码**：花 1-2 分钟把题目读清楚，问清所有不确定的地方。
        
2.  **如何与面试官沟通思路**：
    
    -   **澄清问题**：如果题目模糊，主动提问，确保理解一致。
        
    -   **给出暴力解**：首先提出一个能解决问题的简单但可能低效的暴力解。这表明你理解问题，并且能提供一个基准。
        
    -   **逐步优化，解释优化思路**：
        
        -   “我们可以观察到这里有重复计算，所以可以考虑使用**备忘录**或**DP 表**来优化。”
            
        -   “这个问题似乎有**单调性**，或许可以用**二分查找**。”
            
        -   “这是一个**组合**问题，我们可以用**回溯法**来尝试所有可能，并加入**剪枝**。”
            
        -   “似乎每一步的局部最优选择都能导出全局最优，我们可以尝试**贪心算法**。”
            
    -   **分析复杂度**：每一步优化后，都要分析其**时间复杂度**和**空间复杂度**，并向面试官解释为什么新的方案更优。
        
    -   **讨论边界条件**：在讨论算法时，就主动提及对空输入、单元素、最大值、最小值等边界情况的处理。
        
3.  **编写高质量代码**：
    
    -   **命名规范**：变量、函数名清晰易懂（`left`, `right`, `mid`, `dp`, `visited`）。
        
    -   **代码结构**：逻辑清晰，函数职责单一。
        
    -   **可读性**：适当的注释（解释关键逻辑或复杂步骤）、空行。
        
    -   **防御式编程**：检查输入参数是否合法（如 `null` 或空数组）。
        
4.  **测试用例**：
    
    -   **自己构造测试用例**：至少一个通用用例，一个或多个边界用例（`N=0, 1`，最大值，最小值），以及特殊用例（重复元素，负数等）。
        
    -   **跑一遍自己的代码**：在纸上或脑海中模拟代码执行过程，看是否符合预期。
        
5.  **调试技巧**：
    
    -   **打印中间变量**：在关键步骤打印变量值，帮助理解程序流程。
        
    -   **分治调试**：将大问题拆成小函数，逐个调试。
        
    -   **脑海模拟**：最重要且最常用的调试方式，训练自己“人肉编译器”的能力。
        
6.  **心理建设**：
    
    -   **保持冷静**：遇到难题不要慌张，深呼吸，分步思考。
        
    -   **不懂就问**：如果对题目有疑问，或者在某个地方卡壳，不要害怕寻求面试官的提示。面试官更看重你的**解决问题能力和沟通能力**，而不是你是否能独立完美地解决所有问题。
        
    -   **承认错误**：如果发现思路有误，及时承认并纠正，而不是固执己见。

### Unity 开发中的算法思维

作为 Unity 开发者，算法知识并非只存在于面试中，它在实际项目中有着广泛而深远的运用：

-   寻路算法（A*）：
    
    作为图算法的实际应用，A* 算法是游戏中最常用的寻路算法。它结合了 Dijkstra（寻找最短路径）和 BFS（优先探索近距离）的优点，通过启发式函数（Heuristic Function）来估算从当前节点到目标节点的距离，从而高效地找到最优路径。在 Unity 中，你可以结合 NavMesh 或自己构建网格图来实现 A* 寻路。
    
-   空间划分数据结构：
    
    为了优化性能，尤其是碰撞检测、射线检测、渲染剔除等，我们会用到各种空间划分数据结构：
    
    -   **四叉树（Quadtree）**：用于 2D 空间，将空间递归地划分为四个象限，存储其中的对象。常用于 2D 游戏、地图瓦片管理等。
        
    -   **八叉树（Octree）**：用于 3D 空间，将空间递归地划分为八个八分体，存储其中的对象。常用于 3D 游戏中的场景管理、LOD (Level of Detail) 系统、可见性检测等。
        
    -   这些数据结构本质上是**树**和**图**的应用，利用树的层级结构和图的连接性来加速空间查询。
        
-   **物理模拟、粒子系统中的算法思想**：
    
    -   **物理模拟**：碰撞检测中的几何算法、力学计算中的积分算法、关节约束算法等。
        
    -   **粒子系统**：粒子生成、移动、消亡的数学模型和随机算法；粒子碰撞检测和响应。
        
    -   **LOD (Level of Detail)**：根据距离和重要性，对模型、纹理、粒子等进行不同精度的渲染，这背后也是各种优化算法在支撑。
        
-   **游戏 AI**：除了寻路，AI 的决策树、状态机、行为树等，都体现了图论和树结构的思想。
    
-   **数据结构选择**：在 Unity 中，选择 `List`、`Dictionary`、`HashSet`、`Queue`、`Stack` 等，都依赖于你对这些数据结构底层实现和时间/空间复杂度的理解。例如，频繁的增删改查场景，选择 `Dictionary` 或 `HashSet` 而非 `List`。

### 未来学习建议

算法学习是一个持续的过程，永无止境。

-   **持续刷题的重要性**：
    
    -   “刷题不是目的，而是手段”。通过大量的练习，你会逐渐形成模式识别能力，将新问题与已知模式联系起来。
        
    -   推荐平台：LeetCode、LintCode、Codeforces 等。
        
    -   建议按专题刷题，而不是漫无目的地乱刷。
        
-   **深入学习其他高级算法**：
    
    -   **字符串匹配算法**：如 KMP、Boyer-Moore，在文本处理、字符串搜索中有广泛应用。
        
    -   **网络流**：解决最大流、最小割等问题，常用于匹配、分配等优化问题。
        
    -   **计算几何**：点线面关系、凸包、最近点对等，在图形学和游戏开发中有用。
        
    -   **概率和随机算法**：蒙特卡洛方法、洗牌算法等。
        
-   **从项目实践中发现和解决算法问题**：
    
    -   理论学习最终要回归实践。在 Unity 项目开发中，当你遇到性能瓶颈、逻辑复杂时，主动思考是否能用算法来优化。
        
    -   例如，某个 UI 元素的刷新导致卡顿，是不是数据结构选错了？某个 AI 寻路不合理，是不是 A* 启发式函数没调好？
        
-   **阅读优秀开源代码和经典书籍**：
    
    -   阅读经典算法书籍（如《算法导论》），加深理论理解。
        
    -   分析开源项目中对算法和数据结构的运用。
