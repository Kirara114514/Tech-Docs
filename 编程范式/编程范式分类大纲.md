## 算法系列教程：从入门到精通，直击面试核心

### 第一篇：算法之魂：复杂度分析与数组基础

-   **主题：** 奠定算法基石，理解代码效率，掌握数组常见操作。
    
-   **核心目标：**
    
    -   彻底理解**时间复杂度**和**空间复杂度**，并能分析常见代码的复杂度。
        
    -   掌握数组作为最基本数据结构的操作技巧和经典应用。
        
    -   引入**双指针**和**哈希表**的初步应用思想。
        
-   **大纲与主要内容：**
    
    1.  **引言：为什么需要学习算法？**
        
        -   算法在程序开发中的重要性。
            
        -   面试官为何青睐算法题？考察的是什么？
            
        -   本系列教程的目标与学习方法。
            
    2.  **时间复杂度（Time Complexity）**
        
        -   概念：衡量算法执行时间随输入规模增长的趋势。
            
        -   **大 O 标记法（Big O Notation）**：定义、作用。
            
        -   常见时间复杂度详解：O(1),O(logn),O(n),O(nlogn),O(n2),O(2n),O(n!)。
            
        -   如何分析代码的时间复杂度：循环、递归、多层循环、函数调用。
            
        -   最佳、最坏、平均时间复杂度。
            
    3.  **空间复杂度（Space Complexity）**
        
        -   概念：衡量算法所需额外存储空间随输入规模增长的趋势。
            
        -   如何分析代码的空间复杂度：变量、数组、递归栈等。
            
    4.  **数组（Array）**
        
        -   特性：连续存储、随机访问 O(1)、插入删除效率低。
            
        -   基本操作：创建、访问、遍历、查找、插入、删除。
            
        -   **经典面试题与解法：**
            
            -   **两数之和（Two Sum）**：
                
                -   暴力法 (O(n2))。
                    
                -   **哈希表优化** (O(n) 时间，O(n) 空间)，理解**空间换时间**思想。
                    
            -   **移除元素（Remove Element）/移除重复项（Remove Duplicates from Sorted Array）**：
                
                -   **快慢指针法**：原地修改数组的关键技巧，详细讲解其原理和应用。
                    
            -   **旋转数组（Rotate Array）**：
                
                -   多种解法：额外数组、环状替换、**三次反转法**（原地操作的优雅解法）。
                    
            -   **合并两个有序数组（Merge Sorted Array）**：
                
                -   从后往前合并的技巧，避免数据覆盖。
                    
    5.  **总结与练习题**
        
        -   本篇核心知识点回顾。
            
        -   精选练习题推荐（提供力扣等平台的题目编号）。

### 第二篇：链表的艺术：结构、操作与指针魔法

-   **主题：** 深入理解链表这一重要非连续存储结构，掌握其核心操作和指针运用技巧。
    
-   **核心目标：**
    
    -   熟练掌握单链表、双链表的定义和基本操作。
        
    -   精通**链表反转**这一核心技巧，及其在其他问题中的变种应用。
        
    -   灵活运用**快慢指针**解决链表中的特定问题。
        
-   **大纲与主要内容：**
    
    1.  **引言：链表为何如此重要？**
        
        -   链表与数组的对比：优劣势分析。
            
        -   链表在实际应用中的场景（如 LRU 缓存、操作系统文件系统）。
            
    2.  **链表基础**
        
        -   **节点（Node）**定义：值 (val) 和指向下一个节点 (next) 的指针。
            
        -   **单链表（Singly Linked List）**：结构、头结点、尾结点、虚拟头结点（哨兵节点）的妙用。
            
        -   **双链表（Doubly Linked List）**：额外 `prev` 指针的优势（方便回溯）。
            
        -   基本操作：
            
            -   **创建与初始化**。
                
            -   **遍历**：迭代与递归。
                
            -   **查找**：按值查找、按索引查找。
                
            -   **插入**：头插法、尾插法、中间插入。
                
            -   **删除**：删除指定节点、删除指定值节点。
                
    3.  **经典面试题与解法：**
        
        -   **反转链表（Reverse Linked List）**：
            
            -   **迭代法**（三指针法：`prev`, `curr`, `next_node`）详细推导与图解。
                
            -   **递归法**（理解递归的精髓，从局部到整体）。
                
            -   延伸：反转链表前N个节点。
                
        -   **合并两个有序链表（Merge Two Sorted Lists）**：
            
            -   **迭代法**（双指针，哨兵节点）。
                
            -   **递归法**。
                
        -   **删除链表的倒数第 N 个结点（Remove Nth Node From End of List）**：
            
            -   **快慢指针法**：快指针先走 N 步，然后快慢指针同速前进。
                
        -   **环形链表（Linked List Cycle）**：
            
            -   **快慢指针法（Floyd's Cycle-Finding Algorithm）**：判断是否存在环，并找出入环点。
                
        -   **链表相交（Intersection of Two Linked Lists）**：
            
            -   双指针法：走完自己的路，再走对方的路。
                
    4.  **总结与练习题**

### 第三篇：高效容器：栈、队列与堆

-   **主题：** 掌握特定操作限制下的数据结构，理解其应用场景和内部实现。
    
-   **核心目标：**
    
    -   理解**栈**（LIFO）和**队列**（FIFO）的特性与典型应用。
        
    -   掌握**单调栈**和**优先队列（堆）**这两种进阶数据结构的使用。
        
    -   能够选择合适的容器解决问题。
        
-   **大纲与主要内容：**
    
    1.  **引言：容器的哲学**
        
        -   为何需要特定存取模式的数据结构？
            
        -   实际开发中的应用（例如：Unity 中的 Command Pattern）。
            
    2.  **栈（Stack）**
        
        -   概念：后进先出（LIFO）。
            
        -   基本操作：`push`, `pop`, `peek`, `isEmpty`。
            
        -   实现方式：数组实现、链表实现。
            
        -   **应用场景：** 函数调用栈、表达式求值、括号匹配、浏览器前进后退。
            
        -   **经典面试题与解法：**
            
            -   **有效括号（Valid Parentheses）**：栈的典型应用，匹配括号。
                
            -   **最小栈（Min Stack）**：设计一个支持常数时间获取最小值的栈。
                
            -   **每日温度（Daily Temperatures）/下一个更大元素（Next Greater Element）**：
                
                -   **单调栈**：保持栈内元素单调性，高效解决“下一个更大/更小”问题。
                    
    3.  **队列（Queue）**
        
        -   概念：先进先出（FIFO）。
            
        -   基本操作：`enqueue`, `dequeue`, `front`, `isEmpty`。
            
        -   实现方式：数组实现、链表实现。
            
        -   **应用场景：** 广度优先搜索（BFS）、任务调度、消息队列。
            
        -   **经典面试题与解法：**
            
            -   **用栈实现队列（Implement Queue using Stacks）**：理解栈和队列的特性互补。
                
            -   **用队列实现栈（Implement Stack using Queues）**：加深理解。
                
            -   **循环队列（Circular Queue）**：数组实现的精妙之处。
                
    4.  **堆（Heap）/优先队列（Priority Queue）**
        
        -   概念：一种特殊的完全二叉树，满足堆序性（父节点大于/小于子节点）。
            
        -   分类：最大堆、最小堆。
            
        -   **基本操作：** 插入、删除最大/最小元素、O(logn) 复杂度。
            
        -   应用场景：Top K 问题、Dijkstra 算法、事件调度。
            
        -   **经典面试题与解法：**
            
            -   **前 K 个高频元素（Top K Frequent Elements）**：**小顶堆**的高效应用。
                
            -   **数据流中的中位数（Find Median from Data Stream）**：对顶堆的应用。
                
    5.  **总结与练习题**
        
        -   本篇核心知识点回顾。
            
        -   精选练习题推荐。

### 第四篇：哈希表与字符串：高效查找与文本处理

-   **主题：** 掌握哈希表的高效查找能力，以及字符串的常见操作和技巧。
    
-   **核心目标：**
    
    -   理解**哈希表**的内部原理，掌握其在 O(1) 平均时间复杂度查找上的优势。
        
    -   熟练运用哈希表解决查找、计数、去重等问题。
        
    -   掌握字符串的**双指针**和**滑动窗口**技巧。
        
-   **大纲与主要内容：**
    
    1.  **引言：哈希表的魔力**
        
        -   哈希表的出现解决了什么问题？为什么它如此高效？
            
    2.  **哈希表（Hash Table/HashMap/HashSet）**
        
        -   原理：哈希函数、哈希冲突、冲突解决策略（链地址法、开放寻址法）。
            
        -   优点：平均 O(1) 的查找、插入、删除。
            
        -   缺点：空间开销、最坏 O(n) 情况。
            
        -   哈希集合（HashSet）与哈希映射（HashMap）的区别与联系。
            
        -   **适用场景**：快速查找、去重、计数、缓存。
            
    3.  **经典面试题与解法（哈希表）**：
        
        -   **两数之和（Two Sum）**（再强调，作为哈希表入门）。
            
        -   **字母异位词分组（Group Anagrams）**：通过排序或字符计数作为哈希表的键。
            
        -   **无重复字符的最长子串（Longest Substring Without Repeating Characters）**：
            
            -   **滑动窗口**结合哈希表，详细讲解窗口的扩张与收缩。
                
        -   **最长连续序列（Longest Consecutive Sequence）**：哈希集合优化查找。
            
        -   **多数元素（Majority Element）**：摩尔投票法（拓展）与哈希表计数。
            
    4.  **字符串（String）**
        
        -   特性：不可变性（Immutable）的概念，常见操作的复杂度。
            
        -   **字符串常用操作**：拼接、查找、替换、分割。
            
        -   **字符串与双指针**：
            
            -   **反转字符串（Reverse String）**：原地反转。
                
            -   **验证回文串（Valid Palindrome）**：过滤非字母数字字符后双指针判断。
                
            -   **最长回文子串（Longest Palindromic Substring）**：中心扩散法。
                
        -   **字符串与滑动窗口**（复习与深化）：
            
            -   **找到字符串中所有字母异位词（Find All Anagrams in a String）**。
                
            -   **最小覆盖子串（Minimum Window Substring）**（进阶，体现滑动窗口的复杂边界）。
                
    5.  **总结与练习题**
        
        -   本篇核心知识点回顾。

### 第五篇：排序算法：理论与实践的艺术

-   **主题：** 全面掌握各种经典排序算法的原理、实现、性能分析及适用场景。
    
-   **核心目标：**
    
    -   能够手写常见排序算法（快速排序、归并排序等）。
        
    -   理解不同排序算法的**时间复杂度、空间复杂度**和**稳定性**。
        
    -   面试中能根据场景选择最合适的排序算法。
        
-   **大纲与主要内容：**
    
    1.  **引言：排序无处不在**
        
        -   为何排序如此基础且重要？实际开发中的排序需求。
            
        -   评价排序算法的指标：时间、空间、稳定性。
            
    2.  **O(N^2) 级别的排序算法：**
        
        -   **冒泡排序（Bubble Sort）**：原理、优化（记录是否发生交换）。
            
        -   **选择排序（Selection Sort）**：原理、与冒泡对比。
            
        -   **插入排序（Insertion Sort）**：原理、适合部分有序数据、小规模数据排序。
            
    3.  **O(N log N) 级别的排序算法：**
        
        -   **快速排序（Quick Sort）**：
            
            -   **分治思想**核心。
                
            -   **枢轴（Pivot）选择**策略对性能的影响。
                
            -   分区（Partition）操作的实现细节。
                
            -   平均 O(NlogN)，最坏 O(N2) 分析。
                
            -   **原地排序**的优势。
                
        -   **归并排序（Merge Sort）**：
            
            -   **分治思想**核心。
                
            -   **合并（Merge）过程**的实现细节。
                
            -   始终 O(NlogN)，**稳定排序**的代表。
                
            -   需要额外 O(N) 空间。
                
        -   **堆排序（Heap Sort）**：
            
            -   回顾堆的概念。
                
            -   **建堆（Build Heap）**和**调整堆（Heapify）**操作。
                
            -   利用堆进行排序，原地排序，不稳定。
                
    4.  **非比较排序算法：**
        
        -   **计数排序（Counting Sort）**：
            
            -   原理与适用场景（整数范围有限）。
                
            -   时间 O(N+K)，空间 O(K)。
                
        -   **桶排序（Bucket Sort）**：
            
            -   原理与适用场景（数据分布均匀）。
                
        -   **基数排序（Radix Sort）**：
            
            -   原理与适用场景（多位数排序）。
                
    5.  **排序算法总结与面试策略**
        
        -   各算法性能对比（时间、空间、稳定性）。
            
        -   面试中如何选择合适的排序算法。
            
        -   Unity 中的排序：`Array.Sort`, `List.Sort` 的底层实现。
            
    6.  **总结与练习题**
        
        -   本篇核心知识点回顾。

### 第六篇：树与二叉树：非线性结构之美

-   **主题：** 掌握树的基本概念，重点理解二叉树的遍历、性质和特殊类型。
    
-   **核心目标：**
    
    -   熟练进行二叉树的深度优先遍历（前中后序）和广度优先遍历（层序遍历）。
        
    -   理解**二叉搜索树（BST）**的特性及其应用。
        
    -   能解决二叉树的常见操作和性质判断问题。
        
-   **大纲与主要内容：**
    
    1.  **引言：树形结构的意义**
        
        -   为何引入非线性结构？树在现实世界中的映射。
            
        -   文件系统、DOM 树、游戏场景中的物体层级。
            
    2.  **树的基础概念**
        
        -   定义：根、节点、父节点、子节点、兄弟节点、叶子节点。
            
        -   度、深度、高度、层。
            
    3.  **二叉树（Binary Tree）**
        
        -   定义：每个节点最多有两个子节点。
            
        -   特殊类型：满二叉树、完全二叉树、平衡二叉树。
            
        -   **二叉树遍历：**
            
            -   **深度优先搜索（DFS）**：
                
                -   **前序遍历（Preorder Traversal）**：根-左-右（递归与迭代）。
                    
                -   **中序遍历（Inorder Traversal）**：左-根-右（递归与迭代）。
                    
                -   **后序遍历（Postorder Traversal）**：左-右-根（递归与迭代）。
                    
                -   **递归与迭代实现详解**：利用栈模拟递归。
                    
            -   **广度优先搜索（BFS）**：
                
                -   **层序遍历（Level Order Traversal）**：利用队列实现，逐层访问。
                    
    4.  **二叉搜索树（Binary Search Tree, BST）**
        
        -   定义：左子树所有节点值小于根节点，右子树所有节点值大于根节点。
            
        -   基本操作：查找、插入、删除（复杂）。
            
        -   特性：中序遍历结果有序。
            
    5.  **经典面试题与解法：**
        
        -   **反转二叉树（Invert Binary Tree）**：递归的优雅应用。
            
        -   **二叉树的最大/最小深度（Maximum/Minimum Depth of Binary Tree）**：BFS/DFS 解决。
            
        -   **判断平衡二叉树（Balanced Binary Tree）**：递归判断左右子树高度差。
            
        -   **对称二叉树（Symmetric Tree）**：递归判断镜像对称。
            
        -   **二叉树的最近公共祖先（Lowest Common Ancestor of a Binary Tree）**：递归法。
            
        -   **验证二叉搜索树（Validate Binary Search Tree）**：中序遍历或递归上下界判断。
            
    6.  **总结与练习题**
        
        -   本篇核心知识点回顾。
            
        -   精选练习题推荐。

### 第七篇：图与搜索：复杂关系网的探索

-   **主题：** 掌握图的基本概念、表示方法、遍历算法以及常见图算法。
    
-   **核心目标：**
    
    -   理解图的两种主要表示方法（邻接矩阵、邻接表）。
        
    -   熟练运用**深度优先搜索（DFS）**和**广度优先搜索（BFS）**遍历图。
        
    -   理解**拓扑排序**和**并查集**的应用。
        
-   **大纲与主要内容：**
    
    1.  **引言：图的无限可能**
        
        -   图在现实世界的广泛应用：社交网络、地图、游戏寻路。
            
    2.  **图（Graph）**
        
        -   基本概念：顶点（Vertex）、边（Edge）、有向图、无向图、带权图、连通图、连通分量。
            
        -   **图的表示方法：**
            
            -   **邻接矩阵（Adjacency Matrix）**：概念、优缺点。
                
            -   **邻接表（Adjacency List）**：概念、优缺点、实际应用更广泛。
                
    3.  **图的遍历：**
        
        -   **深度优先搜索（DFS）**：
            
            -   递归与迭代实现，利用栈。
                
            -   应用：判断连通性、寻找路径、**环检测**。
                
        -   **广度优先搜索（BFS）**：
            
            -   利用队列实现。
                
            -   应用：**最短路径**（无权图）、层级遍历、扩散问题。
                
    4.  **经典面试题与解法：**
        
        -   **岛屿数量（Number of Islands）**：DFS/BFS 遍历连通分量。
            
        -   **克隆图（Clone Graph）**：DFS/BFS 遍历与哈希表记录已访问节点，处理循环引用。
            
        -   **课程表（Course Schedule）**：
            
            -   **拓扑排序（Topological Sort）**：概念，Kahn 算法（基于 BFS，入度）和 DFS 算法（基于回溯）。

### 第八篇：动态规划：化繁为简的艺术

-   **主题：** 深入理解动态规划的核心思想，掌握其解题步骤和常见模型。
    
-   **核心目标：**
    
    -   识别具有**最优子结构**和**重叠子问题**的动态规划问题。
        
    -   学会定义**状态**、推导**状态转移方程**。
        
    -   熟练运用**备忘录（Memoization）**和**递推（Tabulation）**两种实现方式。
        
-   **大纲与主要内容：**
    
    1.  **引言：动态规划的魅力与困惑**
        
        -   DP 解决什么问题？它和分治、贪心的区别。
            
        -   DP 问题的典型特征。
            
    2.  **动态规划核心概念：**
        
        -   **最优子结构（Optimal Substructure）**：如何拆分问题。
            
        -   **重叠子问题（Overlapping Subproblems）**：如何避免重复计算。
            
        -   **无后效性**：未来的决策不受过去已做决策的路径影响。
            
        -   **状态（State）**：如何定义 DP[i] 或 DP[i][j] 的含义。
            
        -   **状态转移方程（State Transition Equation）**：从已知状态推导新状态的公式。
            
        -   **Base Case（基本情况）**：DP 数组的初始化。
            
    3.  **动态规划解题步骤：**
        
        -   确定 DP 数组含义。
            
        -   确定递推关系（状态转移方程）。
            
        -   确定初始化条件（base case）。
            
        -   确定遍历顺序。
            
    4.  **动态规划实现方式：**
        
        -   **自顶向下（Top-Down）**：递归 + 备忘录。
            
        -   **自底向上（Bottom-Up）**：迭代 + DP 表。
            
        -   空间优化：滚动数组（Roll Array）。
            
    5.  **经典面试题与解法：**
        
        -   **斐波那契数列（Fibonacci Number）**：最简单的 DP，理解两种实现方式。
            
        -   **爬楼梯（Climbing Stairs）**：类似斐波那契，简单 DP 建模。
            
        -   **打家劫舍（House Robber）**：线性 DP，考虑相邻元素的影响。
            
        -   **最大子数组和（Maximum Subarray）**：Kadane 算法，经典 DP。
            
        -   **零钱兑换（Coin Change）**：完全背包问题变种。
            
        -   **最长递增子序列（Longest Increasing Subsequence, LIS）**：经典 O(N2) DP，以及 O(NlogN) 优化解法（贪心 + 二分查找）。
            
        -   **背包问题（Knapsack Problem）**：
            
            -   **0/1 背包（基础）**：二维 DP 表推导，滚动数组优化。
                
            -   **完全背包**（概念）。
                
    6.  **总结与练习题**
        
        -   本篇核心知识点回顾。

### 第九篇：回溯与贪心：两大常用技巧

-   **主题：** 掌握回溯法（剪枝的暴力搜索）和贪心算法（局部最优到全局最优）的特点与应用。
    
-   **核心目标：**
    
    -   识别适合**回溯法**解决的问题，并能熟练编写回溯代码。
        
    -   理解**贪心算法**的原理，并能判断其适用性。
        
    -   区分回溯、贪心与动态规划。
        
-   **大纲与主要内容：**
    
    1.  **引言：面对选择的策略**
        
        -   如何选择合适的算法来解决特定问题。
            
    2.  **回溯法（Backtracking）**
        
        -   概念：一种通过尝试所有可能的候选解来寻找解的算法，并在找到不符合条件的解时，回溯（撤销）选择，尝试其他路径。本质是**深度优先搜索 + 剪枝**。
            
        -   **回溯三步曲（模板）**：
            
            -   **选择列表（Choices）**：每一步有哪些可能的选择。
                
            -   **路径（Path）**：记录当前已做出的选择。
                
            -   **结束条件（Base Case）**：何时达到一个有效解或无效路径。
                
        -   **剪枝（Pruning）**：优化搜索空间，避免不必要的计算。
            
        -   **经典面试题与解法：**
            
            -   **全排列（Permutations）**：最经典的回溯题，理解 `used` 数组。
                
            -   **组合总和（Combination Sum）**：带重复元素与不带重复元素的组合问题。
                
            -   **子集（Subsets）**：理解递归树的构建。
                
            -   **N 皇后（N-Queens）**（难度较高，可选）：体现复杂剪枝。
                
            -   **括号生成（Generate Parentheses）**：利用有效性剪枝。
                
    3.  **贪心算法（Greedy Algorithm）**
        
        -   概念：在每一步选择中都采取在当前状态下最好或最优（即最有利）的选择，从而希望导致结果是全局最好或最优。
            
        -   **核心挑战：** 证明局部最优能得到全局最优（**贪心选择性质**和**最优子结构性质**）。
            
        -   **经典面试题与解法：**
            
            -   **买卖股票的最佳时机（Best Time to Buy and Sell Stock）**：理解局部最优解如何构成全局最优。
                
            -   **跳跃游戏（Jump Game）**：判断能否到达终点，理解贪心策略。
                
            -   **分发饼干（Assign Cookies）**：典型的贪心分配问题。
                
            -   **柠檬水找零（Lemonade Change）**：简单贪心应用。
                
            -   **无重叠区间（Non-overlapping Intervals）**：按结束时间排序的贪心策略。
                
    4.  **总结与练习题**
        
        -   本篇核心知识点回顾。
            
        -   精选练习题推荐。
