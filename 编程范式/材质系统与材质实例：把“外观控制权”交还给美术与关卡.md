## 正文

# 材质系统与材质实例：把“外观控制权”交还给美术与关卡

> 这篇主要聊**材质与工作流**，不和别的篇章（渲染路径/GI/后处理/灯光/天空）重复。我们把显卡当计算器，把艺术意图翻译成**可复用、可参数化、可量产**的材质资产。

----------

## 1）UE 与 Unity 的“同题不同解”

-   **共同点**：两家都用**节点式可视化编辑**（UE 的 Material Editor、Unity 的 Shader Graph），再把结果编译成底层 HLSL。
    
-   **关键差异**：UE 把“**参数化**”推到极致，形成**材质 → 材质实例 → 实例层级/全局集合**的三段式工作流；Unity 传统上更依赖**材质拷贝/变体 + 代码驱动（MaterialPropertyBlock/全局参数/关键词）**。
    

一句话：

> UE 侧重“**一个父材质统治万物**”；Unity 侧重“**图形目标分层 + 代码侧细粒度驱动**”。

----------

## 2）UE 材质系统速写（从概念到落地）

### 2.1 三个开关先选对：Domain / Blend / Shading Model

-   **Material Domain**：Surface / Deferred Decal / Light Function / Post Process / UI / Volume…（选错域，后面接哪都不对）。
    
-   **Blend Mode**：Opaque / Masked / Translucent / Additive / Modulate（直接决定是否参与深度、是否有不透明度裁剪）。
    
-   **Shading Model**（常用）：
    
    -   **Lit / Default Lit**：标准 PBR。
        
    -   **Subsurface / Subsurface Profile**：皮肤、蜡、果肉这类透光材质。
        
    -   **Clear Coat**：双层清漆（车漆、清漆木器）。
        
    -   **Two Sided Foliage / Cloth / Hair / Eye / Thin Translucent / Single Layer Water / Unlit**：各自服务特定外观与能耗模型。
        

> 选型越准，后面的节点越省力。

### 2.2 节点编辑器的“骨架接线”

-   **PBR 主口**：Base Color / Metallic / Roughness / Specular / Normal / Emissive / Opacity / Opacity Mask / Ambient Occlusion / Tangent / World Position Offset(WPO)…
    
-   **可复用粒度**：
    
    -   **Material Function（MF）**：把“细噪声、磨损、三向投影、Triplanar 混合、色阶映射”等封装成函数，像积木一样复用。
        
    -   **Material Layer / Layer Blend**：把整套 PBR 口子打包成“层”，做**材质叠层**（比如“金属底材 + 漆面 + 污垢”三明治）。
        
-   **两个分支的习惯用法**：
    
    -   **Static Switch Parameter**：**编译期**分支，适合开关“大功能”（是否使用细分/是否开启清漆层）。
        
    -   **动态分支/插值（If/Lerp）**：**运行期**决定权重，适合小成本混合（比如湿度 0~1）。
        

> 大功能用静态开关，小调味用 Lerp，这是一条能救你包体和指令数的铁律。

### 2.3 材质实例（Material Instance, MI）是整套工作流的核心

-   **它是什么**：以“父材质”为模板，**仅重载参数值**（Scalar/Vector/Texture/Static Switch 等），**不重新编译**。
    
-   **它能层级化**：MI 也能当“父”，形成**实例链**。不同关卡/季节/主题皮肤只改最顶层实例，海量资产瞬间换装。
    
-   **运行时控制**：
    
    -   **MID（Material Instance Dynamic）**：蓝图/C++ 在运行时创建实例并改参数；不触发重编译，**改即生效**。
        
    -   **MPC（Material Parameter Collection）**：一套**全局参数**，可被许多材质读取（例如全场景“湿度”“时间”“风强度”），一次更新，处处生效。
        
-   **典型配方**：
    
    1.  一个**Uber 父材质**（功能齐备，静态开关干掉不用的块），
        
    2.  为不同美术主题建**MI**，
        
    3.  运行时用 **MID/MPC** 做动画与互动（受击闪烁、入水变湿、时间驱动流光）。
        

**蓝图/代码侧最小示例（UE）：**

```cpp
// C++（蓝图等价）——给第 0 个材质槽创建 MID，并改参数
UMaterialInstanceDynamic* MID = MeshComp->CreateAndSetMaterialInstanceDynamic(0);
MID->SetScalarParameterValue(TEXT("Wetness"), CurrentWetness);
MID->SetVectorParameterValue(TEXT("Tint"), FLinearColor(1,0.9,0.9));

```

> 这就是“**不改 shader、不重编译**，只改参数”。

## 3）Unity 对照：Shader Graph、变体与脚本驱动

### 3.1 Shader Graph 的日常

-   **属性（Property）**：Color/Float/Vector/Texture 暴露到材质面板；**Sub Graph** 用来封装复用逻辑。
    
-   **目标（Target）**：URP/HDRP 各自的光照与宏定义不同，Graph 需要切到对应 Target。
    
-   **关键词（Keywords）**：`_USE_CLEARCOAT`、`_USE_WETNESS` 之类的开关会生成**变体**（编译/包体成本）。大开关才用关键词，小开关用插值。
    

### 3.2 “材质实例”在 Unity 里的等价做法

-   **多材质拷贝**：最粗暴，适合离线静态皮肤；缺点是**资源爆炸**。
    
-   **MaterialPropertyBlock（推荐）**：**每渲染器**（Renderer）一份“覆盖表”，不克隆材质，不影响其他对象。
    
    ```csharp
    var mpb = new MaterialPropertyBlock();
    renderer.GetPropertyBlock(mpb);
    mpb.SetFloat("_Wetness", wetness);
    mpb.SetColor("_Tint", tint);
    renderer.SetPropertyBlock(mpb);
    
    ```
    
-   **全局参数**：`Shader.SetGlobalFloat("_Rain")` 一次下发，全场景生效（相当于 UE 的 MPC）。
    
-   **注意点**：
    
    -   在 **URP/HDRP** 下，尽量让 Shader Graph **兼容 SRP Batcher**（少用会打破常量缓冲布局的动态属性访问模式）。
        
    -   `renderer.material` 会**实例化**一份材质（产生 GC 与新资源），批量改值优先 `MaterialPropertyBlock` 或 `sharedMaterial`（有风险会影响同材质所有对象）。

## 4）纹理与材质优化（两边通用的狠活）

### 4.1 纹理打包与压缩

-   **通道打包**：把灰度贴图打包到 RGBA（如 **O-R-M-A** / **M-R-A-O**；团队约定即可）。
    
-   **法线**：首选**双通道压缩**（PC 上 BC5，移动端倾向 ASTC 2× RG），避免 BC1/BC3 产生绿紫块。
    
-   **压缩建议**
    
    -   PC/主机：BaseColor/Mask 用 **BC1/BC7**（BC7 质量佳、体积大），Normal 用 **BC5**。
        
    -   移动：**ASTC** 是万金油；大图用较粗 block，UI/锐利细节用细 block。
        
-   **Mipmap 与各向异性**：绝大多数材质**必须开 Mip**；地表和斜视角开一定的 Aniso，噪点会少很多。
    
-   **流式纹理**：超大贴图（地编、角色近景）开启**纹理流**，把内存压力交给引擎。
    

### 4.2 指令数与采样数

-   **合并采样**：能从同一纹理采多通道，就别多开一张；重复的 `normalize/length/pow` 合理共用。
    
-   **采样器（Sampler）复用**：UE 里可把 Sampler Source 设成 **Shared: Wrap/Clamp** 降低状态数量。
    
-   **分支策略**：
    
    -   昂贵功能用**静态开关**（少变体但可控），
        
    -   小权重混合用 **Lerp**（避免 runtime 分支猜测失败）。
        
-   **顶点位移（WPO）**：对缓存与剔除不友好，**只在需要的材质上开**，并控制频率与幅度。
    
-   **调试工具**：UE 的 **Shader Complexity/Quad Overdraw/Material Stats**，Unity 的 **Frame Debugger/RenderDoc**——不看就容易“感觉性能还行”。
    

----------

## 5）实战模板：三个“十分钟起飞”的材质配方

### A. 受击闪烁（Hit Flash）

-   **UE**：父材质里做 `Emissive = FlashColor * FlashStrength`；开一个 **ScalarParameter: Flash**；关卡里创建 **MID**，命中时把 `Flash` 从 0→1→0 做曲线。
    
-   **Unity**：Shader Graph 暴露 `_Flash`，用 **MaterialPropertyBlock** 挂到被打的 Renderer 上，Update 中做衰减。
    

> 无须改 Shader，无包体新增变体。

### B. 全局“下雨变湿”

-   **UE**：MPC 放一个 `Rain`，金属/地表/角色都把 Roughness 与 Normal 的细水膜效果乘进来；关卡切雨强度 = 全场一致“发亮变暗”。
    
-   **Unity**：`Shader.SetGlobalFloat("_Rain")`，各材质 Graph 里同样读全局。
    

> “统一调色台”的典型场景，策划一句话就能调一整个关卡的味道。

### C. 地表材质“去平铺”

-   **三向投影（Triplanar） + 顶点色/噪声混合**：把 A/B 两套地表纹理按低频噪声/顶点色混合，远景看不到重复。
    
-   **落地建议**：做成 **Material Function / Sub Graph**，所有地表统一接入；想换风格只要替换输入纹理集。

## 6）跨引擎的决策清单（落到产线）

1.  **先定“父材质/主 Graph”**：
    
    -   UE：一个**Uber 父材质 + 几个静态开关**；
        
    -   Unity：一个**主 Graph + 少量关键词**（把大功能做成 Sub Graph）。
        
2.  **参数分层**：
    
    -   UE：**MI 层级**负责美术调参；运行时变化走 **MID/MPC**。
        
    -   Unity：静态皮肤用材质拷贝；运行时变化走 **MPB/Global**。
        
3.  **压缩与流**：把**贴图分层与压缩策略**写进导入规则（法线=BC5/ASTC 双通道；ORM 打包；重要纹理流式）。
    
4.  **变体控制**：只给“大功能”一个关键词；其余用 Lerp。版本增长期定期做**Shader Stripping**。
    
5.  **度量先行**：把“**材质复杂度视图/Frame Debugger**”当日检，不合格就回炉。
    

----------

## 7）核心总结

-   **UE 的优势**在于：**材质实例（MI/MID）+ MPC + 函数/层**把“**复用**、**参数化**、**批量换肤**”做到了极致——**改参数而非改 Shader**，是速度与稳定性的源头。
    
-   **Unity 的优势**在于：**Graph + SRP 生态**与**脚本驱动（MPB/Global）**，细粒度控制到每个 Renderer；但要自觉管理**关键词/变体/材质数量**。
    
-   **通用铁三角**：**好结构（父材质/主 Graph） + 好参数（实例/MPB/MPC） + 好度量（复杂度与采样数）**。只要这个三角站稳，项目就能在“风格多样 + 产线稳态 + 性能可控”之间拿到平衡点。
