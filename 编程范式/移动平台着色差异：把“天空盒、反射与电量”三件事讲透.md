## 正文

# 移动平台着色差异：把“天空盒、反射与电量”三件事讲透

> 这篇主要聊两块：**天空体系**（UE 的 Sky Atmosphere + SkyLight，对比 Unity 的 Skybox + 反射探针）和**移动端着色差异**（API、精度、颜色空间、Shader 变体与带宽）。至于渲染路径、材质、后处理、 Lumen/GI，可以翻阅前文。

----------

## 1）为什么从“天空”开始？

天空不是背景布，它是**全场环境光与反射的来源**。一旦天空体系定准：**色温、雾化、反射强度**都会跟着稳住。反过来，天空配置混乱，后面所有灯光与材质都在救火。

----------

## 2）Unreal：Sky Atmosphere + SkyLight 的“物理底片”

### 2.1 Sky Atmosphere：把大气当成模型而非贴图

UE 的 **Sky Atmosphere** 直接模拟瑞利/米氏散射、地球曲率与高度分布，从地面到高空到近太空**一条链**过渡，自然解决“白天蓝、黄昏暖、夜空深”的物理逻辑。你可以调地表半径、尺度高度，甚至做“外星大气”。

**落地心法**：

-   只放**一个** Sky Atmosphere，当“母版天空”。
    
-   把**主定向光（Sun）**与 Atmosphere 关联（Atmosphere Sun Light），让太阳轨迹驱动天空颜色。
    
-   云层建议用 **Volumetric Clouds**（材质驱动、与 Sky+SkyLight互补），做昼夜与天气的中频层次。
    

### 2.2 SkyLight：让“天空”真正照进场景

**SkyLight** 会捕捉远景与天空生成环境光与反射。可用**HDRI Cubemap**指定，也可让它**实时捕捉**场景。要点：

-   **Real Time Capture**（Stationary/Movable 可用）适合**动态昼夜循环**与**实时反射**；但它是重活，虽比蓝图 `RecaptureSky` 平滑（GPU 内完成、不阻塞 CPU），依然要谨慎用。
    
-   如果不开实时捕捉，**改天色后手动 Recapture**（蓝图节点）更新一次。频繁调用会**卡顿**。
    

**工程建议**：

-   开发期用 Real Time Capture 方便看效果；进入性能阶段改为**定时/事件触发**的 Recapture 或使用**静态 HDRI**。
    
-   反射与天空不一致时，先看 SkyLight 用的源（HDRI vs 场景捕捉）是否统一。
    

----------

## 3）Unity：Skybox + Reflection Probe 是“光与反射的分工协作”

### 3.1 Skybox：全局环境与天穹

Unity 的 **Skybox** 负责渲染天穹；在 URP/HDRP 下，它也往往作为**环境反射的兜底来源**（Lighting 窗口环境源）。制作时用 Skybox Shader 创建材质、在 Lighting 里指定为场景 Skybox。

### 3.2 Reflection Probe：把“局部世界”装进立方体

**反射探针**会生成**局部 Cubemap**，与 Skybox 共同决定反射：

-   **Blending 规则**：同一物体最多受**两个探针**影响，按**Importance**与体积、距离进行选择与混合；**Box Projection**可让盒投影近似“平面内真实映射”。URP 中需在 URP Asset 也开启 Box Projection。
    
-   **刷新模式**：`OnAwake`（只第一次）、`EveryFrame`（每帧，极贵）、`ViaScripting`（按需脚本触发）+ **Time Slicing**（分帧更新）。一般移动端选 **ViaScripting + 分帧**，只对**可见且变化大的**区域更新。
    

**工程建议**：

-   **室内多探针**：门厅/廊道放小体积高权重探针，远处留全局低权重探针做兜底。
    
-   **室外+Skybox**：物体处在两个探针边界、权重不满时，剩余权重由**Skybox 环境反射**补齐，避免“反射掉线”。
    

----------

## 4）移动平台着色差异：别跟“带宽与精度”硬刚

移动 GPU（Adreno/Mali/PowerVR）大多是**Tile-Based** 架构：屏幕被切成小瓦片在片上内存里处理，**带宽是第一成本**。这意味着几个常识：**透明少、读写少、分辨率谨慎涨**。

### 4.1 图形 API 与渲染路径

-   **UE（Android/iOS）**：Android 默认 **OpenGL ES 3.2**，较新的设备可上 **Vulkan**，谷歌更新Vulkan适配后，Vulkan性能会显著更好；iOS 走 **Metal**，且可启用“Desktop Renderer”以获得桌面特性（代价更高）。
    
-   **Unity（URP）**：Android（Vulkan/ES3）、iOS（Metal）。URP 官方建议在移动端**禁用深度预热/中间贴图等额外缓冲**与**不需要的照明特性**。
    

### 4.2 HDR 与颜色空间

-   **HDR**：移动端如无强诉求，**关闭 HDR**（减少更高精度缓冲/色调映射路径）。URP 文档明确：如果不需要，关；需要则用合适精度。UE 的 **Mobile HDR** 也建议在慢设备关闭。
    
-   **Linear vs Gamma**：线性空间在物理一致性与调色上更靠谱，但在部分旧机上**Gamma 可能更快**。总体建议：**能上 Linear 就上 Linear**，但要在目标机验收。
    

### 4.3 Shader 精度与数学

-   **Unity HLSL 类型**：`float`/`half`/`fixed` 在移动端**真的有差别**。`half`（16-bit）适合颜色/方向/中间量，`float`保留给位置/UV/大范围运算。避免在关键路径频繁 `half↔float` 转换；向量化使用 2/4 分量有助于 FP16 吞吐。
    
-   **UE**：同理要关注最小精度与 FP16 机会，但以平台编译器为准，重点是**减少分支与纹理采样**，而不是盲信某个精度一键提速。
    

### 4.4 透明与过度绘制（Overdraw）

-   **半透明是移动 GPU 的天敌**：不写深度、常带混合，像素会被反复着色。尽量用**裁剪（Alpha Test）** + 贴图 Cookie 模拟，或把大面积雾/光效转为**体积/屏幕空间的单次合成**。
    
-   **排序与瓦片**：保持**不透明前向、从前到后**的绘制顺序，减少瓦片内的多次覆盖。ARM 的最佳实践也强调**合理清除/无用附件无效化**降低带宽。
    

### 4.5 纹理与反射

-   **压缩**：移动端首选 **ASTC/ETC2**；法线倾向双通道（ASTC RG），天空 HDRI 走高质量压缩。
    
-   **反射**：Unity 的**实时探针**慎用 `EveryFrame`，尽量脚本触发 + **Time Slicing**；UE 的 SkyLight **Real Time Capture**只给**关键镜头/相机附近**开。
    

### 4.6 URP 的“剃刀”：把没用的功能剪掉就快了

URP 里**禁用未使用的功能**会**自动剔除 Shader 变体**，显著缩短编译与降低内存占用：比如不用额外光源阴影、就把对应特性在 URP Asset 关掉。构建日志还能导出变体剔除报告。

----------

## 5）把天空与移动端合体：两条实操流水线

### 5.1 UE：动态昼夜 + 可落地性能

1.  **Sky Atmosphere + Directional Sun** 绑定；Volumetric Clouds 低采样做层次。
    
2.  **SkyLight** 开发期用 Real Time Capture；进优化期改**定点 Recapture**（时段触发或天气切换），避免每帧更新。
    
3.  **移动端**：Android 低端用 ES3.2；高端白名单上 Vulkan；iOS 走 Metal。Mobile HDR 视需求关闭。
    
4.  **着色**：减少半透明特效，能用 Cookie/遮罩就别铺满屏的粒子雾。
    

### 5.2 Unity（URP）：室外开放世界 + 可控反射

1.  **Skybox** 定全局环境；Lighting 里用它作为 **Environment Source**。
    
2.  **反射探针**：室内走密集小探针 + Importance；室外以稀疏探针 + Skybox 兜底。Box Projection 在 URP Asset 与探针上都要勾。
    
3.  **刷新策略**：默认 `OnAwake`，玩家靠近或事件触发时 `ViaScripting` + Time Slicing；尽量别 `EveryFrame`。
    
4.  **移动端设置**：URP Asset 关闭不必要的 **HDR、Opaque/Depth Texture、额外光源阴影**等重项；使用 **Simple Lit/Baked Lit** 替代复杂光照。
    
5.  **颜色与精度**：目标机允许就用 **Linear**；Shader 中合理用 `half`，集中在颜色与方向通道，避免无谓精度来回转换。
    

----------

## 6）排雷清单（踩过就懂）

-   **UE：天空变了反射不变** → 你忘了给 SkyLight **Recapture** 或误以为 Real Time Capture 开着。
    
-   **Unity：反射探针卡顿** → 使用了 `EveryFrame` 且无 Time Slicing；改为 `ViaScripting` 并只在**可见且变化**时更新。
    
-   **移动端发热掉帧** → 打开了 HDR/过多附加缓冲/半透明堆叠；先从 URP/项目设置把**不需要的特性关**掉，再精简透明。
    
-   **昼夜循环曝光乱跳** → 摄像机曝光与天空色温都在变；把“物理曝光链”归摄像机，天空只管物理色，反射按**固定频率**更新。
    
-   **Shader 体量爆炸** → 变体太多。在 URP 里**禁用未用特性**触发**自动剔除**，并用日志核对。
    

----------

## 7）核心总结

-   **天空是光照与反射的基色**：UE 用 **Sky Atmosphere + SkyLight** 做物理一致的天空与环境；Unity 用 **Skybox + Reflection Probe** 精细控制局部反射，Skybox 兜底全场。
    
-   **移动端先谈带宽与精度**：选对 **API**，关掉不需要的 **HDR/缓冲**，**半透明克制**，`half` 合理用，**Shader 变体能剃就剃**。
    
-   **流程化**：开发期追求“所见即所得”（实时捕捉/每帧刷新），优化期改“按需更新/事件触发”。画面稳定，帧率也稳。
