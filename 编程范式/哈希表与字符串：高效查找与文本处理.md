## 第四篇：哈希表与字符串：高效查找与文本处理

### 引言：哈希表的魔力

朋友，恭喜你！我们已经掌握了复杂度分析、线性数据结构（数组、链表）以及受限容器（栈、队列、堆）。今天，我们将揭开一个真正具有“魔力”的数据结构的神秘面纱——**哈希表（Hash Table）**，以及在日常开发中无处不在的**字符串（String）**。

哈希表的出现，极大地解决了我们对数据进行**高效查找、插入和删除**的需求。设想一下，如果你有一本书，要快速找到某个词的定义，你可能需要翻阅索引，然后直接跳到对应的页码。哈希表就扮演了“索引”的角色，它能让你**在平均 O(1) 的时间复杂度内**（也就是常数时间，与数据量大小无关）完成这些操作。这是传统数组或链表都难以企及的效率。

那么，哈希表的魔力源于何处？为什么它能如此高效？

核心在于它的**直接寻址能力**。通过一个特殊的函数——**哈希函数（Hash Function）**，它可以将任意键（Key）映射到一个唯一的索引（Index），从而直接访问到存储在该索引位置的值（Value）。这就像一个超级智能的图书馆，你告诉它书名，它立刻就能告诉你书架的精确位置，而不是让你挨个书架去找。

在 Unity 开发中，哈希表更是无处不在：

-   **资源管理**：通过字符串名称快速查找对应的 `GameObject`、`Material` 或其他资源。
    
-   **游戏状态管理**：存储键值对形式的游戏数据，如玩家属性、配置参数。
    
-   **缓存系统**：快速查找已计算或已加载的数据。
    
-   **实体组件系统（ECS）**：虽然底层复杂，但其快速查找组件的能力，很多时候依赖于类似哈希表的结构。
    
-   **多语言支持**：通过语言键快速查找对应的文本。
    

理解哈希表，是掌握高性能数据处理的关键一步。

----------

### 哈希表（Hash Table/HashMap/HashSet）

#### 原理：哈希函数、哈希冲突与冲突解决

哈希表的核心原理是**键值对（Key-Value Pair）的存储。它利用一个哈希函数**将键映射到数组的某个索引位置，然后将值存储在那里。

-   **哈希函数（Hash Function）**：这是一个将任意大小的输入（键）转换成固定大小的输出（哈希值/索引）的函数。一个好的哈希函数应该：
    
    -   **确定性**：同一个输入总是产生相同的输出。
        
    -   **高效性**：计算速度快。
        
    -   **均匀性**：尽可能均匀地将不同键映射到不同的索引，减少冲突。
        
-   **哈希冲突（Hash Collision）**：由于哈希函数的输出空间通常小于输入的可能空间（例如，将无限个字符串映射到有限的数组索引），不同的键可能会被哈希到相同的索引位置，这就是哈希冲突。这是不可避免的。
    
-   **冲突解决策略**：当发生哈希冲突时，我们需要策略来处理，确保所有键值对都能被正确存储和检索。常见的策略有两种：
    
    1.  **链地址法（Separate Chaining）**：
        
        -   在每个哈希桶（数组的每个索引位置）中，维护一个链表（或红黑树等其他数据结构）。
            
        -   当多个键哈希到同一个索引时，它们都作为节点添加到该索引对应的链表中。
            
        -   **优点**：实现简单，对哈希函数质量要求相对较低，删除方便。
            
        -   **缺点**：需要额外的空间来存储链表节点，查找时可能需要遍历链表（在冲突严重时性能下降）。
            
        -   这是 C# 中 `Dictionary<TKey, TValue>` 和 `HashSet<T>` 的默认实现策略之一。
            
        
        C#
        
        ```
        // 链地址法示意图：
        // 数组索引 0: null
        // 数组索引 1: (KeyA, ValueA) -> (KeyC, ValueC) -> null (KeyA 和 KeyC 哈希到索引 1)
        // 数组索引 2: (KeyB, ValueB) -> null
        // ...
        
        ```
        
    2.  **开放寻址法（Open Addressing）**：
        
        -   当发生冲突时，不使用链表，而是寻找数组中的下一个空闲位置来存储冲突的元素。
            
        -   查找时，如果哈希到的位置不是目标元素，则继续探查下一个位置，直到找到目标元素或遇到空位。
            
        -   常见的探查方法有：线性探查（Linear Probing）、二次探查（Quadratic Probing）、双重哈希（Double Hashing）。
            
        -   **优点**：没有指针开销，缓存友好（数据连续）。
            
        -   **缺点**：实现相对复杂，对哈希函数质量要求高（容易形成聚集），删除元素可能较复杂（需要标记而不是真正删除）。
            
        -   在某些场景和语言中会使用。
            
        
        C#
        
        ```
        // 开放寻址法示意图 (线性探查)：
        // KeyA 哈希到索引 1 -> 存储在 1
        // KeyC 哈希到索引 1 -> 冲突，探查下一个空位 -> 存储在 2
        // KeyB 哈希到索引 2 -> 冲突，探查下一个空位 -> 存储在 3
        // 数组: [_, (KeyA, ValueA), (KeyC, ValueC), (KeyB, ValueB), _]
        
        ```
        

#### 性能考量

-   **平均时间复杂度：O(1)**
    
    -   在理想情况下（哈希函数均匀，冲突很少），哈希表的查找、插入、删除操作的平均时间复杂度都是 O(1)。
        
    -   这是因为它可以通过哈希函数直接计算出存储位置。
        
-   **最坏时间复杂度：O(N)**
    
    -   如果哈希函数设计得非常糟糕，或者所有键都哈希到同一个索引（极端冲突），那么哈希表会退化成一个链表（链地址法）或需要遍历大部分数组（开放寻址法），导致所有操作的时间复杂度退化为 O(N)。
        
-   **空间开销：** 哈希表需要额外的空间来存储哈希桶本身以及处理冲突的数据结构（如链表节点）。此外，为了保持较低的冲突率和良好的性能，通常需要保持一定的**负载因子（Load Factor）**（已存储元素数量 / 总桶数量），当负载因子超过阈值时，哈希表会进行**扩容（Resizing）**，创建一个更大的底层数组，并重新计算所有元素的哈希值并插入到新数组中。这个扩容过程通常是 O(N) 的。
    

#### 哈希集合（HashSet）与哈希映射（HashMap/Dictionary）

-   **哈希映射（HashMap/Dictionary in C#）**：
    
    -   存储**键值对**（Key-Value Pair），每个键都是唯一的，用于映射到相应的值。
        
    -   核心操作：`Put(key, value)` / `Add(key, value)`、`Get(key)`、`Remove(key)`、`ContainsKey(key)`。
        
    -   对应 C# 中的 `System.Collections.Generic.Dictionary<TKey, TValue>`。
        
-   **哈希集合（HashSet）**：
    
    -   存储一组**唯一的值**，不存储键值对。可以看作是只存储键，而值是一个占位符的哈希映射。
        
    -   核心操作：`Add(value)`、`Remove(value)`、`Contains(value)`。
        
    -   用于**去重**和快速判断元素是否存在。
        
    -   对应 C# 中的 `System.Collections.Generic.HashSet<T>`。
        

#### 适用场景

-   **快速查找**：给定一个键，快速找到其对应的值或判断其是否存在。
    
-   **去重**：将元素添加到 `HashSet` 中，自动去除重复元素。
    
-   **计数**：使用 `Dictionary` 存储元素及其出现次数。
    
-   **缓存**：将计算结果或加载数据存储起来，以便快速重复访问。
    
-   **集合操作**：如求交集、并集、差集等。

##### 1. 两数之和 (Two Sum)

-   题目描述：
    
    给定一个整数数组 nums 和一个整数目标值 target，请你在该数组中找出和为目标值 target 的那两个整数，并返回它们的数组下标。
    
    你可以假设每种输入只会对应一个答案。但是，数组中同一个元素在答案里不能重复出现。
    
    示例：
    
    输入: nums = [2, 7, 11, 15], target = 9
    
    输出: [0, 1]
    
    解释: nums[0] + nums[1] == 9
    
-   解题思路：哈希表优化查找
    
    这道题是哈希表入门的经典之作。
    
    **暴力法**是使用两重循环，时间复杂度 O(N2)。但我们可以用哈希表优化到 O(N)。
    
    **步骤：**
    
    1.  创建一个哈希表 `map` (C# 中是 `Dictionary<int, int>`)，用于存储数组中的数字及其对应的索引。键是数字，值是索引。
        
    2.  遍历数组 nums，对于每个数字 num (及其索引 i)：
        
        a. 计算 complement = target - num（即我们需要找到的另一个数字）。
        
        b. 检查 map 中是否已经存在 complement 这个键：
        
        i. 如果存在，说明我们找到了两个数，num 和 complement。返回它们的索引：[map[complement], i]。
        
        ii. 如果不存在，将当前数字 num 及其索引 i 存入 map：map[num] = i。
        
    3.  如果遍历结束后没有找到，通常题目会保证有解，所以实际不会发生。
        
    
    -   **代码实现：**
        
        C#
        
        ```
        using System.Collections.Generic;
        
        public class Solution {
            public int[] TwoSum(int[] nums, int target) {
                // key: 数字值, value: 索引
                Dictionary<int, int> map = new Dictionary<int, int>();
        
                for (int i = 0; i < nums.Length; i++) {
                    int num = nums[i];
                    int complement = target - num; // 找到需要的另一个数
        
                    if (map.ContainsKey(complement)) {
                        // 如果哈希表中已经存在 complement，说明找到了
                        return new int[] { map[complement], i };
                    }
        
                    // 将当前数字及其索引存入哈希表
                    // 注意：这里是先查找再存入，以确保不会用同一个元素两次
                    map[num] = i;
                }
        
                // 根据题目假设，一定有解，所以这里不会执行到
                return null;
            }
        }
        
        ```
        
    -   **复杂度分析：**
        
        -   **时间复杂度：O(N)**。因为我们只遍历了一次数组，哈希表的查找和插入操作在平均情况下是 O(1)。
            
        -   **空间复杂度：O(N)**。在最坏情况下，哈希表会存储数组中的所有元素。

##### 2. 字母异位词分组 (Group Anagrams)

-   题目描述：
    
    给你一个字符串数组，请你将 字母异位词 组合在一起。可以按任意顺序返回结果列表。
    
    字母异位词 是由相同字母不同排列形成的字符串。
    
    示例：
    
    输入: strs = ["eat", "tea", "tan", "ate", "nat", "bat"]
    
    输出: [["bat"], ["nat", "tan"], ["ate", "eat", "tea"]]
    
-   解题思路：哈希表的键设计
    
    这道题的关键在于如何判断两个字符串是否是字母异位词，以及如何将它们作为哈希表的键。
    
    **核心思想：** 如果两个字符串是字母异位词，那么它们排序后的字符串是相同的，或者它们的字符计数（每个字符出现的次数）是相同的。我们可以利用这一点作为哈希表的键。
    
    **方法一：排序字符串作为键**
    
    1.  创建一个哈希表 `map` (C# 中是 `Dictionary<string, IList<string>>`)，键是排序后的字符串，值是一个字符串列表，存储所有属于该异位词组的原始字符串。
        
    2.  遍历输入字符串数组 strs 中的每个字符串 s：
        
        a. 将 s 转换为字符数组，并进行排序。
        
        b. 将排序后的字符数组转换回字符串 sortedStr。这个 sortedStr 将作为哈希表的键。
        
        c. 如果 map 中不存在 sortedStr，则创建一个新的列表并将其添加到 map 中。
        
        d. 将原始字符串 s 添加到 map[sortedStr] 对应的列表中。
        
    3.  遍历结束后，`map` 中所有值（列表）就是我们需要的异位词分组。
        
    
    -   **代码实现：**
        
        C#
        
        ```
        using System.Collections.Generic;
        using System.Linq; // 用于 OrderBy 和 ToArray
        
        public class Solution {
            public IList<IList<string>> GroupAnagrams(string[] strs) {
                // key: 排序后的字符串 (作为异位词的唯一标识)
                // value: 原始字符串列表
                Dictionary<string, IList<string>> map = new Dictionary<string, IList<string>>();
        
                foreach (string s in strs) {
                    char[] charArray = s.ToCharArray();
                    System.Array.Sort(charArray); // 对字符数组进行排序
                    string sortedStr = new string(charArray); // 转换回字符串作为键
        
                    if (!map.ContainsKey(sortedStr)) {
                        map[sortedStr] = new List<string>();
                    }
                    map[sortedStr].Add(s);
                }
        
                // 返回字典中所有值（列表）的列表
                return new List<IList<string>>(map.Values);
            }
        }
        
        ```
        
    -   **复杂度分析：**
        
        -   **时间复杂度：O(NcdotKlogK)**，其中 N 是字符串数组的长度，K 是字符串的最大长度。
            
            -   遍历 N 个字符串：O(N)。
                
            -   每个字符串进行排序：O(KlogK)。
                
            -   哈希表的插入和查找在平均情况下是 O(K)（取决于字符串长度，因为键比较是 O(K)）。
                
            -   所以总时间复杂度是 Ncdot(KlogK+K)，简化为 O(NcdotKlogK)。
                
        -   **空间复杂度：O(NcdotK)**。哈希表会存储所有字符串，以及排序后的字符串作为键。
            
    
    **方法二：字符计数作为键（更优）**
    
    1.  我们可以用一个长度为 26 的整数数组（表示 'a' 到 'z' 的出现次数）来作为键。将这个数组转换为字符串（例如 `"#1#0#0#2..."`）作为键。
        
    2.  遍历每个字符串，统计其字符出现次数，然后构建这个“计数字符串”作为键。
        
    3.  后续操作与方法一类似。
        
    
    -   **代码实现：**
        
        C#
        
        ```
        using System.Collections.Generic;
        using System.Text; // 用于 StringBuilder
        
        public class Solution {
            public IList<IList<string>> GroupAnagrams(string[] strs) {
                Dictionary<string, IList<string>> map = new Dictionary<string, IList<string>>();
        
                foreach (string s in strs) {
                    int[] counts = new int[26]; // 记录每个字符出现次数
                    foreach (char c in s) {
                        counts[c - 'a']++;
                    }
        
                    // 将字符计数数组转换为一个字符串作为哈希表的键
                    StringBuilder sb = new StringBuilder();
                    for (int i = 0; i < 26; i++) {
                        sb.Append('#'); // 分隔符
                        sb.Append(counts[i]);
                    }
                    string key = sb.ToString();
        
                    if (!map.ContainsKey(key)) {
                        map[key] = new List<string>();
                    }
                    map[key].Add(s);
                }
        
                return new List<IList<string>>(map.Values);
            }
        }
        
        ```
        
    -   **复杂度分析：**
        
        -   **时间复杂度：O(NcdotK)**，其中 N 是字符串数组的长度，K 是字符串的最大长度。
            
            -   遍历 N 个字符串：O(N)。
                
            -   每个字符串进行字符计数：O(K)。
                
            -   构建计数字符串：O(26) 即 O(1)。
                
            -   哈希表的插入和查找在平均情况下是 O(1) (键比较是 O(26))。
                
            -   所以总时间复杂度是 Ncdot(K+1)，简化为 O(NcdotK)。
                
        -   **空间复杂度：O(NcdotK)**。哈希表存储所有字符串和计数键。
            
-   **思考：** 这个题目体现了哈希表的灵活性：如何根据问题的特性，设计出能够唯一标识一类事物的**哈希键**。字符计数法在字符集固定且较小的情况下比排序更高效。

##### 3. 无重复字符的最长子串 (Longest Substring Without Repeating Characters)

-   题目描述：
    
    给定一个字符串 s ，请你找出其中不含有重复字符的 最长子串 的长度。
    
    示例：
    
    输入: s = "abcabcbb"
    
    输出: 3
    
    解释: 因为无重复字符的最长子串是 "abc"，所以其长度为 3。
    
-   解题思路：滑动窗口结合哈希表
    
    这道题是**滑动窗口（Sliding Window）**算法的经典应用，并且通常与哈希表结合使用来高效判断窗口内是否有重复字符。
    
    **滑动窗口**：顾名思义，就像在字符串上有一个可以左右移动和伸缩的“窗口”。我们维护两个指针，`left` (窗口左边界) 和 `right` (窗口右边界)。`right` 指针负责扩张窗口，`left` 指针负责收缩窗口。
    
    **步骤：**
    
    1.  初始化 `left = 0` (窗口左边界)，`maxLength = 0` (最长无重复子串长度)。
        
    2.  创建一个哈希集合 `charSet` (C# 中是 `HashSet<char>`)，用于存储当前窗口内的字符，以便快速判断重复。
        
    3.  右指针 right 扩张窗口： 遍历字符串 s，right 从 0 到 s.Length - 1：
        
        a. 将 s[right] 加入 charSet。
        
        b. 如果 s[right] 已经在 charSet 中存在 (发现重复字符)：
        
        i. 收缩窗口： left 指针开始向右移动。每次移动，将 s[left] 从 charSet 中移除，直到 charSet 中不再包含 s[right]。
        
        ii. 移除后，再次将 s[right] 加入 charSet (因为此时窗口已经不包含重复的 s[right] 了)。
        
        c. 更新 maxLength = Math.Max(maxLength, right - left + 1) (当前窗口的长度)。
        
    4.  返回 `maxLength`。
        
    
    -   **图解 (以 "abcabcbb" 为例):**
        
        -   `s = "abcabcbb"`
            
        -   `left = 0, maxLength = 0, charSet = {}`
            
        -   `right = 0, c = 'a'`: `charSet.Add('a')`. `charSet = {'a'}`. `maxLength = max(0, 0-0+1) = 1`.
            
        -   `right = 1, c = 'b'`: `charSet.Add('b')`. `charSet = {'a', 'b'}`. `maxLength = max(1, 1-0+1) = 2`.
            
        -   `right = 2, c = 'c'`: `charSet.Add('c')`. `charSet = {'a', 'b', 'c'}`. `maxLength = max(2, 2-0+1) = 3`.
            
        -   `right = 3, c = 'a'`: `charSet.Contains('a')` 为 `true` (重复)。
            
            -   `charSet.Remove(s[left=0] = 'a')`. `left++`. `charSet = {'b', 'c'}`. `left=1`.
                
            -   此时 `charSet` 中仍无 `a`。继续 `charSet.Add('a')`. `charSet = {'b', 'c', 'a'}`.
                
            -   `maxLength = max(3, 3-1+1) = 3`.
                
        -   `right = 4, c = 'b'`: `charSet.Contains('b')` 为 `true` (重复)。
            
            -   `charSet.Remove(s[left=1] = 'b')`. `left++`. `charSet = {'c', 'a'}`. `left=2`.
                
            -   此时 `charSet` 中仍无 `b`。继续 `charSet.Add('b')`. `charSet = {'c', 'a', 'b'}`.
                
            -   `maxLength = max(3, 4-2+1) = 3`.
                
        -   ...以此类推，最终结果 3。
            
    -   优化：使用 Dictionary<char, int> 存储字符及其最后出现的索引
        
        这种方式可以更高效地收缩窗口，避免一次次 Remove。当发现重复字符 s[right] 时，直接将 left 移动到 map[s[right]] + 1 的位置，并且把小于新 left 索引的字符从哈希表中移除（或更新其索引）。更简单的做法是：当发现重复时，直接将 left 跳到重复字符上一次出现的位置的下一位。
        
        C#
        
        ```
        using System.Collections.Generic;
        using System;
        
        public class Solution {
            public int LengthOfLongestSubstring(string s) {
                if (s.Length == 0) {
                    return 0;
                }
        
                // key: 字符, value: 字符在字符串中最后出现的索引
                Dictionary<char, int> charIndexMap = new Dictionary<char, int>();
                int maxLength = 0;
                int left = 0; // 窗口左边界
        
                for (int right = 0; right < s.Length; right++) {
                    char currentChar = s[right];
        
                    // 如果当前字符在窗口内（即在map中存在，且其索引大于等于当前left），
                    // 说明有重复，需要移动 left
                    if (charIndexMap.ContainsKey(currentChar) && charIndexMap[currentChar] >= left) {
                        left = charIndexMap[currentChar] + 1; // 将左边界移动到重复字符的下一个位置
                    }
        
                    // 更新或添加当前字符的最新索引
                    charIndexMap[currentChar] = right;
        
                    // 更新最大长度
                    maxLength = Math.Max(maxLength, right - left + 1);
                }
        
                return maxLength;
            }
        }
        
        ```
        
    -   **复杂度分析：**
        
        -   **时间复杂度：O(N)**。`left` 和 `right` 指针都只会向前移动，每个字符最多被访问两次（一次 `right` 扫描，一次 `left` 移动）。哈希表的查找和插入都是 O(1)。
            
        -   **空间复杂度：O(C)**，其中 C 是字符集的大小（例如，对于 ASCII 字符是 128 或 256）。哈希表最多存储不重复字符的数量。
            
-   **思考：** 滑动窗口是处理字符串/数组子串/子数组问题的强大模式。结合哈希表，能高效地维护窗口内的数据状态。理解如何扩张和收缩窗口是关键。

##### 4. 最长连续序列 (Longest Consecutive Sequence)

-   题目描述：
    
    给定一个未排序的整数数组 nums，找出数字连续的最长序列（即序列中的数字是连续的）的长度。
    
    请你设计并实现时间复杂度为 O(n) 的算法。
    
    示例：
    
    输入: nums = [100, 4, 200, 1, 3, 2]
    
    输出: 4
    
    解释: 最长数字连续序列是 [1, 2, 3, 4]。它的长度为 4。
    
-   解题思路：哈希集合优化查找
    
    如果直接排序，时间复杂度是 O(NlogN)。要达到 O(N)，我们需要借助哈希集合的 O(1) 查找特性。
    
    **核心思想：** 将所有数字存入一个 `HashSet` 中，以便快速检查某个数字是否存在。然后，遍历 `HashSet` 中的每个数字。对于每个数字 `num`，我们只在它是某个连续序列的“起点”时才开始计算其所在序列的长度。
    
    如何判断一个数字是否是序列的起点？
    
    如果 num - 1 不在 HashSet 中，那么 num 就是一个潜在的序列起点。
    
    **步骤：**
    
    1.  将所有 `nums` 中的数字存入一个 `HashSet<int> numSet`。这允许我们 O(1) 时间检查一个数字是否存在。
        
    2.  初始化 `longestStreak = 0`。
        
    3.  遍历 numSet 中的每个数字 num：
        
        a. 检查 num 是否是序列的起点： 如果 numSet.Contains(num - 1) 为 false，说明 num 是一个新序列的起点。
        
        b. 如果 num 是起点，则开始计算当前连续序列的长度 currentStreak：
        
        i. 初始化 currentNum = num。
        
        ii. 初始化 currentStreak = 1。
        
        iii. 循环： 只要 numSet.Contains(currentNum + 1) 为 true，就说明序列还在继续：
        
        * currentNum++。
        
        * currentStreak++。
        
        iv. 更新 longestStreak = Math.Max(longestStreak, currentStreak)。
        
    4.  返回 `longestStreak`。
        
    
    -   **图解 (以 [100, 4, 200, 1, 3, 2] 为例):**
        
        -   `numSet = {100, 4, 200, 1, 3, 2}`
            
        -   `longestStreak = 0`
            
        -   遍历 `numSet`:
            
            -   **num = 100:** `numSet.Contains(99)`? No. 100 是起点。
                
                -   `currentNum = 100`, `currentStreak = 1`.
                    
                -   `numSet.Contains(101)`? No.
                    
                -   `longestStreak = max(0, 1) = 1`.
                    
            -   **num = 4:** `numSet.Contains(3)`? Yes. (4不是起点，跳过)
                
            -   **num = 200:** `numSet.Contains(199)`? No. 200 是起点。
                
                -   `currentNum = 200`, `currentStreak = 1`.
                    
                -   `numSet.Contains(201)`? No.
                    
                -   `longestStreak = max(1, 1) = 1`.
                    
            -   **num = 1:** `numSet.Contains(0)`? No. 1 是起点。
                
                -   `currentNum = 1`, `currentStreak = 1`.
                    
                -   `numSet.Contains(2)`? Yes. `currentNum = 2`, `currentStreak = 2`.
                    
                -   `numSet.Contains(3)`? Yes. `currentNum = 3`, `currentStreak = 3`.
                    
                -   `numSet.Contains(4)`? Yes. `currentNum = 4`, `currentStreak = 4`.
                    
                -   `numSet.Contains(5)`? No.
                    
                -   `longestStreak = max(1, 4) = 4`.
                    
            -   **num = 3:** `numSet.Contains(2)`? Yes. (3不是起点，跳过)
                
            -   **num = 2:** `numSet.Contains(1)`? Yes. (2不是起点，跳过)
                
        -   最终 `longestStreak = 4`。
            
    -   **代码实现：**
        
        C#
        
        ```
        using System.Collections.Generic;
        using System;
        
        public class Solution {
            public int LongestConsecutive(int[] nums) {
                if (nums.Length == 0) {
                    return 0;
                }
        
                // 将所有数字存入哈希集合，以便 O(1) 查找
                HashSet<int> numSet = new HashSet<int>(nums);
                int longestStreak = 0;
        
                // 遍历集合中的每个数字
                foreach (int num in numSet) {
                    // 只有当 num 是一个连续序列的起点时才处理
                    // 即 num - 1 不在集合中
                    if (!numSet.Contains(num - 1)) {
                        int currentNum = num;
                        int currentStreak = 1;
        
                        // 不断查找下一个连续的数字
                        while (numSet.Contains(currentNum + 1)) {
                            currentNum++;
                            currentStreak++;
                        }
        
                        // 更新最长序列长度
                        longestStreak = Math.Max(longestStreak, currentStreak);
                    }
                }
        
                return longestStreak;
            }
        }
        
        ```
        
    -   **复杂度分析：**
        
        -   **时间复杂度：O(N)**。
            
            -   将所有元素存入 `HashSet`：O(N)。
                
            -   外层循环遍历 `HashSet` 中的 N 个元素：O(N)。
                
            -   内层 `while` 循环：看起来像嵌套循环，但实际上每个数字（包括 `num` 和 `currentNum + 1` 序列中的所有数字）都只会被 `Contains` 操作访问一次。例如，一旦 1 被处理为起点，2、3、4 都会在内层循环中被访问，但它们不会在外部循环中再次作为起点被处理，因为 `numSet.Contains(num - 1)` 会判断为 `true`。因此，所有元素的总访问次数是 O(N)。
                
        -   **空间复杂度：O(N)**。`HashSet` 存储了所有唯一的数字。
            
-   **思考：** 这个题目再次强调了哈希集合在快速查找元素上的巨大优势。通过巧妙地选择何时开始计算序列长度（即只从序列的起点开始），我们避免了重复计算，从而实现了 O(N) 的时间复杂度。

##### 5. 多数元素 (Majority Element)

-   题目描述：
    
    给定一个大小为 n 的数组 nums，返回其中的多数元素。多数元素是指在数组中出现次数 大于 n/2 的元素。
    
    你可以假设数组是非空的，并且给定的数组总是存在多数元素。
    
    示例：
    
    输入: nums = [3, 2, 3]
    
    输出: 3
    
-   解题思路 1：哈希表计数
    
    这是最直观的解法。用哈希表统计每个元素的出现次数，然后遍历哈希表，找出计数大于 n/2 的元素。
    
    **步骤：**
    
    1.  创建一个 `Dictionary<int, int>` `counts` 来存储每个数字的出现次数。
        
    2.  遍历数组 nums：
        
        a. 对于每个 num，增加其在 counts 中的计数。
        
        b. 如果 counts[num] 已经大于 nums.Length / 2，立即返回 num (因为题目保证存在多数元素)。
        
    3.  （如果循环结束还没返回，说明有逻辑问题或题目假设有变，但通常不会）
        
    
    -   **代码实现：**
        
        C#
        
        ```
        using System.Collections.Generic;
        
        public class Solution {
            public int MajorityElement(int[] nums) {
                Dictionary<int, int> counts = new Dictionary<int, int>();
                int n = nums.Length;
        
                foreach (int num in nums) {
                    counts[num] = counts.GetValueOrDefault(num, 0) + 1; // 获取或默认值
                    if (counts[num] > n / 2) {
                        return num;
                    }
                }
                // 题目保证存在多数元素，所以不会执行到这里
                return -1;
            }
        }
        
        ```
        
    -   **复杂度分析：**
        
        -   **时间复杂度：O(N)**。遍历一次数组，哈希表操作平均 O(1)。
            
        -   **空间复杂度：O(N)**。在最坏情况下（所有元素都不相同），哈希表会存储所有 N 个元素。
            
-   解题思路 2：摩尔投票法 (Boyer-Moore Majority Vote Algorithm) - 拓展
    
    这是一个非常巧妙且高效的算法，可以在 O(N) 时间复杂度和 O(1) 空间复杂度下找到多数元素。它利用了多数元素出现次数超过一半的特性。
    
    核心思想：
    
    我们维护一个 candidate (候选人) 和一个 count (计数器)。
    
    -   遍历数组：
        
        -   如果 `count` 为 0，则将当前元素设为 `candidate`。
            
        -   如果当前元素等于 `candidate`，`count++`。
            
        -   如果当前元素不等于 `candidate`，`count--`。
            
    -   最终 `candidate` 就是多数元素。
        
    
    为什么有效？
    
    想象一下，多数元素就像一个“王者”，它的支持者比其他所有元素的反对者加起来还要多。每次遇到一个“支持者”，它的“票数”就加一；遇到一个“反对者”，它的“票数”就减一。由于王者支持者众多，无论如何抵消，最终王者的票数都不会归零，而其他元素的票数最终都会被抵消掉。
    
    -   **代码实现：**
        
        C#
        
        ```
        public class Solution {
            public int MajorityElement(int[] nums) {
                int candidate = nums[0]; // 初始候选人
                int count = 1;          // 初始计数为 1
        
                for (int i = 1; i < nums.Length; i++) {
                    if (count == 0) {
                        candidate = nums[i]; // 计数为 0，更新候选人
                        count = 1;
                    } else if (nums[i] == candidate) {
                        count++; // 相同，计数增加
                    } else {
                        count--; // 不同，计数减少
                    }
                }
                return candidate;
            }
        }
        
        ```
        
    -   **复杂度分析：**
        
        -   **时间复杂度：O(N)**。只遍历一次数组。
            
        -   **空间复杂度：O(1)**。只使用了常数个额外变量。
            
-   **思考：** 摩尔投票法是一个非常精妙的算法，虽然它不直接使用哈希表，但它提供了一个在特定条件下（存在多数元素）比哈希表更优的空间复杂度解决方案。理解这两种解法能让你在面试中展示更全面的知识储备。

### 字符串（String）

字符串在编程中无处不在，Unity 中更是如此（例如 `GameObject.name`、UI 文本、网络消息）。

#### 特性：不可变性（Immutable）

在 C# 中，字符串是**不可变（Immutable）**的。这意味着一旦一个字符串对象被创建，它的内容就不能被改变。

-   当你对字符串进行拼接、替换等操作时，实际上是创建了一个**新的**字符串对象，而不是修改了原字符串。
    
-   这种特性使得字符串在多线程环境下更安全，但也可能导致性能问题，尤其是在大量字符串操作时产生过多中间对象，造成垃圾回收（GC）开销。
    
-   因此，当需要频繁修改字符串内容时，应该使用 `System.Text.StringBuilder` 类。
    

#### 字符串常用操作

-   **拼接**：`string1 + string2` 或 `string.Concat()`。`StringBuilder.Append()` 是高效的拼接方式。
    
-   **查找**：`string.Contains()`, `string.IndexOf()`, `string.StartsWith()`, `string.EndsWith()`。
    
-   **替换**：`string.Replace()`。
    
-   **分割**：`string.Split()`。
    
-   **截取子串**：`string.Substring()`。
    
-   **大小写转换**：`string.ToLower()`, `string.ToUpper()`。
    
-   **去空格**：`string.Trim()`, `string.TrimStart()`, `string.TrimEnd()`。
    

#### 字符串与双指针

双指针是处理字符串问题非常常用的技巧，可以实现原地操作或高效遍历。

##### 1. 反转字符串 (Reverse String)

-   题目描述：
    
    编写一个函数，其作用是将输入的字符串反转过来。输入字符串以字符数组 s 的形式给出。
    
    不要为另一个数组分配额外的空间，你必须原地修改输入数组、使用 O(1) 的额外空间解决这一问题。
    
    示例：
    
    输入: s = ["h", "e", "l", "l", "o"]
    
    输出: ["o", "l", "l", "e", "h"]
    
-   解题思路：双指针原地反转
    
    这是最简单的双指针应用。使用两个指针，一个从头开始，一个从尾部开始，不断交换它们指向的字符，直到它们相遇或交错。
    
    **步骤：**
    
    1.  初始化 `left = 0`，`right = s.Length - 1`。
        
    2.  循环直到 left < right：
        
        a. 交换 s[left] 和 s[right]。
        
        b. left++，right--。
        
    
    -   **代码实现：**
        
        C#
        
        ```
        public class Solution {
            public void ReverseString(char[] s) {
                int left = 0;
                int right = s.Length - 1;
        
                while (left < right) {
                    // 交换 s[left] 和 s[right]
                    char temp = s[left];
                    s[left] = s[right];
                    s[right] = temp;
        
                    left++;
                    right--;
                }
            }
        }
        
        ```
        
    -   **复杂度分析：**
        
        -   **时间复杂度：O(N)**。遍历了数组的一半。
            
        -   **空间复杂度：O(1)**。只使用了常数个额外变量。

##### 2. 验证回文串 (Valid Palindrome)

-   题目描述：
    
    如果在将所有大写字母转换为小写字母、并移除所有非字母数字字符之后，短语是回文式的，那么它就是 回文的 。
    
    给定一个字符串 s，返回 true 如果它是回文的，否则返回 false 。
    
    示例：
    
    输入: s = "A man, a plan, a canal: Panama"
    
    输出: true
    
    解释："amanaplanacanalpanama" 是回文串。
    
-   解题思路：双指针 + 字符过滤
    
    这道题需要结合双指针和字符判断技巧。我们只关心字母和数字字符，且不区分大小写。
    
    **步骤：**
    
    1.  初始化 `left = 0`，`right = s.Length - 1`。
        
    2.  循环直到 left < right：
        
        a. 移动 left 指针： 如果 s[left] 不是字母数字字符，left++，继续循环。
        
        b. 移动 right 指针： 如果 s[right] 不是字母数字字符，right--，继续循环。
        
        c. 如果 s[left] 和 s[right] 都是字母数字字符：
        
        i. 将它们转换为小写。
        
        ii. 如果它们不相等，返回 false。
        
        iii. 如果相等，则 left++，right--。
        
    3.  循环结束后，如果都没有返回 `false`，则说明是回文串，返回 `true`。
        
    
    -   **C# 字符判断辅助函数：**
        
        -   `char.IsLetterOrDigit(char c)`：判断字符是否为字母或数字。
            
        -   `char.ToLower(char c)`：将字符转换为小写。
            
    -   **代码实现：**
        
        C#
        
        ```
        using System;
        
        public class Solution {
            public bool IsPalindrome(string s) {
                int left = 0;
                int right = s.Length - 1;
        
                while (left < right) {
                    // 移动 left 指针，直到指向字母数字字符
                    while (left < right && !char.IsLetterOrDigit(s[left])) {
                        left++;
                    }
                    // 移动 right 指针，直到指向字母数字字符
                    while (left < right && !char.IsLetterOrDigit(s[right])) {
                        right--;
                    }
        
                    // 此时 s[left] 和 s[right] 都是字母数字字符，进行比较
                    if (left < right) { // 避免 left 越过 right
                        if (char.ToLower(s[left]) != char.ToLower(s[right])) {
                            return false; // 不匹配，不是回文
                        }
                        left++;
                        right--;
                    }
                }
                return true; // 遍历结束，是回文
            }
        }
        
        ```
        
    -   **复杂度分析：**
        
        -   **时间复杂度：O(N)**。两个指针最多遍历一次字符串。
            
        -   **空间复杂度：O(1)**。只使用了常数个额外变量。

##### 3. 最长回文子串 (Longest Palindromic Substring)

-   题目描述：
    
    给你一个字符串 s，找到 s 中最长的回文子串。
    
    示例：
    
    输入: s = "babad"
    
    输出: "bab" (注意 "aba" 也是有效答案)
    
-   解题思路：中心扩散法 (Expand Around Center)
    
    暴力法是 O(N3) (遍历所有子串并判断是否回文)。
    
    动态规划是 O(N2) 时间，O(N2) 空间。
    
    中心扩散法 是一种更直观的 O(N2) 时间，O(1) 空间的解法。
    
    **核心思想：** 回文串是关于其中心对称的。中心可能是一个字符（奇数长度的回文串，如 "aba" 的中心是 'b'），也可能是两个字符之间的空隙（偶数长度的回文串，如 "abba" 的中心是 'b' 和 'b' 之间）。我们可以遍历所有可能的中心，然后从中心向两边扩散，检查是否是回文。
    
    **步骤：**
    
    1.  初始化 `start = 0` (最长回文子串的起始索引)，`maxLength = 0` (最长回文子串的长度)。
        
    2.  遍历字符串 s 的每个索引 i，将其作为回文串的中心：
        
        a. 以 i 为中心（奇数长度回文串）：调用辅助函数 ExpandAroundCenter(s, i, i)，得到以此为中心的最长回文子串长度 len1。
        
        b. 以 i 和 i+1 之间为中心（偶数长度回文串）：调用辅助函数 ExpandAroundCenter(s, i, i + 1)，得到以此为中心的最长回文子串长度 len2。
        
        c. 取 len = Math.Max(len1, len2)。
        
        d. 如果 len > maxLength：
        
        i. 更新 maxLength = len。
        
        ii. 更新 start：start = i - (len - 1) / 2。 (这里需要推导一下：奇数长度回文串中心是 i，长度为 len，起始就是 i - (len-1)/2；偶数长度回文串中心是 i 和 i+1之间，长度为 len，起始是 i - (len/2 - 1) + 1 = i - len/2 + 1，这两个公式合并就是 i - (len - 1) / 2）
        
    3.  最后返回 `s.Substring(start, maxLength)`。
        
    
    -   **辅助函数 `ExpandAroundCenter(s, left, right)`：**
        
        -   给定字符串 `s` 和两个指针 `left`, `right`（作为初始中心）。
            
        -   **循环**：当 `left >= 0` 且 `right < s.Length` 且 `s[left] == s[right]` 时：
            
            -   `left--`。
                
            -   `right++`。
                
        -   循环结束后，`left` 和 `right` 指针已经越过了回文串的边界。回文串的长度是 `right - left - 1`。
            
    -   **代码实现：**
        
        C#
        
        ```
        using System;
        
        public class Solution {
            public string LongestPalindrome(string s) {
                if (string.IsNullOrEmpty(s) || s.Length < 1) {
                    return "";
                }
        
                int start = 0; // 最长回文子串的起始索引
                int maxLength = 0; // 最长回文子串的长度
        
                for (int i = 0; i < s.Length; i++) {
                    // 以 i 为中心（奇数长度回文串）
                    int len1 = ExpandAroundCenter(s, i, i);
                    // 以 i 和 i+1 之间为中心（偶数长度回文串）
                    int len2 = ExpandAroundCenter(s, i, i + 1);
        
                    int len = Math.Max(len1, len2);
        
                    // 如果当前找到的回文串更长
                    if (len > maxLength) {
                        maxLength = len;
                        // 计算起始索引: i 是中心点，(len - 1) / 2 是到左边界的距离
                        start = i - (len - 1) / 2;
                    }
                }
                return s.Substring(start, maxLength);
            }
        
            // 辅助函数：从中心向两边扩散，返回回文串的长度
            private int ExpandAroundCenter(string s, int left, int right) {
                while (left >= 0 && right < s.Length && s[left] == s[right]) {
                    left--;
                    right++;
                }
                // 退出循环时，left 和 right 已经越界，
                // 实际回文串长度是 (right - 1) - (left + 1) + 1 = right - left - 1
                return right - left - 1;
            }
        }
        
        ```
        
    -   **复杂度分析：**
        
        -   **时间复杂度：O(N2)**。外层循环遍历 N 个中心点，内层 `ExpandAroundCenter` 函数在最坏情况下遍历 N/2 次（即扩散到字符串两端）。
            
        -   **空间复杂度：O(1)**。只使用了常数个额外变量。
            
-   **思考：** 中心扩散法是解决回文子串问题的通用且高效的策略。理解如何处理奇数和偶数长度的回文串是关键。

##### 1. 找到字符串中所有字母异位词 (Find All Anagrams in a String)

-   题目描述：
    
    给定两个字符串 s 和 p，找到 s 中所有 p 的 字母异位词 的子串，返回这些子串的起始索引。不考虑答案输出的顺序。
    
    字母异位词 指字母相同，但排列不同的字符串。
    
    示例：
    
    输入: s = "cbaebabacd", p = "abc"
    
    输出: [0, 6]
    
    解释:
    
    起始索引等于 0 的子串是 "cba", 它是 "abc" 的异位词。
    
    起始索引等于 6 的子串是 "bac", 它是 "abc" 的异位词。
    
-   解题思路：滑动窗口 + 字符计数哈希表
    
    这道题是滑动窗口的又一个经典应用。我们需要比较窗口内的字符计数是否与 p 的字符计数相同。
    
    **步骤：**
    
    1.  处理边界情况：如果 `s.Length < p.Length`，直接返回空列表。
        
    2.  统计 `p` 的字符频率：创建一个 `int[] pCount` (长度 26) 来存储 `p` 中每个字符的频率。
        
    3.  初始化滑动窗口：
        
        a. 创建一个 int[] sCount (长度 26) 来存储当前窗口内字符的频率。
        
        b. 初始化 left = 0。
        
        c. 将字符串 s 的前 p.Length 个字符的频率统计到 sCount 中。
        
    4.  检查初始窗口：如果 `sCount` 和 `pCount` 相同，将 `0` 添加到结果列表 `result` 中。
        
    5.  滑动窗口： 遍历 s，right 从 p.Length 到 s.Length - 1：
        
        a. 窗口扩张： s[right] 入窗，增加 sCount[s[right]]。
        
        b. 窗口收缩： s[left] 出窗，减少 sCount[s[left]]，然后 left++。
        
        c. 每次滑动后，检查当前窗口 sCount 是否与 pCount 相同。如果相同，将 left 添加到 result 中。
        
    
    -   **如何判断两个字符计数数组是否相同？**
        
        -   可以直接循环比较两个数组的 26 个位置。
            
        -   或者维护一个 `matchCount` 变量，记录有多少个字符的频率是匹配的。每次 `sCount` 某个字符频率发生变化时，更新 `matchCount`。当 `matchCount` 达到 26 时，说明完全匹配。
            
    -   **代码实现 (使用字符计数数组进行比较)：**
        
        C#
        
        ```
        using System.Collections.Generic;
        
        public class Solution {
            public IList<int> FindAnagrams(string s, string p) {
                IList<int> result = new List<int>();
                if (s.Length < p.Length) {
                    return result;
                }
        
                int[] pCount = new int[26];
                int[] sCount = new int[26];
        
                // 统计 p 的字符频率
                foreach (char c in p) {
                    pCount[c - 'a']++;
                }
        
                // 初始化第一个窗口（长度为 p.Length）
                for (int i = 0; i < p.Length; i++) {
                    sCount[s[i] - 'a']++;
                }
        
                // 检查初始窗口是否是异位词
                if (IsAnagram(pCount, sCount)) {
                    result.Add(0);
                }
        
                // 滑动窗口
                for (int right = p.Length; right < s.Length; right++) {
                    // 窗口扩张：right 指针指向的字符入窗
                    sCount[s[right] - 'a']++;
                    // 窗口收缩：left 指针指向的字符出窗
                    sCount[s[right - p.Length] - 'a']--; // right - p.Length 是当前窗口最左边的字符索引
        
                    // 检查当前窗口是否是异位词
                    if (IsAnagram(pCount, sCount)) {
                        result.Add(right - p.Length + 1); // 当前窗口的起始索引
                    }
                }
        
                return result;
            }
        
            // 辅助函数：判断两个字符计数数组是否相同
            private bool IsAnagram(int[] pCount, int[] sCount) {
                for (int i = 0; i < 26; i++) {
                    if (pCount[i] != sCount[i]) {
                        return false;
                    }
                }
                return true;
            }
        }
        
        ```
        
    -   **复杂度分析：**
        
        -   **时间复杂度：O(N+K)**，其中 N 是 `s` 的长度，K 是 `p` 的长度。
            
            -   初始化 `pCount`：O(K)。
                
            -   初始化 `sCount`：O(K)。
                
            -   滑动窗口：O(N−K) 次循环，每次循环 O(1) 的字符计数操作，以及 O(26) 的 `IsAnagram` 比较。因此总时间复杂度是 O(K+(N−K)cdot26)，简化为 O(N)。
                
        -   **空间复杂度：O(C)**，其中 C 是字符集大小（26）。用于存储 `pCount` 和 `sCount`。
            
-   **思考：** 这个题目是滑动窗口和哈希表（或字符计数数组）结合的经典范例，用于在字符串中查找满足特定条件的子串。理解如何高效地维护窗口内的状态是关键。

##### 2. 最小覆盖子串 (Minimum Window Substring)

-   题目描述：
    
    给你一个字符串 s 、一个字符串 t 。返回 s 中涵盖 t 所有字符的最小子串。如果 s 中不存在涵盖 t 所有字符的子串，则返回空字符串 "" 。
    
    注意：
    
    对于 t 中重复字符，若两个字符都出现在 t 中，则子串中两个字符也必须都出现。
    
    示例：
    
    输入: s = "ADOBECODEBANC", t = "ABC"
    
    输出: "BANC"
    
    解释: 最小覆盖子串 "BANC" 包含 'A'、'B'、'C'。
    
-   解题思路：滑动窗口 + 哈希表 + 计数器
    
    这道题是滑动窗口的最高难度之一，非常考验对窗口扩张、收缩和计数状态维护的理解。
    
    **核心思想：**
    
    1.  用哈希表 `tMap` 记录 `t` 中每个字符的频率（目标频率）。
        
    2.  用哈希表 `windowMap` 记录当前滑动窗口中每个字符的频率。
        
    3.  维护一个 `matchCount` 变量，表示当前窗口中有多少个字符（及其频率）已经满足了 `t` 的要求。
        
    4.  右指针 `right` 扩张窗口，左指针 `left` 收缩窗口。
        
    
    **步骤：**
    
    1.  预处理：
        
        a. 创建一个 Dictionary<char, int> tMap 存储 t 中每个字符及其所需频率。
        
        b. 如果 tMap 为空，或 s 为空，或 s.Length < t.Length，直接返回 ""。
        
    2.  初始化：
        
        a. left = 0 (窗口左边界)。
        
        b. minLength = int.MaxValue (最小覆盖子串长度)。
        
        c. startIndex = -1 (最小覆盖子串的起始索引)。
        
        c. Dictionary<char, int> windowMap (当前窗口字符频率)。
        
        d. matchCount = 0 (当前窗口中满足 t 要求的独立字符个数)。
        
    3.  右指针 right 扩张窗口： 遍历 s，right 从 0 到 s.Length - 1：
        
        a. 获取当前字符 charR = s[right]。
        
        b. 将 charR 加入 windowMap (频率加 1)。
        
        c. 判断 charR 是否“有效”： 如果 tMap.ContainsKey(charR) 并且 windowMap[charR] <= tMap[charR]，说明 charR 是 t 中需要的字符，并且当前窗口中的数量还没超过 t 的需求，matchCount++。
        
        d. 判断何时收缩窗口： 当 matchCount == tMap.Count 时，表示当前窗口已经覆盖了 t 的所有字符。此时，尝试收缩窗口以寻找更小的子串：
        
        i. 更新最小长度： 如果 right - left + 1 < minLength，更新 minLength 和 startIndex。
        
        ii. 获取窗口最左边字符 charL = s[left]。
        
        iii. left++ (左边界右移)。
        
        iv. 更新 matchCount： 如果 tMap.ContainsKey(charL) 并且 windowMap[charL] <= tMap[charL]，说明移出去的这个 charL 刚好是 t 中所需要的（或者说它出去了会破坏匹配），matchCount--。
        
        v. 将 charL 从 windowMap 中移除 (频率减 1)。
        
        vi. 重复收缩： 继续 while 循环，直到 matchCount < tMap.Count（即当前窗口不再完全覆盖 t 的字符）。
        
    4.  循环结束后，如果 `minLength` 仍然是 `int.MaxValue`，返回 `""`；否则返回 `s.Substring(startIndex, minLength)`。
        
    
    -   **代码实现：**
        
        C#
        
        ```
        using System.Collections.Generic;
        using System;
        
        public class Solution {
            public string MinWindow(string s, string t) {
                if (s.Length == 0 || t.Length == 0 || s.Length < t.Length) {
                    return "";
                }
        
                // tMap 存储 t 中字符及其所需频率
                Dictionary<char, int> tMap = new Dictionary<char, int>();
                foreach (char c in t) {
                    tMap[c] = tMap.GetValueOrDefault(c, 0) + 1;
                }
        
                // windowMap 存储当前窗口中字符的频率
                Dictionary<char, int> windowMap = new Dictionary<char, int>();
        
                int left = 0;
                int minLength = int.MaxValue;
                int startIndex = -1;
        
                // countMatch 表示当前窗口中，有多少个字符（包括数量）已经满足 t 的需求
                // 例如 t="AABC"，那么需要两个A，一个B，一个C。
                // 如果 windowMap 中 'A' 的数量达到 2，'B' 达到 1，'C' 达到 1，则认为匹配
                int matchCount = 0; // 记录已匹配的 T 中字符种类数 (不是总字符数)
                                    // 更好的做法是记录“所需字符”被满足的总数，这里使用所需字符的唯一种类数
        
                // right 指针扩张窗口
                for (int right = 0; right < s.Length; right++) {
                    char charR = s[right];
                    windowMap[charR] = windowMap.GetValueOrDefault(charR, 0) + 1;
        
                    // 如果当前字符在 tMap 中，且窗口中该字符的数量未超过 tMap 中的需求
                    if (tMap.ContainsKey(charR) && windowMap[charR] == tMap[charR]) {
                        matchCount++; // 增加匹配的字符种类数
                    }
        
                    // 当 matchCount 达到 tMap.Count (所有 T 中的字符种类都已匹配)
                    while (matchCount == tMap.Count) {
                        // 更新最小覆盖子串
                        if (right - left + 1 < minLength) {
                            minLength = right - left + 1;
                            startIndex = left;
                        }
        
                        // 收缩窗口
                        char charL = s[left];
                        windowMap[charL]--; // 移除左侧字符
        
                        // 如果移除的字符是 T 中需要的，并且移除后该字符的数量小于 T 的需求
                        if (tMap.ContainsKey(charL) && windowMap[charL] < tMap[charL]) {
                            matchCount--; // 减少匹配的字符种类数
                        }
                        left++; // 左边界右移
                    }
                }
        
                return startIndex == -1 ? "" : s.Substring(startIndex, minLength);
            }
        }
        
        ```
        
    -   **复杂度分析：**
        
        -   **时间复杂度：O(N+M)**，其中 N 是 `s` 的长度，M 是 `t` 的长度。
            
            -   预处理 `tMap`：O(M)。
                
            -   主循环：`right` 指针遍历 `N` 次。`while` 循环中 `left` 指针也最多遍历 `N` 次。`Dictionary` 操作平均 O(1)。所以总时间是 O(N+M)。
                
        -   **空间复杂度：O(C)**，其中 C 是字符集大小（256，因为可能包含大小写字母、数字和符号）。用于存储 `tMap` 和 `windowMap`。
            
-   **思考：** 最小覆盖子串是滑动窗口的终极考察，它要求你精确地维护窗口内字符的频率和匹配状态。理解 `matchCount` 的意义以及何时扩张、何时收缩窗口，是解决此题的关键。
    

### 总结与练习

本篇我们深入探索了**哈希表（Hash Table）**和**字符串（String）**这两大在数据处理中至关重要的概念。

-   **哈希表**：我们理解了其基于**哈希函数**的 O(1) 平均查找效率，以及通过**链地址法**或**开放寻址法**处理**哈希冲突**的原理。`Dictionary` 和 `HashSet` 是哈希映射和哈希集合在 C# 中的具体体现。它们是实现快速查找、去重、计数和缓存的利器。
    
-   **字符串**：我们强调了 C# 中字符串的**不可变性**，以及何时应该使用 `StringBuilder`。我们还重点学习了字符串操作中**双指针**和**滑动窗口**这两个强大技巧。
    
    -   **双指针**：用于原地操作（如反转字符串）、过滤和比较字符（如验证回文串）、以及从中心扩展（如最长回文子串）。
        
    -   **滑动窗口**：通过维护一个动态窗口来解决子串/子数组问题。结合哈希表（或计数数组），它能在 O(N) 时间内高效解决如查找字母异位词、无重复字符最长子串、最小覆盖子串等复杂问题。

-   **滑动窗口技巧**：处理字符串子串问题，**扩张**和**收缩**窗口逻辑。
    
-   **哈希表与滑动窗口的结合**：高效维护窗口内字符频率或状态。
