# UnLua C++接口暴露机制分析

## 问题背景

主人提出的技术问题：
> "UE的C++接口是怎么暴露给UnLua扫描生成胶水代码的？UnLua作为一个UE插件，它本身是不是就是C++写的，所以能读C++代码？然后就扫描、生成一堆lua胶水代码就行了？"

## 初步分析

### 1. UnLua插件架构
- **语言实现**：UnLua作为UE插件，确实是C++写的
- **插件类型**：UE插件分为运行时插件和编辑器插件
- **集成方式**：通过UE的插件系统集成到项目中

### 2. C++接口暴露机制
- **反射系统**：UE有强大的反射系统（UHT + UPROPERTY/UFUNCTION宏）
- **代码生成**：Unreal Header Tool（UHT）在编译前生成反射代码
- **元数据**：宏标记的类、属性、函数会生成元数据供反射使用

### 3. 可能的扫描机制
- **反射数据读取**：UnLua可能读取UE生成的反射数据（.generated.h文件）
- **运行时查询**：通过UE的反射API查询类、属性、函数信息
- **代码生成**：根据反射信息生成Lua绑定代码

### 4. 胶水代码生成
- **绑定类型**：可能生成静态绑定或动态绑定
- **函数包装**：将C++函数包装为Lua可调用的函数
- **对象生命周期**：处理C++对象和Lua对象的生命周期管理
