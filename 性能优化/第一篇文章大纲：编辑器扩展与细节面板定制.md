

### 第一篇文章大纲：编辑器扩展与细节面板定制

本文将重点介绍如何扩展 Unreal 的编辑器功能，让你的开发工作流更加高效。我们将对比 Unreal 的编辑器扩展方式与 Unity 的 `EditorWindow` 和 `PropertyDrawer`。

1.  **引言：**
    
    -   引擎的编辑器是开发者日常工作的主要界面。
        
    -   通过扩展编辑器，可以创建自定义工具，简化复杂的工作流程。
        
2.  **Unity 编辑器扩展回顾：**
    
    -   **`EditorWindow`：** 讲解如何使用 `EditorWindow` 脚本来创建自定义的编辑器窗口。
        
    -   **`PropertyDrawer`：** 讲解如何使用 `PropertyDrawer` 脚本来定制 Inspector 面板中的属性显示。
        
3.  **Unreal 编辑器扩展详解：**
    
    -   **蓝图工具（Blutility）：**
        
        -   **核心概念：** 讲解 **Blutility** 是一种特殊的蓝图，可以用于在编辑器中执行任务。
            
        -   **使用场景：** 举例说明如何使用 **Blutility** 来批量处理资源、生成关卡对象等。
            
    -   **Editor Utility Widget（编辑器实用工具控件）：**
        
        -   **核心概念：** 讲解 **Editor Utility Widget** 是一种特殊的 UMG 控件，可以在编辑器中显示为一个独立的窗口。
            
        -   **优点：** 允许美术和设计师使用蓝图来创建可视化的自定义工具。
            
    -   **C++ 编辑器模块（Editor Module）：**
        
        -   讲解如何使用 C++ 编写更底层的编辑器扩展，实现更复杂的功能。
            
4.  **细节面板（Details Panel）定制：**
    
    -   **Unreal 的定制方式：**
        
        -   **`DetailCustomization`：** 讲解如何使用 `DetailCustomization` 来完全自定义一个类的细节面板。
            
        -   **`PropertyTypeCustomization`：** 讲解如何使用 `PropertyTypeCustomization` 来定制特定类型的属性。
            
5.  **核心总结：** 总结 Unreal 的编辑器扩展系统提供了蓝图和 C++ 两种方式，这使得不同技术背景的团队成员都可以参与到工具开发中。

### 第二篇文章大纲：引擎插件与构建工具

本文将深入探讨 Unreal 的插件系统和构建工具，这是组织和编译大型项目的基础。我们将对比 Unity 的 UPM 包和其构建流程。

1.  **引言：**
    
    -   插件是封装和分发可复用功能模块的最佳方式。
        
    -   构建工具是确保代码正确编译和链接的关键。
        
2.  **引擎插件（Plugin）：**
    
    -   **Unreal 插件：**
        
        -   **核心概念：** 讲解 Unreal 的**插件**是一个独立的模块，可以包含代码和资源，并可以被多个项目共享。
            
        -   **优点：** * **模块化：** 将项目拆分成独立的模块，便于维护。
            
            -   **可插拔：** 可以轻松地启用或禁用插件，方便测试和版本管理。
                
    -   **Unity UPM 包对比：**
        
        -   讲解 Unity 的 **UPM（Unity Package Manager）**包系统，并与 Unreal 插件进行类比。
            
3.  **构建工具：**
    
    -   **Unreal Build Tool (UBT)：**
        
        -   **核心概念：** 讲解 **UBT** 是 Unreal 的自动化构建工具，它负责管理 C++ 代码的编译和链接。
            
        -   **`*.Build.cs` 文件：** 讲解每个模块都有一个 `.Build.cs` 文件，用于定义其依赖、头文件路径和宏。
            
    -   **Unity 编译流程对比：**
        
        -   讲解 Unity 引擎的自动编译流程。
            
        -   **`Assembly Definition`：** 讲解如何使用 `asmdef` 文件来控制编译边界，从而提升编译速度。
            
4.  **持续集成（CI）与自动化测试：**
    
    -   **Unreal 的自动化流程：**
        
        -   讲解 Unreal 如何使用**命令行工具**（如 UBT 和 UAT）来自动化打包和测试流程。
            
        -   **Automation Tests：** 讲解 Unreal 的**自动化测试**框架，用于编写单元测试和功能测试。
            
    -   **Unity 的自动化流程对比：**
        
        -   讲解 Unity 的 **Test Framework** 和命令行模式如何用于自动化测试和 CI。
            
5.  **核心总结：** 总结 Unreal 的插件和构建工具为大型项目的开发提供了强大的支持，尤其是在模块化和自动化方面。

### 第三篇文章大纲：源码定制、调试与日志系统

本文将深入探讨 Unreal 开放源码的优势，并详细讲解其强大的调试和日志系统，这是解决复杂问题的必备技能。

1.  **引言：**
    
    -   开放源码是 Unreal 的一大优势，它允许开发者对引擎进行深度定制。
        
    -   好的调试和日志系统是提高开发效率的保障。
        
2.  **源码定制：**
    
    -   **Unreal 源码开放：**
        
        -   **核心概念：** 讲解 Unreal 的引擎源码是公开的，开发者可以根据需要修改引擎代码或提交补丁。
            
        -   **优点：** 允许开发者修复引擎 bug、添加新功能，或为特定项目进行性能优化。
            
    -   **Unity 源码定制对比：**
        
        -   讲解 Unity 只有少数开发者能够获得引擎源码，大多数开发者无法修改引擎内部。
            
3.  **调试手段：**
    
    -   **Unreal 的调试：**
        
        -   **C++：** 讲解如何使用 **Visual Studio** 或 **Rider** 附加到 Unreal 进程进行 C++ 代码的调试。
            
        -   **蓝图：** 讲解如何在编辑器中直接对蓝图进行断点调试。
            
    -   **Unity 的调试对比：**
        
        -   讲解 Unity 开发者如何使用 VS 或 Rider 附加到编辑器进行 C# 脚本调试。
            
4.  **日志系统：**
    
    -   **Unreal `UE_LOG`：**
        
        -   **核心概念：** 讲解 **`UE_LOG`** 宏，它允许开发者将日志分类（Category）并分级（Verbosity）。
            
        -   **输出：** 讲解日志会输出到 **Output Log** 窗口，并且可以被过滤和保存。
            
    -   **Unity `Debug.Log` 对比：**
        
        -   讲解 Unity 的 `Debug.Log`、`Debug.LogWarning` 和 `Debug.LogError`。
            
5.  **配置管理：**
    
    -   **Unreal `.ini` 文件：**
        
        -   讲解 Unreal 如何通过 `.ini` 文件来分层管理配置，并且不同平台的配置可以相互覆盖。
            
    -   **Unity `ProjectSettings`：**
        
        -   讲解 Unity 的配置主要保存在 **ProjectSettings** 资产中。
            
6.  **核心总结：** 总结 Unreal 的开放源码、强大的调试和日志系统，为开发者提供了极大的便利，使其在处理复杂问题和定制化需求时更加得心应手。
